## What was built

**MoaSwitcher (`moa-switcher.tsx`)**
- `currentUserId: string` required prop 추가. 게스트 마운트(/t)는 `moas` 미전달 → 정적 pill 조기 반환이라 버튼이 애초에 렌더되지 않는다.
- `rows` 로컬 미러(`useState<Trip[]>(moas ?? [])` + `useEffect` 재동기화) — 낙관적 편집/삭제를 담고, `router.refresh()`로 RSC가 새 배열을 내려주면 서버 진실로 되돌아간다.
- pill 제목을 `rows`에서 파생(`shownTitle`) — 리네임 직후 상단 pill도 즉시 바뀐다.
- **행 구조(핵심):** 행을 `flex` div로 감싸고 `DropdownMenuItem`(이름 영역, `min-w-0 flex-1`)과 연필·X 버튼을 **형제**로 두었다. 버튼이 Item의 자손이 아니므로 Radix가 selection 이벤트를 만들 수 없다 → 연필을 눌러도 메뉴가 닫히지 않는다.
- **입력창 keyDown:** ① `stopPropagation()`(Radix Content typeahead가 한글 타이핑 중 포커스를 항목으로 튕기는 것 차단) → ② `isComposing || keyCode === 229` 가드(조합 확정용 Enter를 저장으로 오인 금지) → ③ Enter=저장 / Escape=취소.
- `DropdownMenu onOpenChange` — 메뉴가 닫히면 `editingId` 리셋(다음 오픈 시 유령 입력창 방지).
- 저장: `draft.trim()`이 빈 문자열이거나 무변경이면 저장 없이 종료. 아니면 낙관적 갱신 → `TripUpdateSchema.parse({ title })`(길이 상한 위반 시 throw) → `updateTrip` → `router.refresh()`. 실패 시 롤백 + `toast('이름을 바꾸지 못했어요', { variant: 'error' })`.
- 삭제: `window.confirm('...담긴 장소도 함께 사라져요.')` 승인 시에만 낙관적 제거 → `deleteTrip`. 실패 시 롤백 + 에러 토스트. 성공 시 현재 모아면 `router.push('/moa')`(리다이렉트 허브에 위임), 아니면 `router.refresh()`.

**MoaIsland** — `<MoaSwitcher ... currentUserId={currentUserId} />` 한 줄 추가. 그 외 diff 0.

**테스트** — 기존 5케이스는 `currentUserId="u1"` prop만 추가(assertion 무변경). 신규 10케이스: 소유자 게이팅(혼합 목록) · 형제 구조(`within(menuitem).queryByLabelText('이름 수정') === null` + 연필 클릭 후 `push` 미호출) · 리네임(`updateTrip(client,'t1',{title:'새 이름'})` + 목록·pill 반영) · IME 조합 Enter 무시 · Esc 취소 · 공백 제목 미저장 · 저장 실패 롤백 · confirm 취소/승인(문구 부분 매칭) · 현재 모아 삭제 `push('/moa')` · 삭제 실패 롤백.
