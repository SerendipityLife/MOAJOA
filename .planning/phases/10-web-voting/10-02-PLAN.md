## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser island → @moajoa/api vote/join helpers | User-initiated mutations cross from untrusted client to RLS-gated RPCs |
| SSR cached server page → client island | Session-specific state must stay out of the cached anon fetcher |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-09 | Elevation of Privilege | island casting votes for a non-member | mitigate | castVote hits `votes: insert if can vote in board` RLS → `can_vote_board` requires accepted membership (0001). The island cannot bypass; a non-member's castVote is rejected server-side regardless of client state. |
| T-10-10 | Tampering | optimistic count manipulation in the client | accept | Client-side count is display-only and reconciled by router.refresh() / getVoteCounts (server truth via SECURITY DEFINER RPC). A tampered local count does not persist. |
| T-10-11 | Information Disclosure | session/cookies leaking into SSR cache | mitigate | Island is `'use client'`; `auth.getUser()` runs in the browser only. The server page passes NO session data into the cached fetcher (cache.ts unchanged) — preserves the P0 cookies-in-unstable_cache fix. |
| T-10-12 | Spoofing (XSS) | rendering place/vote text | mitigate | All place names/counts rendered as JSX text (React default escaping); no dangerouslySetInnerHTML. Mirrors place-summary-list T-08-06. |
| T-10-13 | Repudiation | who cast a vote | accept | votes.user_id defaulted to auth.uid() by trigger (0001); votes are attributable server-side. No additional audit needed for v1. |
</threat_model>
