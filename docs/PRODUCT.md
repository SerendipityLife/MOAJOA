# MOAJOA — 제품 설명 (전면 개편 비전)

> 서비스가 무엇이고, 왜 경쟁우위가 되는지, 무엇을 어떤 순서로 만드는지를 한 곳에.
> 이 문서는 **전면 개편(re-개편) 이후의 제품 방향**이 단일 출처예요.
> 개편 *이전*의 구현 베이스라인과 디자인 토큰(색·타이포·간격)은 [docs/PRODUCT-AND-DESIGN.md](PRODUCT-AND-DESIGN.md) 와 [docs/ARCHITECTURE.md](ARCHITECTURE.md) 를 따릅니다.
>
> **상태:** 전면 개편 설계 확정 · **최종 업데이트:** 2026-06-21

---

## 1. 한 줄 정의

**링크 하나로 시작해 예약·정산까지 끝내는 AI 여행 플랫폼.**
콘텐츠에서 발견한 장소를 자동으로 지도·일정으로 만들고, 친구와 정하고, 예약하고, 결제 내역까지 정리한다. 그렇게 쌓인 데이터가 다음 여행을 더 똑똑하게 만든다.

> 포지셔닝: 우리는 **여행에서 돈이 실제로 쓰이는 순간(예약·결제)** 을 제품의 한가운데 둔다. 그 순간이 여행 커머스에서 전환이 가장 비싼 자리다.

---

## 2. 문제

콘텐츠 덕분에 "여기 가고 싶다"를 **발견**하는 건 쉬워졌다. 그러나 그 발견을 실제 **내 여행**으로 바꾸는 과정 — 모으고 · 정하고 · 짜고 · 예약하고 · 정산하는 일 — 은 여전히 수십 개의 앱과 수작업으로 쪼개져 있다.

- 영상·글 속 장소를 캡처해 지도에 일일이 옮겨 적는다.
- 친구와 카톡으로 링크를 흩뿌리며 "여기 어때?"를 반복한다.
- 숙소·티켓·교통을 플랫폼 여러 곳을 돌며 예약한다.
- 예약 메일과 영수증이 메일함에 흩어지고, 카드·통화·환율·결제 시점이 뒤죽박죽이다.

**발견은 넘치는데 실행이 안 되는** 이 간극이 시장이다.

---

## 3. 해법 — 발견부터 정산까지의 자동화 레이어

```
링크 던지기
   ↓
① 발견·수집   AI가 영상·글 속 장소 자동 추출 (좌표·별칭·맥락 포함)
   ↓
② 결정        친구와 보드 공유 → 실시간 투표 → 확정 (옵션)
   ↓
③ 설계(플랜)  확정 장소 + '필수 장소' → 동선·일정 자동 설계
   ↓
④ 예약·결제   멀티플랫폼 가격비교 → 제휴 예약 연결 (수수료 발생)
   ↓
⑤ 정산(가계부) 예약 메일 전달 → AI 파싱 → 카드·통화·환율·결제시점 자동 정리
```

> 시작 앞단: **일정 정하기** — "정해졌으면 바로 입력 / 미정이면 일행과 날짜 투표(초대 링크)". 여기서 대표(결제자)도 식별된다.

---

## 4. 선순환 플라이휠 (핵심 투자 포인트)

발견 → 결정 → 설계 → 예약 → 정산이 한 바퀴 돌 때마다 **실제 예약·사용 데이터**가 쌓이고, 그 데이터가 다음 여행자의 발견·추천을 더 정확하게 만든다. 사용자↑ → 데이터↑ → 추천↑ → 사용자↑.

특히 여행을 마친 뒤의 **가계부(기록)** 가 다음 여행의 **발견** 을 끌어내, 여행이라는 저빈도 서비스의 최대 약점인 **재방문**을 구조적으로 해결한다.

## 5. 제공 기능 (정리본)

`MVP` = 1차 출시 · `이후` = phase 2+

**① 발견·수집** `MVP`
- 링크(유튜브·블로그·인스타) → AI 장소 자동 추출 (좌표·별칭·맥락)
- 지도 보드 자동 생성 · 수동 핀 추가(보조)

**② 결정 (협업)** `MVP`
- 보드 친구 공유(앱 없이 웹 열람) · 실시간 투표 → 확정

**③ 설계 (플랜)** `MVP`
- 확정 장소 + 필수 장소 선택 → 동선·일정 자동 설계 · 날짜별 정리

**④ 예약·결제** `MVP(딥링크)`
- 멀티플랫폼 가격비교 · 제휴 예약(숙소·액티비티·교통·유심)

**⑤ 정산 (여행 가계부)** `MVP`
- 개인 전용 주소로 예약 메일 전달 → AI 파싱 → 가계부 자동 정리
- 카드·플랫폼·통화(₩/$)·환율 차이·결제 시점까지
- 우리 앱을 안 거친 예약(직접 예약 항공권 등)도 포착

**⑥ 여행 당일 실시간** `이후 · 프리미엄`
- 항공 게이트·지연·수하물 벨트 추적 + 알림 (유료 API)
- 공항 주차 혼잡도·만차 알림 (무료 공공데이터)

**⑦ 데이터·확장** `기반 · 장기`
- 데이터 플라이휠(추천 고도화) · 글로벌·다국어 확장(일본 → 동남아·유럽)

## 6. 네비게이션 / IA 결정

- **하단 탭 = 활성 여행의 4단계: `지도 · 플랜 · 예약 · 가계부`** (전역 탭 아님)
- **여행 1개 → 목록 없이 바로 그 여행으로 진입** (4탭이 곧 홈)
- **새 여행 = 탭이 아니라 컨텍스트 액션**: 온보딩 · 여행 종료 후 · 홈 헤더 `＋` · 공유시트
- **여행 전환·내 정보 = 헤더**: 좌상단 `현재 여행 ▾`(전환) / 우상단 프로필
- 진입 로직: 세션 없음 → `welcome` · 여행 1개 → `trip/<id>/plan` · 여러 개 → 마지막 여행
- 라우팅 방향: `app/trip/[id]/(tabs)/{map,plan,book,ledger}` (탭바 항상 유지)

---

## 7. 핵심 UX 결정

- **플랜 = 사용자 트리거 AI 초안.** 추출은 장소를 모으고, plan 탭의 "플랜 만들기"로 AI가 초안을 짠다 (한 trip에 링크 여러 개 모은 뒤). 투표로 보내지 않는다 → 이탈 차단, AI가 대신 짜주는 "아하 순간". *(2026-06-22 변경: "추출 직후 자동" → "사용자 트리거". Phase 18 CONTEXT D-01)*
- **투표는 옵션.** 같은 플랜 위에 "친구와 같이 정하기"로 얹는다 (AI 추천 루트 = 기본 / 투표 기반 = 선택).
- **예약은 두 곳에:** 플랜 안 *인라인 카드*(맥락 노출) + *통합 예약 체크리스트*(대표가 한 번에).
- **가계부는 마찰 0:** 메일 전달 한 번 → AI 파싱. (캡처 여러 장 업로드 X)

## 8. 수익 모델 (BM)

| 수익원 | 내용 | 시점 |
|---|---|---|
| **거래 수수료** (핵심) | 숙소 3~5% · 액티비티 5~8% · eSIM↑ (Travelpayouts·Stay22 → Booking·Klook 등) | **Day 1 (딥링크, 선투자 0)** |
| 프리미엄 구독 | 게이트·주차 실시간 알림, 고급 일정, 광고 제거 | MVP~ |
| 로컬 상점 광고 | 예약·결제 데이터 기반 스폰서 노출 | 데이터 밀도 후 |
| 여행 데이터 | 고순도 인텐트 데이터 B2B (관광청·여행사) | 장기 |

- 블렌디드 수수료 ≈ **4~5%** (실제 제휴율로 검증 필요)
- 단위 경제 스케치(가정): 트립당 우리 경유 예약 ₩300,000 × 4~5% ≈ 트립당 약 ₩13,000~15,000. *수치는 외부 제출 전 실측·재확인.*

## 9. 타깃 · 시장

- **초기 타깃:** 콘텐츠로 직접 여행을 설계하는 2030 한국인 해외 자유여행객
- **시작 시장:** 일본(도쿄) — 한국인은 일본 방문 외국인 중 최대 규모. *정확 통계는 인용 전 재확인.*
- **확장:** 도시(오사카·교토) → 동남아·유럽, 다국어

## 10. 스코프 컷라인

**MVP에 포함**
- 일정 정하기(분기) · 추출 · 사용자 트리거 AI 플랜 · (옵션)투표 · 가격비교/딥링크 예약 · 메일전달 가계부
- iOS 앱 + Web 열람/공유

**MVP에서 제외 (이후)**
- 방문 인증(GPS·영수증) — **제거**
- 별칭 투표 시스템 · 포인트/레벨/리워드 · 장소 리뷰 시스템 — 보류
- 여행 당일 실시간(게이트·주차) — phase 2 프리미엄
- 인앱 네이티브 결제(MOR) · 데이터 라이선싱 — 장기

## 11. 주요 결정 로그

- 방문 인증(GPS·영수증) 제거 → 신뢰 근거는 **실제 예약·결제 데이터 + 콘텐츠 기반 추출**로.
- 추출 후 투표 강제 X → **추출 즉시 플랜**, 투표는 옵션.
- 예약 = **딥링크(Travelpayouts·Stay22)로 MVP 시작**, API/화이트라벨은 트래픽 후.
- 가계부 = **개인 전용 전달 주소**(TripIt/Expensify 패턴) + 기존 추출 AI 재활용 파싱.
- **Android는 BM상 필수** — 대표(결제자)가 Android일 수 있음. (Expo라 재작성 아님 / 임시로 웹 예약 병행 가능)
- 하단 탭 = 여행 4단계, 새 여행 FAB 제거, 1개면 바로 진입.

## 12. 플랫폼

- **iOS (Expo):** 저장·공유·플랜·예약·가계부 (주력)
- **Android (Expo):** 대표(결제자) 대응 — 예약·가계부 1급. (임시: 반응형 Web 예약)
- **Web (Next.js):** 비로그인 공유 열람 + 예약/가계부 백업 경로
