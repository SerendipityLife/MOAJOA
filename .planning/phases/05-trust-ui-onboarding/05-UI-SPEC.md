### Weight assignment policy (Phase 5 확장 lock)

- **400 regular** (기본): 본문, 메타, 토스트, 안내 카드 본문 줄, step indicator 완료 단계, 링크 리스트 행 status 텍스트(상태 단어 제외).
- **500 medium** (NEW — 미세 라벨 위계): **low_confidence 배지**, **step indicator 미래 단계**. 두 곳 모두 12px와 짝지어 사용 — 본문 14px/400 또는 16px/400과 시각적으로 구별되는 "메타 위계 안의 추가 강조" 레이어.
- **600 semibold** (강조): primary CTA, 현재 step 라벨 (step indicator), list-item-title (보드 이름 / 핀 장소명 / 영상 제목), 화면 헤더, 안내 카드 헤더 줄.

### Line height policy

- 본문 (`text-base`, `text-sm`) → `leading-normal` (1.5) — Korean 가독성
- 헤더 (`text-2xl`, `text-lg`) → `leading-tight` (1.25)
- Step indicator 한 줄당 lineHeight 1.4 권장 (시각적 stack tightness — `leading-snug` 또는 `leading-[1.4]`)
- 안내 카드 본문 줄 `leading-normal` (1.5)
- 12px micro 라벨 (`text-xs`) → `leading-tight` (1.25) — 짧은 라벨이므로 컴팩트

### Hierarchy verification (Dimension 1 / 4)

- **Step indicator 4단계 (완료/현재/미래/미래) 한 컬럼:**
  - 완료: 14 / 400 / neutral-500 · ● 채워진 dot · color step
  - 현재: 16 / 600 / brand-500 · ● 채워진 dot · color + size + weight 모두로 우위
  - 미래(2개): 12 / 500 / neutral-300 · ○ 빈 dot · 가장 약함 (size 작음 + weight 중간 + 색 매우 옅음)
  - **3-tier hierarchy 검증:** size step (16→14→12) + weight step (600→400→500) + color step (brand-500 → neutral-500 → neutral-300) + dot 모양 (● / ○). 중복 시각 cue로 사용자가 어느 단계에 있는지 명확.
  - **font-medium(500)이 font-regular(400)보다 시각적으로 더 무겁지만 size가 12px로 작아 위계는 14/400보다 약함** (size dominance 원칙 활용 — checker Dimension 4 cross-check OK).

- **Low_confidence 배지 "신뢰도 낮음" (12 / 500 amber-700 + amber-50 bg) vs source_kind 배지 "AI" (14 / 400 neutral-700 + neutral-100 bg):**
  - 두 배지 모두 chip 형태지만 색·크기·weight 모두로 구분. low_confidence가 더 작지만 색(amber) 강조로 의미 우위 ("주의가 필요함") 전달.
  - 한 행에 두 배지 나란히 (`flex-row gap-1`) — 시각적 충돌 없음.

- **안내 카드 헤더 (14/600 neutral-900) + 본문 (14/400 neutral-700):**
  - size 동일, weight step 200 + color step neutral-900↔700. iOS Phase 3 list-item-title 패턴과 동일.

→ checker Dimension 1 (위계 명확성) + Dimension 4 (typography scale, dimension_override 적용) 통과 예상.

- Toast 완료/실패: `"{N}개 핀 추가됨"`, `"분석 실패: {reason}"`
- Bottom sheet 기본: `"AI"` / `"수동"`, `"이름 수정"`, `"영상에서 위치 보기"`, `"삭제"`
- Destructive confirm: Alert `"핀 삭제"` + `"정말 삭제할까요?"` + `[취소][삭제]`

### Destructive actions in Phase 5

| Action | Confirmation pattern |
|--------|----------------------|
| `[잘못됨]` (low_confidence reject) | **확인 다이얼로그 없음 — 즉시 soft delete (`hidden_at = now()`)**. 사유: 핀 자체가 "낮은 신뢰도" 상태(사용자가 의심하는 핀)이므로 confirm 추가가 마찰. soft delete + v2 undo로 충분. |
| `[삭제]` (영구 삭제, Phase 3 유지) | Phase 3 Alert `"핀 삭제"` + `"정말 삭제할까요?"` + `[취소][삭제]` — 그대로 |
| `× ` 안내 카드 dismiss | **확인 없음 — 즉시 영구 dismiss**. AsyncStorage 글로벌 키. 사유: 카드 자체가 noticeable but reversible 느낌이며 잘못 눌러도 정보 가치 손실 미미. |
| 링크 리스트 row `failed` 탭 → retry | 확인 없음 — 즉시 `triggerExtraction` 재호출. Edge Function idempotency 가드가 있음 (Phase 2 D-02). |

### 카피 가이드라인 (Phase 5 적용)

- 정중체 (`~요/~돼요`) — Phase 3 동일 톤
- step indicator 라벨: 동사 `~중` 형태 — 진행감 강조 ("가져오는 중", "읽는 중", "찾는 중", "표시하는 중")
- 길이: step 라벨 ≤ 14 글자, 배지 ≤ 8 글자, 안내 카드 헤더 ≤ 18 글자 / 본문 ≤ 24 글자
- "AI" 인격화: "AI가 자신 없어해요" 한 곳에만 — 의도적 따뜻함 (사용자 신뢰 회복 surface). 다른 곳에서는 "AI" 라벨로만 등장.
- 부정확/모호 회피: "잠시 후", "곧" OK. 구체 초 표시 X.
