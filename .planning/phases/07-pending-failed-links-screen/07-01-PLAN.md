<objective>
Phase 3에서 만든 "저장 실패 N개" 배너의 **누락된 목적지 화면**을 구현해 깨진 동선을 완성한다. 배너 탭 → `/boards/_failed`는 현재 not-found로 빠진다. 이 plan은 (1) 실패 항목을 보여주는 풀스크린 라우트 `app/boards/failed.tsx`를 새로 만들고, (2) 배너의 push 타깃을 `/boards/failed`로 고치고, (3) 행별/전체 재시도 + 스와이프 삭제 + 실행취소를 기존 `lib/pending.ts` 함수에 배선한다.

신규 도메인 로직은 거의 없다 — `listFailedPending` / `retryFailedPending` / `deleteFailedPending` / `drainPendingLinks`는 이미 존재한다. 핵심은 **UI 배선 + 순수 표시 헬퍼 2개**(사유 한국어 매핑, 상대시각)다.

Purpose: 사용자가 실패한 저장 링크의 사유를 확인하고 직접 재시도/삭제할 수 있게 해 dogfooding 중 "저장이 묵묵부답"인 막다른 길을 없앤다.
Output: 새 라우트 화면 1개 + 순수 헬퍼 모듈 1개 + `FailedPendingLink` export + 배너 타깃 수정 + 단위/렌더 테스트.
</objective>
