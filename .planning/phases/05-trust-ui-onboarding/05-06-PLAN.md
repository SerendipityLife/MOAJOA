# 05-06-PLAN — Onboarding Card Banner (ONBOARD-02)

**Wave:** 5 (last iOS plan — depends 05-04 for [id].tsx sequential ownership)
**Owners:** iOS
**Requirements covered:** ONBOARD-02 (ONBOARD-01 backend 부분은 05-01 Part 3/4가 처리)

## Goal

빈 보드(places.length=0 AND links.length=0) 진입 시 URL TextInput 위에 dismissible 안내 카드를 표시. dismiss는 AsyncStorage 글로벌 키로 영구. iOS-only (Web 변경 0).

UI-SPEC §"Component States" 6 + CONTEXT D-19/D-20/D-21.

## Why

- ONBOARD-02: 첫 보드 진입 시 "유튜브 링크 붙여넣어 보세요" — 빈 상태 마찰 제거
- D-21: 모든 빈 보드에서 표시 — 첫 보드 한정 X (구현 단순 + 학습 가치 동일)
- D-20: 글로벌 1회 — 보드별 키는 v1 과잉
- AsyncStorage dep는 monorepo에 이미 존재 (`apps/ios/package.json:23` 2.2.0 — 검증 완료, 추가 dep 작업 X)

## Verification Loop

```
1. 신규 가입 또는 AsyncStorage clear → 첫 보드("내 첫 여행") 진입
2. URL TextInput 위에 amber 카드 표시 — 헤더·본문·× 가시
3. 다른 보드(빈, 다른 ID) 진입 — 카드 동일 표시
4. URL 추가 → 카드 즉시 숨김 (links.length > 0 분기)
5. URL 삭제 후 다시 빈 상태 — AsyncStorage falsy면 다시 표시 (dismiss 안 했으니)
6. × 탭 → 즉시 unmount, AsyncStorage 'true' set
7. 앱 재시작 → 빈 보드 진입 — 카드 표시 안 됨
8. AsyncStorage.removeItem(OnboardKeys.LinkCardDismissed) (개발자 도구) → 다시 표시
```

## Tasks

### T1 — `lib/onboarding.ts` AsyncStorage wrapper (신규)

**File:** `apps/ios/lib/onboarding.ts` (NEW — UI-SPEC §"Design System" + canonical_refs 명시)

```ts
import AsyncStorage from '@react-native-async-storage/async-storage';
import { OnboardKeys } from '@moajoa/core';

/**
 * Phase 5 ONBOARD-02 — D-20.
 * Type-safe wrapper around AsyncStorage for onboarding dismiss flags.
 * Keys come from @moajoa/core to keep storage layout in one place.
 */
export async function isLinkCardDismissed(): Promise<boolean> {
  try {
    const v = await AsyncStorage.getItem(OnboardKeys.LinkCardDismissed);
    return v === 'true';
  } catch (e) {
    console.warn('[onboarding] read failed, treating as not dismissed:', e);
    return false;
  }
}

export async function dismissLinkCard(): Promise<void> {
  try {
    await AsyncStorage.setItem(OnboardKeys.LinkCardDismissed, 'true');
  } catch (e) {
    console.warn('[onboarding] write failed:', e);
  }
}
```

**Verify:** TS compiles. AsyncStorage SDK 54 호환 (이미 monorepo `2.2.0`).

> **Position decision:** UI-SPEC wireframe은 카드를 헤더 직후 + URL input 위. `mx-6` 카드 자체에서 좌우 margin 처리 — `View className="px-6"` 바깥에 위치해야 alignment 자연. URL input 바로 위에 카드를 두면 헬퍼 텍스트와 자연 연결.

**Surgical:** 다른 코드 변경 0. `linkCardDismissed`가 null인 동안(첫 paint) 카드 미표시 — flicker 회피.

## Risk / Pitfalls

- **AsyncStorage `null` initial state flicker:** loading 상태(null) 동안 카드 미표시 — 안전. dismiss=true 사용자에게 잠깐 표시 후 사라지는 깜빡임 회피.
- **handle_new_auth_user + first_board 트리거 race:** 05-01에서 같은 트랜잭션 보장. 신규 가입 → 즉시 보드 목록에 "내 첫 여행" 1개 — empty state 안 보임. 본 plan 클라이언트 코드는 보드 진입 후이므로 race 영향 X.
- **OnboardKeys 충돌:** 키는 `@moajoa/onboard:link_card_dismissed` — namespace prefix로 다른 AsyncStorage 사용처(예: Supabase auth)와 충돌 X.
