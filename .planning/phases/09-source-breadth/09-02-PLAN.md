## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Edge Function → external blog host | Server-side `fetch()` of user-supplied blog URL — primary SSRF surface |
| extracted HTML → bodyText | Untrusted remote HTML parsed into text fed to the LLM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure / Tampering (SSRF) | fetchBlogContent fetch | mitigate | `assertFetchableUrl(url)` (Plan 01) called BEFORE fetch — rejects private/loopback/metadata IPs + non-http(s). Test 5 asserts reject before mockFetch invoked. Verifiable: `grep assertFetchableUrl blog.ts` + deno test. |
| T-09-02 | Tampering (script injection from remote HTML) | DOMParser / Readability output | mitigate | Only `textContent` is extracted (never `article.content` HTML); plain text flows to Claude. No rendering of remote HTML. Verifiable: adapter returns `textContent`, no `innerHTML`/`dangerouslySetInnerHTML`. |
| T-09-04 | Denial of Service (slow/huge response) | blog fetch | mitigate | `AbortSignal.timeout(10_000)` caps hang; downstream `claude.ts:134` 12000-char slice caps LLM input. Verifiable: `grep AbortSignal.timeout blog.ts`. |
| T-09-05 | Info Disclosure (Korean mojibake / wrong charset) | response decode | accept→mitigate | EUC-KR/MS949 defensive decode (Pitfall 3); UTF-8 default otherwise. Test 4 asserts no mojibake. Not a security threat per se but a correctness guard tracked here for completeness. |
