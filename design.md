> 서비스 색상 시스템의 단일 진실 공급원(single source of truth).
> 2026-07-18 · **Indigo Mono** 팔레트 도입 (출처: `references/reference_01`, RealtimeColors).

## 1. 브랜드 컬러

MOAJOA는 **하나의 인디고 hue(~243°)** 로 정의된다. 파랑·노랑 두 액센트가 서로 부딪히던 구
팔레트를 폐기하고, 단색 계열로 통일해 이질감을 없앴다.

| | 이름 | HEX | 역할 |
|---|---|---|---|
| 🔵 | **Primary** | `#2F27CE` | **주 액션** — 버튼·FAB·링크·지도 핀. 흰 글씨 9.07:1 ✅ |
| 🟣 | **Secondary** | `#DEDCFF` | **연한 면** — 활성 탭 필, 보조 CTA 계열, 캔버스 |
| ⚡ | **Accent** | `#433BFF` | **하이라이트** — 포커스 링, 활성 표시, 링크 강조 |
| ✒ | **Text / Background** | `#050315` / `#FBFBFE` | 잉크 / 배경 (쿨) |

**핵심 원칙 — 명도로 위계를 판단한다:**

- 단일 hue라 "무엇이 강조인가"를 **명도**로만 읽는다. 진한 인디고(`brand-600`)=액션,
  밝은 인디고(`brand-500`)=포커스/활성, 연한 라벤더(`100`/`200`)=면.
- Secondary(라벤더)는 **글씨가 아니라 면**이다. 실제로 쓰는 구간은 램프 상단이다.
- 구 팔레트의 "블루=담기 / 노랑=함께하기" **2색 액션 구조는 폐기**됐다. 이제 함께하기 CTA도
  같은 인디고 계열(라벤더 면)이라 두 액션이 서로 이질적으로 보이지 않는다.

> **구 팔레트(Banana Mania × Royal Blue)와의 차이:** 구 Royal Blue `#4886E4`는 흰 글씨 3.62:1로
> AA 미달이라 *"500=아이덴티티 / 600=액션"* 으로 쪼개야 했다. 새 Primary `#2F27CE`는 9.07:1이라
> **그 분리 규칙은 폐기됐다.** 노랑(Banana)은 완전히 제거됐고, 경고색(amber)만 시맨틱으로 남는다.

## 3. 접근성 규칙 (필수)

모든 값은 WCAG 2.1 상대 휘도 공식으로 실측 검증됨. **이 표를 어기는 조합은 머지하지 않는다.**

| 조합 | 비율 | 판정 | 규칙 |
|---|---|---|---|
| 잉크 `#050315` on 배경 `#FBFBFE` | **19.5:1** | ✅ AAA | 본문 기본값 |
| 흰 글씨 on `brand-600` (Primary) | **9.07:1** | ✅ AAA | **프라이머리 버튼·핀·링크의 기본값** |
| 흰 글씨 on `brand-500` (Accent) | **6.32:1** | ✅ AA | 액센트 solid |
| `brand-700` on `banana-200` (라벤더 필) | **8.57:1** | ✅ AAA | 활성 탭 라벨 |
| `brand-600` on `banana-100` (라벤더 캔버스) | **7.78:1** | ✅ AAA | 라벤더 면 위 파란 본문/링크 |
| 잉크 on `banana-400` (세컨더리 CTA) | **9.78:1** | ✅ AAA | [함께 정하기] |
| `neutral-500` on 흰 배경 | **6.37:1** | ✅ AA | 보조 텍스트 |
| 흰 글씨 on `brand-400` | 4.2:1 | ⚠ 대형만 | 장식 아이콘 전용, 본문 금지 |

**요약 3줄:**
1. 버튼·핀·링크는 **`brand-600` + 흰 글씨** (9.07:1). 구 팔레트의 "버튼엔 500 금지" 규칙은 이제 없다.
2. 라벤더 면 위에는 **잉크 아니면 `brand-700`**. 둘 다 8:1을 넘는다.
3. 라벤더(`banana-*`)는 **면이지 글씨가 아니다.** `brand-400` 이하도 텍스트로 쓰지 않는다.

## 4. 서피스 전략 — 듀오(Duo)

MOAJOA는 지도 제품이다. 지도 타일 자체가 컬러풀하므로 **지도 주변은 중립적이어야 하고, 지도 밖에서는
브랜드가 마음껏 숨쉰다.**

### 🟣 라벤더 캔버스 — "첫인상" 화면
배경 = `banana-100` `#EDECFF`. 카드는 흰색으로 띄운다.

- 랜딩 / 마케팅 (`/`)
- 온보딩 (`/onboarding`)
- 공유 투표 페이지 (`/poll/[code]`, `/t/[slug]`) — 링크 받은 친구가 보는 첫 화면

### 🔵 인디고 캔버스 — 로그인 (`/login`)
`/login`만 예외다. 배경 = `brand-700` + `brand-600` radial glow, 액센트는 라벤더·흰색.

| 조합 | 대비 |
|---|---|
| `banana-100` 글씨 on `brand-600` (글로우 최악 지점) | 7.78:1 ✅ |
| `banana-100` 글씨 on `brand-700` (기본 캔버스) | 9.77:1 ✅ |
| 흰 워드마크 on `brand-700` | 11.4:1 ✅ |
| `banana-100` CTA + `brand-900` 글씨 | 13.6:1 ✅ |

> 히어로를 브랜드 색으로 꽉 채우는 화면이 하나는 필요하고, 그 자리가 로그인이다.

### ⚪ 화이트 서피스 — "작업" 화면
배경 = `#FBFBFE` / 스캐폴드 `neutral-50`. 지도 타일이 주인공이라 캔버스는 중립을 유지한다.

- 지도 보드 (`/moa/[id]`) · 탐색 (`/discover`) · 내 정보 (`/me`)

다만 **크롬(chrome)과 액션에는 인디고가 확실히 드러난다** — 캔버스만 흰색이지, 앱이 무채색이라는 뜻이 아니다:

| 요소 | 색 | 근거 |
|---|---|---|
| **지도 위 컨트롤** (스위처 pill · 플로팅 독) | 흰 글래스 `#FFFFFF/85` + 흰 헤어라인 + `shadow-float`(인디고 색조) | 하나의 "떠있는 레이어" 언어. 흰 시트 위 흰 캡슐이 사라지지 않게 그림자·헤어라인으로 분리 |
| **↳ 선택된 탭만** | `banana-200` 라벤더 필 + `brand-700` 아이콘/라벨 | 라벤더 필 위 라벨 8.57:1 ✅ |
| **↳ 미선택 탭** | `neutral-500` | 흰 배경 6.37:1 ✅ |
| **[함께 정하기]** | `banana-400` 라벤더 + 잉크 (Button `secondary`) | 담기=인디고 FAB, 함께하기=라벤더. 같은 hue라 이질감 없이 명도로만 구분 |
| **FAB (+)** | `brand-600` `#2F27CE` | 담기 = 인디고. `shadow-fab`(인디고 색조) + 인셋 하이라이트 |
| **하이라이트 카드 · "내가 투표함" 뱃지** | `banana-50` / `banana-100` | |

> **빈 상태(empty state)** 는 라벤더 칩 아이콘 + 흰 배경으로 구성한다 (큰 색면은 쓰지 않는다).
> **핀·장소 카드·칩은 흰 배경 위 인디고 계열을 유지한다** — 지도 위 고밀도 요소까지 라벤더로 칠하면 타일과 싸운다.

## 9. 구현 노트

> **구현 상태 (2026-07-18):** 이 문서는 Indigo Mono 팔레트의 **확정 스펙(SSOT)이며 웹 코드에
> 반영 완료**다. [`globals.css`](apps/web/app/globals.css) `@theme`·[`lib/palette.ts`](apps/web/lib/palette.ts)
> 모두 인디고로 교체됐고, 컴포넌트는 토큰(`bg-brand-*`·`bg-banana-*`·`text-neutral-*`)을 통해 자동 반영된다.
> 하드코딩 색은 없다 (카카오 로그인 `#FEE500`은 3rd-party 브랜드 규정상 유지).

**적용 범위: 웹 전용.** [`packages/ui-tokens/`](packages/ui-tokens/)는 **건드리지 않는다** — iOS와
공유되는 패키지이고 v2.1 웹 퍼스트 기간 동안 iOS는 동결 상태다 (CLAUDE.md §5). 토큰을 바꾸면
iOS 시각이 같이 깨진다. iOS 동결 해제 시 이 문서를 근거로 포팅한다.

웹에는 팔레트를 소비하는 경로가 **두 개**라 두 곳에 정의한다. 둘 다 이 문서에서 파생된다:

| 파일 | 소비자 | 이유 |
|---|---|---|
| [`apps/web/app/globals.css`](apps/web/app/globals.css) `@theme` | Tailwind 유틸 전체 | ui-tokens 프리셋을 웹에서만 덮어씀 |
| [`apps/web/lib/palette.ts`](apps/web/lib/palette.ts) | 지도 마커 SVG | 마커는 `data:` URL에 hex를 문자열로 넣어 CSS 변수가 닿지 못함 |

> 색을 바꿀 때는 **두 파일을 함께** 고친다. `packages/ui-tokens`의 구 팔레트는 iOS 전용이며,
> 웹 코드에서 `@moajoa/ui-tokens`의 `colors.brand`를 직접 import하면 안 된다 (구 색이 샌다).
> 잉크(`neutral-900`)도 웹에서는 `#050315`로 덮었다 — `lib/palette.ts`가 `colors.neutral`을 스프레드한 뒤 900만 재정의한다.

**알려진 이슈 (이번 범위 밖, 기록만):**
- **카테고리 칩 "문화"(243°)가 브랜드 hue와 겹친다** — 칩 구현 시 비-인디고로 재선정(§7.2). 현재 코드엔 칩 없음.
- **멤버 핀 `#9450E2`(268°)가 브랜드(243°)와 ~25°** — 명도·이니셜로 구분되나, 멤버 팔레트를 재튜닝할 기회가 있으면 벌려둔다.
- **토큰명 `banana-*`가 실제로는 라벤더** — 혼동 방지용 전면 리네임은 앱 전역 클래스 교체라 별도 작업으로 남겨둠.
