# Phase 19 — UI Design Contract

> Visual and interaction contract for the Date Voting (일정 미정 분기) phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> All tokens are pre-locked from `packages/ui-tokens/src/index.ts` and the existing `vote-island.tsx` / `onboarding.tsx` / `trip/create.tsx` patterns. **No new tokens.** This phase mirrors the established place-voting island and the iOS field-card create form.

## Spacing Scale

Declared values (Tailwind 4px base scale; tokens map `spacing.N` = N×4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, presence-dot gaps, chip inline padding |
| sm | 8px | Compact element spacing (vote-card list `space-y-2`, chat bubble gaps) |
| md | 16px | Default element/card padding (`p-4`), field-card inner padding |
| lg | 24px | Section padding (`px-6` iOS screen gutter), card-to-card breaks |
| xl | 32px | Layout gaps between major island sections (투표 → 집계 → 채팅) |
| 2xl | 48px | Major section breaks on the web poll page |
| 3xl | 64px | Page-level top/bottom spacing on landing |

Exceptions:
- iOS touch targets: brand cards and CTAs use `py-4` (≈16px vertical, 56px total height) — meets 44px min. Calendar grid cells (grid mode) minimum 40px (`size-10`) tappable, 44px on iOS via padding.
- Bottom-sheet top radius uses `radii.3xl` (28px) per token, not a spacing value.

## Typography

Mirrors `vote-island.tsx` (web) and `onboarding.tsx` / `create.tsx` (iOS). Weights restricted to **regular (400)** + **semibold (600)** for this phase. Display headline on iOS onboarding reuses existing `text-4xl font-semibold` and is out of scope to change.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 (`normal`) |
| Label | 16px (`text-base`) | 600 semibold | 1.5 (`normal`) |
| Heading | 18px (`text-lg`) | 600 semibold | 1.25 (`tight`) |
| Display | 24px (`text-2xl`) | 600 semibold | 1.25 (`tight`) |

Caption / metadata (counts, presence count, timestamps): 12px (`text-xs`), regular (400) only, color `neutral-400/500`. Inherited metadata convention from `vote-island.tsx`, scoped to counts/presence/timestamps — NOT a 5th heading size and NOT a general-purpose tier (checker FLAG resolved: dropped "medium" to keep the 2-weight restriction).

## Color

60/30/10 split inherited from `packages/ui-tokens`. Accent (MOAJOA Blue) is **rationed** — same discipline as `onboarding.tsx` (only the enabled card + base CTA carry brand; disabled is neutral).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `neutral-50 #F9FAFB` | Page background, card surfaces, sheet surfaces |
| Secondary (30%) | `neutral-100 #F3F4F6` / `neutral-200 #E5E7EB` | Card fills, borders (`border-neutral-200`), inactive toggle backgrounds, tinted scaffold |
| Accent (10%) | MOAJOA Blue — `brand-500 #2979FF`, `brand-600 #2563EB`, `brand-700 #1D4ED8`, `brand-50 #F5F7FF`, `brand-100 #E0EAFF` | See reserved list below |
| Destructive | `semantic.danger #EF4444` | 확정(투표 마감) confirm action emphasis + 댓글 삭제 confirm only |

Accent reserved for (explicit — NEVER "all interactive elements"):
- **Primary CTAs** — `bg-brand-500` (iOS) / `bg-brand-600` (web primary button): "날짜 투표 시작하기", "이 날짜로 확정", "이 여행에 함께하기".
- **Selected / voted toggle state** — `가능` toggle filled `bg-brand-500 border-brand-500 text-white` (mirrors the voted ❤️ in vote-island.tsx).
- **Active calendar cells** (grid mode) — a day marked 가능 = `bg-brand-500` cell.
- **Confirmed-date highlight** — the "확정: {range}" result block uses `brand-50` surface + `brand-600` text/border.
- **Activated onboarding "미정" card** — `brand-50` icon chip, brand chevron, enabled (mirrors the 정해짐 card).
- **Presence dots/avatars** — small `brand-500` dots (or brand-tinted initials) for "지금 보는 중" viewers.
- **Tinted hover/selected surfaces** — `hover:bg-brand-50`, selected card `border-brand-300 bg-brand-50/30` (existing vote-island idiom).

Neutral-only (NOT brand): unvoted/불가 toggles, secondary buttons, chat compose input border, inactive calendar days, metadata text.

## Copywriting Contract

All copy Korean, warm/casual (matches existing "가고싶어", "곧 제공돼요", "어디로 떠나세요?").

| Element | Copy |
|---------|------|
| Primary CTA (host, 미정 create) | **날짜 투표 시작하기** |
| Primary CTA (voter toggle — available) | **가능** (filled brand when selected) |
| Primary CTA (voter toggle — unavailable) | **불가** (neutral when selected) |
| Primary CTA (host confirm) | **이 날짜로 확정** |
| Primary CTA (conversion, post-confirm) | **이 여행에 함께하기** |
| Secondary CTA (share) | **초대 링크 복사** · **코드 공유** |
| Soft nudge (post-vote, optional, not paywall) | **결과 정해지면 알림 받기** |
| Empty state heading (0 votes, host card) | **아직 아무도 투표하지 않았어요** |
| Empty state body (0 votes) | **초대 링크를 친구에게 보내 가능한 날짜를 모아보세요.** + 초대 링크 복사 button |
| Empty state (voter, no comments) | **첫 메시지를 남겨보세요 💬** |
| Error state (vote save fail) | **투표를 저장하지 못했어요. 잠시 후 다시 시도해주세요.** (toast, `variant: 'error'` — mirrors existing vote-island toast) |
| Error state (nickname required) | **닉네임을 입력해야 투표할 수 있어요.** |
| Error state (comment send fail) | **메시지를 보내지 못했어요. 다시 시도해주세요.** |
| Destructive confirmation (확정) | **이 날짜로 확정하면 투표가 마감돼요.** / 본문: 다시 투표를 받으려면 새 투표를 만들어야 해요. / 버튼: **확정하기** (brand) · **취소** |
| Destructive confirmation (댓글 삭제) | **이 메시지를 삭제할까요?** / 버튼: **삭제** (`danger #EF4444`) · **취소** |
| Presence microcopy | **지금 {N}명 보는 중** (0이면 숨김; 1이면 "지금 보는 중") |
| Aggregate microcopy (range) | **{N}명 가능** · 최다 후보 옆 **최다** 배지 |
| Aggregate microcopy (grid) | **{date} · {N}명 가능** |
| Host summary card (plan tab) | **날짜 투표 진행 중** · 본문: 참여 {N}명 · 최다 후보 {date} |
| "일정 미정" trip badge | **일정 미정** (neutral badge on trip list) |

## Screen Contracts

Five surfaces. Each defines layout, components, interaction states, and the contract the executor must satisfy. Icon refs: Ionicons (iOS) / lucide-react (web).

### Screen 1 — iOS Onboarding "미정" Card Activation
**File:** `apps/ios/app/onboarding.tsx` (L65-83, currently disabled neutral stub)
**Goal:** Activate the disabled "곧 제공" stub into a brand-accented, enabled entry to 미정 trip create.

**Layout:** Stacked under the 정해짐 card. Mirror the 정해짐 card structure exactly (`Pressable`, `cardShadow`, `rounded-2xl px-4 py-4 flex-row items-center active:bg-neutral-50`).

**Token changes from current stub:**
- Container: `bg-neutral-100` → `bg-white` + `style={cardShadow}` + becomes `Pressable` with `onPress={() => router.push('/trip/create?dateless=1')}` (route param TBD by planner — D-04 dateless create path).
- Icon chip: `bg-neutral-200` → `bg-brand-50`, icon color `#9CA3AF` → `#2979FF` (`brand-500`). Keep `help-circle-outline` (or swap to `calendar-outline` per planner — semantic for "투표").
- Title: `text-neutral-400` → `text-neutral-900` (`text-base font-semibold`).
- Subtitle: `text-neutral-400` → `text-neutral-500` (`text-sm`), copy "날짜 투표로 함께 정해요".
- **REMOVE** the "곧 제공" neutral badge (`View` L77-79).
- **ADD** `<Ionicons name="chevron-forward" size={18} color="#D1D5DB" />` (mirrors 정해짐 card chevron).
- `accessibilityRole="button"`, label "날짜 투표로 일정 정하기".

**States:** default (brand-accented, enabled) · pressed (`active:bg-neutral-50`). No disabled state.

### Screen 2 — iOS Dateless Trip Create
**File:** reuse `apps/ios/app/trip/create.tsx` structure (dateless variant — D-04)
**Goal:** Same field-card form but **dates omitted** (title via city + city only).

**Layout:** Header `취소 / 새 여행 / spacer`. Single field card: **여행지** (required, `bg-brand-50` location chip, `FieldTag required`) — identical to existing. **OMIT the 여행 날짜 card and the needsEnd helper entirely.** Keep the representative caption ("만든 사람이 대표(결제자)예요").

**CTA:** label **날짜 투표 시작하기** (not "여행 만들기"). `canSave = !!cityCode && !!cityKo && !saving` (no date gate). Active `bg-brand-500 text-white`, disabled `bg-neutral-200 text-neutral-400` (existing idiom).

**Submit:** create dateless trip (`start_date`/`end_date` null — TripCreate dateless variant/RPC, D-04) → `router.replace('/trip/${trip.id}/plan')`. The poll is created/attached here or on first plan-tab visit (planner decides; UI lands on plan tab either way).

**States:** city unselected (CTA disabled neutral) · city selected (CTA brand) · saving (CTA disabled) · error → `Alert.alert('여행 만들기 실패', …)` (existing pattern).

### Screen 3 — iOS Plan Tab "날짜 투표" Management Card
**File:** `apps/ios/app/trip/[id]/plan.tsx` — surgical state-machine branch (D-05). Shown **only** when trip is dateless AND has an active (status≠closed) poll.
**Goal:** Inline host control surface: progress summary + share + confirm. After 확정 → card disappears, normal plan proceeds.

**Layout (card at top of plan tab):** `bg-white rounded-2xl` + `cardShadow`, `px-4 py-4`.
- Header row: `calendar` Ionicon in `bg-brand-50` chip + title **날짜 투표 진행 중** (`text-base font-semibold text-neutral-900`).
- Summary line: **참여 {N}명 · 최다 후보 {date}** (`text-sm text-neutral-500`). When 0 votes → empty-state copy ("아직 아무도 투표하지 않았어요").
- Share row: two buttons — **초대 링크 복사** + **코드 공유** (neutral-outline secondary buttons, `border-neutral-200`, brand text on press).
- Confirm button: **확정** → opens confirm flow (brand-500 primary, full-width or trailing).

**Confirm flow (INLINE from this card — D-09):** host taps 확정 →
- **range mode:** bottom-sheet (`@gorhom/bottom-sheet`) listing candidate options with per-option tally; host taps one → confirm dialog.
- **grid mode:** bottom-sheet showing per-day aggregate; host selects a **연속 블록** (N박) start/end → confirm dialog.
- Confirm dialog copy = Destructive confirmation (확정): "이 날짜로 확정하면 투표가 마감돼요." → 확정하기 (brand) writes `trip.start_date/end_date` + poll `status='closed'`.

**Realtime:** subscribe to poll channel (reuse `realtime.ts` `subscribe*` idiom, `*:{trip_id}` channel) — participation count / 최다 후보 update live.

**States:** active+0votes (empty state) · active+votes (summary) · confirm-sheet open · confirmed (card unmounts; plan tab renders normal Phase 18 states).

### Screen 4 — Web Anonymous Voting Island
**File:** new `'use client'` island mirroring `apps/web/app/t/[slug]/_components/vote-island.tsx` (D-02). Single shared URL; public SSR cache cookies-free → all poll/vote/chat state hydrates client-side (10-PATTERNS GOTCHA — never touch the cached anon render). Route `/poll/[code]` vs `/t/[slug]` is planner's discretion.
**Goal:** Invitee landing — one-page scroll: nickname → vote → live tally → chat, with presence.

**Layout (single column, mobile-first, `max-w` centered):** vertical sections separated by `xl` (32px) gaps.

**4a. Nickname gate (first visit, required — D-01):**
- Inline card: input + **시작하기** button. Stored with device token (localStorage UUID) — returning visitor skips.
- Empty/invalid → error copy "닉네임을 입력해야 투표할 수 있어요." Input `border-neutral-200`, focus `border-brand-300`.

**4b. Vote input (two modes — `date_polls.mode`, D-07):**
- **range:** list of candidate date-range cards (`space-y-2`, `border rounded-lg`), each with a **가능 / 불가** toggle. 가능 selected = `bg-brand-500 border-brand-500 text-white` (mirrors voted ❤️). 불가 selected = `bg-neutral-100 border-neutral-300 text-neutral-600`. Binary only.
- **grid:** month calendar; tap available days to fill. 가능 cell = `bg-brand-500 text-white`; default = `bg-white border-neutral-200`; drag-to-paint via gesture (optional). Binary 가능/불가.
- Optimistic update + rollback + error toast (exact `vote-island.tsx` onToggleVote pattern). Save fail → "투표를 저장하지 못했어요. 잠시 후 다시 시도해주세요."

**4c. Live aggregate (Doodle-style — D-09, LIVE):** below the input. Shows tally **and WHO** (nicknames). range = per-option "{N}명 가능" + nickname chips + **최다** badge on leader (`brand-600` relative bar, reuse vote-island popularity bar idiom). grid = per-day count + nickname reveal. Updates in realtime as others vote (Supabase Realtime on the poll channel).

**4d. Chat thread (CHAT-STYLE inline — D-11):** bottom of the same scroll. Flat thread. Nickname + message bubbles (own = brand-tinted right-aligned `brand-50`; others = neutral left-aligned `neutral-100`). Compose input + 보내기 at bottom. Delete: own (device-token match) / host any → 댓글 삭제 confirm (`danger`). Empty → "첫 메시지를 남겨보세요 💬". Realtime append.

**4e. Presence UI (Supabase Realtime Presence on same channel — D-11):** small header strip — **지금 {N}명 보는 중** with `brand-500` presence dots / nickname-initial avatars. Anonymous nicknames only. Hidden at 0.

**States:** pre-nickname (gate only) · voting (input active) · voted (toggles reflect my state, tally live) · save-error (toast + rollback) · empty-tally (0 votes → "아직 아무도 투표하지 않았어요" + share nudge) · empty-chat.

### Screen 5 — Web Confirmed Result + Conversion CTA
**File:** same island, rendered when poll `status='closed'` (D-10). Replaces the voting UI.
**Goal:** Customer-conversion moment — payoff then signup.

**Layout:**
- **Confirmed highlight block:** `bg-brand-50 border border-brand-100 rounded-xl p-4`. Heading **확정: {date range}** (`text-2xl font-semibold text-brand-700` / `text-neutral-900`) with a `Check` lucide icon in `brand-600`.

- **Conversion CTA:** primary button **이 여행에 함께하기** (`bg-brand-600 text-white rounded-lg`) → magic-link signup (D-03). Below it, secondary soft line: "함께 정한 일정을 앱에서 이어가요" (`text-xs text-neutral-500`).
- Chat thread remains visible (read-only or still-open per planner) below the result.

**States:** confirmed (result + CTA) · post-signup (handed to authed flow — out of this island's scope).

## Cross-Screen Interaction Notes

- **Accent discipline:** only the elements in the Accent reserved list carry brand. Unvoted/불가/secondary controls stay neutral — same rationing as `onboarding.tsx`.
- **Realtime channel:** reuse the `*:{trip_id}` (or `*:{poll_id}`) broadcast/presence idiom from `realtime.ts` (`subscribeExtractProgress`/`subscribePlanProgress`). One channel carries votes + comments + presence (D-11).
- **Anonymous identity:** nickname + localStorage device-token UUID is the shared identity across vote + comment (D-01/D-11). No login on the voting path; login only at the conversion CTA (D-03).
- **Public SSR cache safety:** the web island hydrates all mutable state client-side; the cached anon SSR render (cookies-free `unstable_cache`) is never touched — carry over `vote-island.tsx`'s hydration discipline verbatim.
