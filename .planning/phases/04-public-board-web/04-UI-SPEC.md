**Breakpoint (D-11 lock):** `md` (768px) 한 단계만 사용. `sm`, `lg`, `xl` 사용 안 함.

**OG image 1200×630 내부 spacing:**

- 좌측 텍스트 stack 간격: 제목 ↔ 도시 `24px`, 도시 ↔ 핀 수 `16px`, 핀 수 ↔ 워드마크 `64px`

- 좌우 panel 사이: 1px `neutral-200` divider (또는 생략 — researcher 재량)

### Weight assignment policy

- **400 regular** (기본): 본문 단락, 메타 정보 (owner, 도시, 핀 수, 작성자), helper 텍스트, empty state body, error 본문, 푸터 보조 텍스트, 영상 카드의 작성자 이름.
- **600 semibold** (강조): H1 보드 제목, 섹션 헤더 ("영상 출처 N개"), 핀 리스트 item 제목 (영상 제목), empty/not-found heading, 푸터 워드마크 "MOAJOA".

### Line height policy

- 본문 (`text-base`, `text-sm`) → `leading-normal` (1.5) — Korean 가독성, iOS Phase 3 정렬
- 헤더 (`text-2xl`, `text-lg`) → `leading-tight` (1.25)
- OG image — 제목 1.2, 도시·핀 수 1.4 (제한된 캔버스에서 vertical density 확보)

### Reassignment audit (기존 page.tsx 정렬)

| Use site (현재 코드) | Before | After (Phase 4 lock) |
|----------|--------|-------|
| `text-xs uppercase tracking-wide text-neutral-500` (owner 라벨) | 12 / 400 + uppercase | `text-sm text-neutral-500` (14 / 400) — uppercase·tracking 제거, Korean 본문에서 부자연. 의미는 같지만 시각 위계는 size step + color로 확보 |
| `text-3xl font-semibold` (보드 제목 H1) | 30 / 600 | `text-2xl font-semibold` (24 / 600) — display 토큰 lock 정렬 |

### Hierarchy verification (Dimension 1 / 4)

- **H1 보드 제목 (24/600) + owner 라벨 (14/400, neutral-500) + 도시·핀 수 메타 (14/400, neutral-600):** size step Δ10px + weight step 200 + color step으로 강한 위계.
- **섹션 헤더 "영상 출처 N개" (18/600) + 카드 제목 (16/600) + 작성자 (14/400):** 헤더와 카드 제목은 600/600 동일 weight이지만 size step 2px + 컨테이너 구분(섹션 ↔ 카드 내부)으로 위계 충돌 없음.
- **OG image 제목 (48/600) + 도시 (28/400) + 핀 수 (24/400) + 워드마크 (24/600):** size step 20→4→0, weight step만으로 워드마크와 핀 수 분리. 카톡 1200×630 thumbnail에서 보드 제목 우위 명확.

핀 마커 탭 → `window.open()` 새 탭으로 YouTube `?t=Xs`. 시각 변화는 없음(클라이언트 click handler).

| Element | Behavior |
|---------|----------|
| Map marker (Google Maps `Marker`) | `marker.addListener('click', () => window.open(youtubeUrl, '_blank', 'noopener'))` |
| URL 생성 (D-15) | `https://www.youtube.com/watch?v={video_id}&t={source_timestamp_sec}s` — `source_timestamp_sec`이 null이면 `?t=` 부분 생략 |
| video_id 추출 (D-15 lock) | helper `extractYouTubeVideoId(url: string): string \| null` — regex `(?:youtube\.com\/(?:watch\?v=\|embed\/)\|youtu\.be\/)([\w-]{11})` |
| 매핑 (D-16) | `places.find(p => p.id === ...).link_id` → `links.find(l => l.id === link_id).url` |

### 3. Map gestureHandling (D-12)

