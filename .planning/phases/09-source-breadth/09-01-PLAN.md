<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Edge Function | Authenticated caller supplies `link_id`; the link's `url` is user-controlled and will be fetched server-side in later plans |
| Edge Function → external host | Server-side `fetch()` of an arbitrary user-supplied URL (the core SSRF surface this plan guards) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure / Tampering (SSRF) | `assertFetchableUrl` in source.ts (consumed by blog.ts fetch in Plan 02) | mitigate | Reject non-`http(s)` schemes and private/loopback/link-local/metadata IP literals (127/8, 10/8, 172.16/12, 192.168/16, 169.254/16 incl. 169.254.169.254, ::1, fc00::/7) and `localhost` BEFORE any fetch. Pure function, unit-tested with `assertThrows`. Verifiable: `grep "169.254" source.ts` + `deno test source.test.ts`. |
| T-09-02 | Tampering (script injection from extracted content) | instagram.ts / SourceContent.bodyText | mitigate | Adapters return plain text only (`bodyText`), never rendered HTML; instagram.ts returns no body at all (throws). Claude receives plain text. Verifiable by construction — no `innerHTML`/`dangerouslySetInnerHTML`. |
| T-09-03 | Spoofing (LLM hallucination via altered prompt) | claude.ts sourceKind branch | accept (existing controls) | sourceKind changes only the body label + timestamp guidance; `source_quote` required + confidence filter + citation post-filter (index.ts:137) remain. Youtube regression-0 snapshot test guards against accidental prompt drift. |
</threat_model>
