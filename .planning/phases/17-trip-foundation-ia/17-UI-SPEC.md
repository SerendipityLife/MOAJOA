## Spacing Scale

Declared from `ui-tokens.spacing` (rem→px @ 16px base). The shared scale is already 4-point-aligned. Phase 17 uses this subset (every value a multiple of 4):

| Token | Value | Usage |
|-------|-------|-------|
| `1` | 4px | Icon-to-label gaps, badge inner padding (`px-1.5 py-0.5` 필수/선택 tag), chip gaps |
| `2` | 8px | Compact spacing (header row gaps, picker row inner gap), `hitSlop` baseline |
| `3` | 12px | Card→card vertical gap reference, list row vertical padding (`py-3` picker rows) |
| `4` | 16px | Default field-card inner padding (`px-4 py-4`), card→card gap (`mb-4`), content gap |
| `5` | 20px | Screen horizontal padding (`px-5` — matches `boards/new.tsx` + pickers) |
| `6` | 24px | Home/onboarding horizontal padding (`px-6` — matches `boards.tsx`), empty-state side padding |
| `8` | 32px | Section breaks, empty-state center block padding (`px-8`) |
| `10` | 40px | Home greeting top inset (`pt-10` — matches `boards.tsx`) |
| `12` | 48px | List bottom inset (`pb-12` / `paddingBottom: 48`), onboarding bottom band |

Exceptions:
- **Touch targets:** every tappable control — bottom-tab item, header trip-switcher, header profile button, picker rows, "여행 만들기" CTA, disabled 미정 path — must present **≥ 44×44px hit area** (Apple HIG). Visual size may be smaller; pad the `Pressable` or use `hitSlop` (existing pattern: `hitSlop={8}`).
- **Tab-bar item top padding:** `tabBarItemStyle: { paddingTop: 6 }` (6px) — fixed control geometry from existing `(tabs)/_layout.tsx`, not a scale token. Keep as-is.
- **Field-icon tile:** `w-11 h-11` (44px) brand-50 rounded tile on each trip-creation field card (existing `boards/new.tsx` pattern) — fixed 44px control dimension that doubles as the HIG touch floor.
- **Picker search field height:** `h-12` (48px) pill (existing `CityPicker`). Fixed control height.

## Typography

iOS sizes map to `ui-tokens.typography.sizes` (rem→px). Korean line-heights use `lineHeights.tight` (1.25) for headings/display, `lineHeights.normal` (1.5) for body — the Korean-readability rule already in tokens. **2 weights used: regular (400) and semibold (600).** The shared tokens also define `medium` (500) and `bold` (700), but Phase 17 restricts itself to 400/600 — headings/names render at 600, everything else at 400. Hierarchy comes from **size + weight**, never from uppercasing (Korean copy is never transformed).

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Display | 36px (`4xl`) | semibold (600) | 1.25 (`leading-tight`) | Onboarding 0-trip headline + home greeting (`boards.tsx` 인사말 pattern). 36px size alone carries display prominence. |
| Heading | 20px (`xl`) | semibold (600) | 1.25 | Trip-creation screen title ("새 여행"), picker sheet titles ("여행지 선택" / "여행 날짜"), empty-state heading |
| Body / field value | 16px (`base`) | semibold (600) for chosen values + button labels, regular (400) for prose / placeholders / descriptions | 1.5 | Field card selected value (e.g. "도쿄"), CTA button label, picker row city name, profile/switcher labels, empty-state body |
| Label / caption | 12–14px (`xs`/`sm`) | regular (400); semibold (600) only on the 필수/선택 tag and section eyebrow | 1.5 (1.25 for chips) | Field eyebrow label ("여행지" 14/600 tracking), 필수/선택 tag (12/700→use 600), helper text, "곧 제공" stub copy, tab labels (11px — see note), date helper |

Rules:
- **Never exceed these 4 roles.** No new font sizes beyond the `ui-tokens.sizes` set already used.
- **Only 2 weights: 400 and 600.** The existing `boards/new.tsx` uses `font-bold` (700) in places — Phase 17 normalizes new/edited surfaces to **600 (semibold)** for headings/values and 400 for the rest. (Surgical: only touch the lines this phase writes; do not refactor untouched 700s elsewhere.)
- **Tab-bar labels:** `fontSize: 11, fontWeight: '600'` is a fixed system-tab-bar style from existing `(tabs)/_layout.tsx`. 11px is below the 12px caption floor but is a platform tab-bar convention — **keep as-is** (exception, not a new type role).
- Place/city/trip names are `numberOfLines={1}` in switcher rows and field values.

## Color

60/30/10 split mapped onto `ui-tokens.colors`. **Brand blue (`brand.500` #2979FF) is the single accent and is rationed** — it is NOT the default for every interactive element. Most surface is white-on-tinted-scaffold neutrals.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `surface.raised` #FFFFFF + `neutral.50` #F9FAFB / `surface.background` #F5F6F8 | White cards & sheets on a near-white / tinted scaffold. Trip-creation scaffold = `neutral.50`; home = white; pickers = `neutral.50`. |
| Secondary (30%) | `neutral.{100..400}` (#F3F4F6 / #E5E7EB / #D1D5DB / #9CA3AF) | Field-card chrome via soft shadow (no border), inactive controls, picker search pill (`neutral.200/70`), placeholders (#9CA3AF), chevrons (#D1D5DB), disabled 미정 path fill, dividers |
| Accent (10%) | `brand.500` #2979FF (+ `brand.50` #F5F7FF tint, `brand.600` #2563EB pressed) | See reserved-for list below |
| Destructive | `semantic.danger` #EF4444 | "여행 삭제" confirmation + delete glyph only |

**Accent (`brand`) reserved for — exactly these elements (never "all interactive"):**
1. **Active bottom-tab** — `tabBarActiveTintColor: #2979FF` (inactive = `#9CA3AF`). Existing tab-bar contract.
2. **Primary CTA** — "여행 만들기" / "저장" button fill (`bg-brand-500`, white label) on the trip-creation screen; disabled = `bg-neutral-200` + `text-neutral-400`.
3. **Selected / required affordances in forms** — field-icon tiles (`bg-brand-50` + #2979FF glyph), the 필수 tag (`bg-brand-50` / `text-brand-500`), the selected city checkmark (#2979FF), date-range endpoints (`bg-brand-500` circle) and mid-range fill (`#E0EAFF` = `brand.100`), the "저장하면 영상 속 장소를 자동으로 찾아드려요" inline hint (`text-brand-500`).
4. **Empty-state nudge** — the plan-tab empty-state action line ("추출하면 플랜이 생겨요" CTA accent) and the onboarding "일정 정해짐" primary path (`bg-brand-500`).
5. **Header trip-switcher chevron / active trip name accent** (the "현재 여행 ▾" affordance may tint the chevron `brand.500`; the trip title itself stays `neutral.900`).

**Disabled / "준비 중" path color contract (D-11):** the "일정 미정" branch and the 예약/가계부 tabs landing as stubs use **neutral only** — `neutral.200` fill, `neutral.400` text, NO brand accent (a disabled control must never look primary). The "곧 제공" / "준비 중" badge uses `neutral.100` bg + `neutral.500` text. (Mirror of the optional `FieldTag` neutral styling already in `boards/new.tsx`.)

**Radii (from `ui-tokens.radii`):** field cards & sheets `radii.2xl` 24px (`rounded-2xl` — existing `boards/new.tsx`); picker rows `radii.2xl`; bottom-sheet top corners `radii.3xl` 28px (`rounded-t-3xl` — existing `DatePickerSheet`); search/switcher pills `radii.full`; field-icon tiles `radii.xl` 16px (`rounded-xl`); CTA button `radii.2xl` (matches the creation screen's existing save button).
**Shadows:** flat. Cards lift off the scaffold with the existing soft ambient `cardShadow` (`shadowColor #1E293B`, `opacity 0.06`, `radius 10`) — no borders on field cards. Tab bar uses the existing `shadows.nav`-style top shadow from `(tabs)/_layout.tsx`. No new shadow tokens.

**Icon-only control accessibility labels (Dimension 2 — every glyph-only control):**
- Header new-trip `+` → `accessibilityLabel="새 여행 만들기"`

- Header trip-switcher (if chevron-only) → `accessibilityLabel="여행 전환"`
- Picker close `✕` → `accessibilityLabel="닫기"` (existing pickers should carry this)

**Tone rules:** no ALL-CAPS, no double-exclamation, no English in user-facing copy (MOAJOA wordmark excepted). Disabled-path copy ("곧 제공", "곧 제공돼요") must read as a friendly promise, never an error.

## Screen Contracts

### Screen 1 — 진입 분기 · `apps/ios/app/index.tsx` (NAV-01)
- No session → `<Redirect href="/welcome" />` (existing, unchanged).
- Session present → branch on `decideEntryRoute(trips, lastTripId)` (RESEARCH Pattern 1): `0` → `/onboarding`; `1` → `/trip/{id}/plan`; `N` → `/trip/{lastValidTripId}/plan`.
- **No visible UI of its own** — renders `null` while resolving (matches existing `authed === null` pattern), then redirects. The plan tab is the landing surface (Claude's Discretion lock: land on `plan`, empty).

### Screen 2 — 온보딩 / "일정 정해졌나요?" 분기 · `apps/ios/app/onboarding.tsx` (SETUP-01, D-11)
- Full-bleed warm intro (mirror `welcome.tsx` photo+scrim tone OR a lighter brand-50 scaffold — planner's call; keep the display headline + body copy above).
- Two stacked path cards:
  - **정해짐** card — `bg-white` `rounded-2xl` `cardShadow`, brand-50 icon tile (calendar), label + 부제, chevron-forward. Tap → `/trip/create` (creation screen). **Enabled, brand-accented affordance.**
  - **미정** card — visually muted: `neutral.100` fill, `neutral.400` text, `곧 제공` neutral badge, NO chevron, `disabled` (Pressable no-op). (D-11 — IA completeness without a dead-end.)
- Primary CTA at base optional: **여행 만들기** routes the same as the 정해짐 card.

### Screen 3 — "일정 정해짐" 트립 생성 · `apps/ios/app/trip/create.tsx` (SETUP-01/02, D-08/09/10)
- **Reuse `boards/new.tsx` structure** (header 취소 / 새 여행 / spacer · scrollable field cards · bottom CTA).
- Field cards (each: 44px brand-50 icon tile + eyebrow label + 필수/선택 tag + value/placeholder + chevron):
  1. **여행지** (required) → opens existing `CityPicker`. Value = `cityKo` at 16/600.
  2. **여행 날짜** (required this phase, D-09) → opens existing `DatePickerSheet`. Range required; end≥start; day-trip = end=start. Value = `formatDateRangeKo`.
  3. *(Optional, planner's call to keep)* 메모 / 장소 가져오기 / 공개 여부 cards exist in `boards/new.tsx` — **Phase 17 may carry them forward unchanged** (out of new scope but not harmful) OR trim to the two required fields. The date field's tag flips from 선택 → 필수.
- **Representative (D-10, SETUP-02):** no UI field — `representative_id` is set to the creator automatically (trigger or client). Surface the `만든 사람이 대표(결제자)예요` caption so the user understands.
- Bottom CTA **여행 만들기** — `bg-brand-500` enabled (city+date valid) / `bg-neutral-200` disabled. On success → `router.replace('/trip/{id}/plan')`.
- Error → `Alert('여행 만들기 실패', …)`, button re-enables.

### Screen 4 — Trip 4-tab shell · `apps/ios/app/trip/[id]/(tabs)/_layout.tsx` (NAV-02)
- `<Tabs>` with `headerShown: false` (header owned by parent Stack), 4 screens **always-visible tab bar**, **no FAB** (FAB removed — new-trip is a header action, D / PRODUCT §11):
  | name | title | glyph (FontAwesome6, matches existing tab tone) |
  |------|-------|------|
  | `map` | 지도 | `map-location-dot` (solid) |
  | `plan` | 플랜 | `route` (solid) |
  | `book` | 예약 | `bed` or `ticket` (solid) |
  | `ledger` | 가계부 | `receipt` (solid) |
- Tab-bar style = existing white bar + hairline + soft top shadow + `tabBarActiveTintColor #2979FF` / inactive `#9CA3AF`, label 11/600. **Default landing tab = `plan`** (Claude's Discretion lock).
- Anti-pattern guard (RESEARCH Pitfall 1): header = parent Stack, tab bar = `(tabs)/_layout` — never both in one layout, never double-nested `(tabs)`.

### Screen 5 — Trip-scoped header · `apps/ios/app/trip/[id]/_layout.tsx` (NAV-03)
- `<Stack>` with custom `header: () => <TripHeader tripId={id} />` (`useGlobalSearchParams` for `id`).
- Layout: **left** = "현재 여행 ▾" trip-switcher (trip title `neutral.900` 16/600 + `brand.500` chevron) → opens 여행 전환 sheet (list of trips + `+ 새 여행` + each row's 삭제 via existing `boards.tsx` delete Alert). **right** = profile glyph → `/me`. Optional `+ 새 여행` inline if not in the switcher sheet.
- Header background `white`, hairline bottom border (`neutral.100`).

### Screen 6 — Tab content states (this phase = mostly empty/stub)
- **map** — reuse the existing board map/place-list content (ported from `boards/[id].tsx`). Empty → map-tab empty state copy above. (Full map UI is existing; Phase 17 only re-homes it under `trip/[id]/(tabs)/map`.)
- **plan** — empty state (heading + body copy above) driving extract/share. Phase 18 fills it. Center block: brand-50 circle + Ionicons `sparkles`/`map-outline`, heading 20/600, body 16/400 `neutral.500`, action line `brand.500`. (Mirror `boards.tsx` `ListEmptyComponent`.)
- **book / ledger** — neutral "곧 제공돼요" stub (heading + body copy above). NO brand accent. Center block with a muted `neutral.300` glyph + `neutral.500` text.
