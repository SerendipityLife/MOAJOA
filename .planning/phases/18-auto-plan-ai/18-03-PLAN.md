<objective>
Build the `generate-plan` Edge Function as a near-clone of `extract-youtube`: service-role admin client, `auth.getUser` cost-abuse gate PLUS a `can_edit_trip` membership check, load the trip's placeable places (filtering out (0,0)-coord and hidden), ask `claude-sonnet-4-6` to geo-cluster them into days + order + an unplaced pool, post-process adjacent-only legs through the Google **Routes API** `computeRoutes` (TRANSIT/WALK/DRIVE, minimal FieldMask to stay in the Essentials tier), write `plans` + `plan_items`, broadcast progress on the trip-scoped channel, and log Anthropic + Routes cost.

Purpose: The AI system core (PLAN-01/03/04). It is ~80% recombination of shipped extract-youtube primitives + two new external surfaces (Routes API, a second Claude prompt). The risk concentrates in Claude plan-quality and the Routes call shape — both mitigated by the same defensive validation extraction uses and a minimal FieldMask.

Output: `supabase/functions/generate-plan/{index.ts, deno.json, pipeline/{claude.ts, claude.test.ts, routes.ts, routes.test.ts}}`. TDD on the pipeline (deno tests are writable before implementation); the handler orchestration is integration-verified.

Task 2 includes a live two-point Tokyo call to confirm enablement before declaring done; if the key is unenabled, the EF degrades gracefully (null legs → "이동시간 —") but enablement is the real gate. Billing alerts ($5/$20/$50, EXTRACT-06) already cover spend.
