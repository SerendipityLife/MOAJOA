# Phase 29 — UI Design Contract (Chat Unification · 채팅 단일화)

> Visual and interaction contract. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> 잠긴 결정 D-01~D-03a(29-CONTEXT.md)는 **불변 입력** — 이 스펙은 재량 영역(presence 통일·게이트 시점·embedded 정리)만 채우고 **[AUTO]**로 표시한다.

### 1. `/t/[slug]` dates 공유 (D-01 — both 경로 수렴)

**비join (현행 유지 — [AUTO A-2]):**
```
[MOAJOA 크롬 + 초대 카드 — 기존]
날짜 정하기                      ← 기존 heading (both 미러)
[PollVoteIsland — 투표 UI]      ← 자체 presence 스트립 없음 (A-4)
[NicknameGateSheet — 첫 투표 시]
```
- 비join dates에 장소 섹션·채팅 teaser를 **추가하지 않는다** — join 전 채팅 이력은 RLS상 어차피 불가시(멤버 SELECT), 표면 최소(§3.2).

**join 후 (both과 동일 — D-01 핵심):**
```
[MoaIsland 풀 마운트 — hideHostControls]
  [모으기] 시트: pollSlot(날짜 정하기 섹션) 최상단
              → 장소 0이면 기존 빈 상태 "아직 담은 장소가 없어요"
              → 장소 있으면 기존 PlaceList 그대로 (특수 분기 0)
  [채팅] 탭: MoaChat 풀 기능 — 영속 이력·presence·멘션 칩·답장
  [마이]: 기존 계정 액션
```
- 게이트 시점 **[AUTO A-3]**: 기존 lazy 패턴 유지 — dates의 첫 참여 액션(투표)에서 게이트 → `ensureGuestMember` → join → MoaIsland 마운트. **채팅 진입 자체가 별도 게이트를 추가로 요구하지 않는다**(join이 선행 조건이므로 자동 충족).
- 멘션 칩: dates에 장소가 있으면 기존 칩 nav(모으기 탭 전환+스크롤+ring-2 하이라이트) 무료 재사용, 장소 0이면 칩 미해석→미렌더(Pitfall 9 기존 동작).

### 2. `/poll/[code]` (D-03 — 한마디 자리에 통일 채팅)

```
[MOAJOA 브랜드 라인 — 기존 무수정]
[언제 같이 떠날까요? 헤더 — 기존 무수정]
[PollVoteIsland — 투표 UI, PollChat·presence 스트립 제거됨]
─────────────────────────────  ← mt-8 border-t pt-6 (기존 한마디 섹션 위치·구분선 계승)
채팅                            ← 섹션 헤딩 14px/600 neutral-700 (기존 "한마디" 헤딩 스타일 계승, 라벨만 교체)
[MoaChat — h-[400px] 컨테이너]  ← A-6. presence 스트립·말풍선·compose row 전부 기존 시각
[MOAJOA 푸터 — 기존 무수정]
```

- **비멤버 상태 [AUTO A-7]:** 채팅 섹션은 항상 렌더. 이력은 멤버 전용(RLS SELECT)이므로 비멤버에겐 빈 목록 + 전용 빈 상태 카피 `참여하면 지금까지의 대화를 볼 수 있어요` (멤버 0건 빈 상태 "첫 메시지를 남겨보세요 💬"와 구분). compose row는 활성 — **첫 전송 시도 시 NicknameGateSheet** → `signInAnonymously`+`joinMoa`(guest-surface `ensureGuestMember` 패턴, D-03a-1) → 전송 진행 + 이력 hydrate + presence track. 투표 첫 참여도 동일 게이트 공유(어느 쪽이 먼저든 1회).
- **poll 마감(closed)과 채팅은 독립 [AUTO A-8]:** 채팅은 trip 소속(`trip_messages`)이지 poll 소속이 아니다 — poll status가 closed여도 compose는 열린 채 유지. 구 카피 "투표가 마감되어 메시지를 남길 수 없어요"는 은퇴(HC-7).
- **멘션 칩 [AUTO A-9]:** /poll엔 장소 리스트 UI가 없으므로 `placesById={}` 전달 → 칩 전부 미해석·미렌더(Pitfall 9 기존 경로), `onChipTap` no-op, `replyToPlaceId=null` 고정(답장 작성 UI 미노출). 결정론적 — /poll에서 장소 해석을 시도하지 않는다.
- **presence [AUTO A-4]:** MoaChat 내장 presence 스트립("지금 N명 보는 중")이 /poll의 **유일한** 보는-중 표면. poll-vote-island 자체 스트립은 제거. presence 채널은 `moa:{tripId}`로 단일화(비멤버는 카운트 밖일 수 있음 — 0이면 스트립 숨김, 기존 동작).
