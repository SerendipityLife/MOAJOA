# Phase 18 — UI Design Contract

> Visual and interaction contract for the 플랜 (plan) tab inside a trip. Generated by gsd-ui-researcher (--auto mode), verified by gsd-ui-checker.
>
> **Screen:** `apps/ios/app/trip/[id]/(tabs)/plan.tsx` — the default landing tab inside a trip. Phase 17 shipped the empty-state stub; Phase 18 fills the plan content (button → skeleton → draft itinerary → editing).
>
> **Auto-mode note:** No user questions were asked. Every value is pre-populated from CONTEXT.md (D-01..D-14), RESEARCH.md, REQUIREMENTS.md (PLAN-01..05), the shipped Phase 17 chrome, and the existing `@moajoa/ui-tokens` token contract. Defaults are explicitly flagged.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn N/A — this is Expo / React Native, not web) |
| Preset | not applicable |
| Component library | NativeWind classes + `@gorhom/bottom-sheet@^5.2.14` (installed) for sheets/modals; `react-native-reanimated-dnd@2.0.0` OR hand-rolled gesture-handler+Reanimated 4 for drag (RESEARCH Open Q1 — sketch first) |
| Token source | `@moajoa/ui-tokens` (shared Tailwind preset — `packages/ui-tokens/src/index.ts`). Single source of truth for brand/neutral/semantic colors, spacing, radii, typography. **Do not add per-phase tokens.** |
| Icon library | `@expo/vector-icons` — `Ionicons` for actions/glyphs (matches plan.tsx stub + header.tsx), `FontAwesome6` only for the tab bar (locked Phase 17). Use Ionicons within plan content. |
| Font | Pretendard (loaded via expo-font), OS Korean fallbacks. All copy is Korean. |

**Reuse mandates (do NOT re-invent):**
- Sheet idiom → mirror `components/boards/pin-sheet.tsx` (`@gorhom/bottom-sheet`; visible content in an inner `<View className="...">` because NativeWind on `BottomSheetView` can silently no-op — RESEARCH Pitfall 3 / pin-sheet.tsx L4-8).
- Destructive confirm → mirror `header.tsx` `Alert.alert(...)` / boards delete pattern OR a centered confirm modal in the `header.tsx` switcher idiom (`bg-black/30` scrim + `bg-white rounded-t-3xl` sheet).
- Progress UI → mirror `components/boards/step-indicator.tsx` (white `rounded-3xl` card, `ActivityIndicator color="#2979FF"`, step rows with brand/neutral/outline dots).
- Place row → mirror `components/boards/place-list.tsx` (white `rounded-2xl` card, `ROW_SHADOW`, vibe-tinted `w-10 h-10 rounded-xl` icon leading, `text-sm font-semibold` title + `text-xs neutral-500` subtitle, brand-50 stadium chip trailing).
- Empty state → keep the existing plan.tsx stub idiom (brand-50 circle, Ionicons, `text-xl font-semibold` heading + `text-base neutral-500` body).

## Spacing Scale

The project token scale is rem-based and is largely an 8-point scale with a few standard 4px half-steps already in shipped components (`mb-2.5`=10px, `py-0.5`=2px). Declared values for this phase (px; multiples of 4 except the documented half-steps below):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, chip inner padding (`px-1`/`mr-1`) |
| sm | 8px | Compact element spacing, dot-to-label gaps (`gap-2`) |
| md | 16px | Default vertical rhythm between rows, card inner padding (`p-4`), step-list `gap:16` |
| lg | 24px | Screen horizontal padding (`px-6` — matches place-list `paddingHorizontal:24`), day-section top margin |
| xl | 32px | Major section breaks (placed days ↔ 미배치 pool divider) |
| 2xl | 48px | List bottom padding (`paddingBottom:48` — matches place-list) |

Exceptions (carried from shipped components — keep for visual continuity, do NOT "fix"):
- `mb-2.5` (10px) between place cards (place-list.tsx row gap).
- `py-0.5` / `mt-0.5` (2px) for tight sub-label offset inside a card.
- `gap:10` / `gap:12` inside the progress card (step-indicator.tsx — tested contract).
- Touch targets: every glyph-only control (drag handle, mode toggle, 필수 star, remove, pool add) carries a **≥44×44** hit area via `hitSlop` or explicit `w-11 h-11` (matches header.tsx). This is the iOS minimum, not a spacing-scale violation.

## Typography

Three roles + one display, drawn directly from the token `sizes`/`weights` and the shipped plan.tsx + place-list + step-indicator contracts. **Two weights in normal use: 400 (regular) and 600 (semibold).** (700/bold appears only inside the reused step-indicator card title, which is a tested contract carried verbatim — not a new declaration for this phase.)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display (empty/heading) | 20px (`text-xl`) | 600 semibold | 1.25 tight | Empty-state heading, sheet titles ("여행 전환" idiom = `text-xl font-semibold`), 미배치 풀 section title |
| Body | 16px (`text-base`) | 400 regular | 1.5 normal | Empty-state body copy, "초안" descriptive line, modal body copy. `leading-relaxed`(1.65) only for the multi-line empty body (matches stub). |
| Label / item title | 14px (`text-sm`) | 600 semibold | 1.25 tight | Place-item name in a day row (matches place-list title), primary CTA button label |
| Caption / meta | 12px (`text-xs`) | 400 regular (500 medium for de-emphasized future steps) | 1.25 tight | Place subtitle (category·address), 이동시간 leg label, "Day N" date sublabel, source chip text |

Line-height rationale: token `lineHeights` are tightened for Korean (tight 1.25 / normal 1.5 / relaxed 1.65). Body copy uses 1.5; dense list rows and labels use 1.25 (matches place-list/step-indicator).

## Color

60/30/10 split mapped onto the shipped MOAJOA token palette. The app scaffold is white/near-white; cards are white; brand blue is rationed.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` white (`bg-white`) + `#F5F6F8` surface.background for the scaffold behind cards | Screen background, day-section ground, card surfaces |
| Secondary (30%) | neutral scale — `#111827` neutral.900 (titles), `#6B7280` neutral.500 (subtitles/meta), `#F3F4F6` neutral.100 (hairlines/dividers), `#E5E7EB` neutral.200 (drag-handle, inactive toggle track) | Text hierarchy, dividers, inactive controls, skeleton blocks |
| Accent (10%) | `#2979FF` brand.500 (`brand-500`); `#F5F7FF` brand.50 tint for accent surfaces; `#2563EB` brand.600 for chip text on brand-50 | See reserved-for list below |
| Destructive | `#EF4444` semantic.danger / `danger` | "플랜 다시 만들기" overwrite confirm CTA, 일정에서 제거 / 미배치로 보내기 destructive glyph (trash/down-arrow) — matches header.tsx `#EF4444` |

**Accent (brand.500) reserved for — explicit list, never "all interactive elements":**
1. The primary CTA **"플랜 만들기"** button (filled brand-500 background, white label) — the one prominent accent on the pre-generation screen.
2. The "플랜 다시 만들기" / regenerate secondary action **label + leading icon** (brand-500 text on white, ghost style — secondary to the destructive confirm inside the modal).
3. The **current/active progress step** dot + label during generation (`#2979FF`, mirrors step-indicator.tsx) and the `ActivityIndicator`/skeleton shimmer accent.
4. The **active travel-mode segment** (전철/도보/차 toggle) — selected segment uses brand-500 fill or brand-50 tint + brand-600 label; unselected stays neutral.
5. The **필수 장소 (anchor) marker** when set — filled brand-500 star/pin glyph; unset is a neutral-400 outline star.
6. The **"이동시간" leg pill** leading icon and value chip when present (brand-50 stadium chip + brand-600 text, mirroring the place-list source `play` chip). Null/failed legs render neutral.
7. The **"친구와 같이 정하기" collaborative toggle** active state + the 미배치 풀 item **"일정에 추가"** affordance glyph.

Everything else (place rows, day headers, pool rows, body text, dividers) stays white/neutral. Category vibe tints on the leading place-icon (`vibe.tint`/`vibe.color` from `lib/category`) are carried from place-list.tsx and are NOT counted against the 10% accent budget (they are a separate, pre-existing categorical palette).

Status semantics (carried, not new): `success` `#16A34A`/`#10B981` is reserved for confirmed/done states only (e.g. progress "done" could read neutral; reserve green for a future confirm). Do not introduce green as a general accent.

## Copywriting Contract

All copy Korean, informal-polite (-요), matching the shipped voice ("아직 플랜이 없어요", "링크를 공유해 시작하기", "여행 전환").

| Element | Copy |
|---------|------|
| Primary CTA (places exist, no plan) | **플랜 만들기** |
| Primary CTA (generating, disabled) | **플랜을 짜고 있어요…** (button disabled + spinner; double-tap guard per RESEARCH Pitfall 5) |
| Regenerate action (draft exists) | **플랜 다시 만들기** |
| Draft label (always-on banner/chip on a draft plan) | **초안** (small stadium chip near the plan title; D-02 — "초안" must be visually explicit) |
| Empty state — no places yet (heading) | **아직 플랜이 없어요** (keep stub) |
| Empty state — no places yet (body) | **유튜브 링크를 추출하면 영상 속 장소로 플랜을 만들 수 있어요.** (keep stub idiom; one accent action line: `링크를 공유해 시작하기`) |
| Empty state — places exist, no plan (heading) | **장소가 모였어요** |
| Empty state — places exist, no plan (body) | **모은 장소로 동선과 날짜별 일정을 짜드릴게요. 아래 버튼을 눌러보세요.** (with prominent 플랜 만들기 button below) |
| 미배치 풀 section title | **미배치** |
| 미배치 풀 helper (when pool non-empty) | **일정에 넣지 않은 장소예요. 추가하거나 그대로 둘 수 있어요.** |
| 미배치 풀 helper (coordinateless places) | **위치 정보가 없어 자동 배치에서 빠졌어요.** (D-09 — (0,0) places) |
| Leg travel-time label (present) | **{n}분** (e.g. `14분`; brand-50 chip with a transit/walk/drive glyph matching active mode) |
| Leg travel-time label (null/failed) | **이동시간 —** (graceful; RESEARCH A1/A2 — null legs render neutral) |
| Travel-mode toggle labels | **전철 / 도보 / 차** (default = 전철 / transit, D-08) |
| 필수 장소 affordance (unset → set) | **필수 장소로 지정** / set state shows a filled star + **필수** tag |
| Error state — generation failed | **플랜을 만들지 못했어요. 잠시 후 다시 시도해 주세요.** (1-tap retry button: **다시 시도**) — mirrors TRUST-03 retry idiom |
| Error state — no placeable places | **자동 배치할 장소가 없어요. 위치가 있는 장소를 먼저 모아주세요.** (all places (0,0)/hidden → nothing to plan) |
| Destructive confirmation — regenerate overwrite | **플랜 다시 만들기**: title `플랜을 다시 만들까요?` · body `초안을 다시 만들면 지금까지 편집한 내용이 사라져요. 필수로 표시한 장소는 그대로 반영돼요.` · confirm `다시 만들기` (destructive red) · cancel `취소` (D-11) |
| Destructive confirmation — remove place from day | inline destructive glyph (trash/down-arrow `#EF4444`); no modal needed (reversible — place returns to 미배치 풀, D-13). Optional toast: **미배치로 옮겼어요.** |
| Collaborative toggle | **친구와 같이 정하기** (toggle on → share affordance appears; D-14 — toggle + flag + share only, no in-plan vote UI in Phase 18) |

## Interaction & State Contract

Phase-specific because this screen is interaction-heavy (drag, realtime, modes). Each state below is a render contract the executor must satisfy.

### State A — Empty, no places (`places.length === 0`)
- Keep the shipped stub verbatim: brand-50 circle + `sparkles` Ionicon, `아직 플랜이 없어요` heading, body, `링크를 공유해 시작하기` accent line. No 플랜 만들기 button (nothing to plan).

### State B — Pre-generation (places exist, no draft plan)
- Heading `장소가 모였어요` + body, then a **prominent filled brand-500 "플랜 만들기" button** (`rounded-lg`/12px radius per token `radii.lg`, white 14px/600 label, ≥44px height, full-width within `px-6`). This is the screen's single accent (D-01: user-triggered, never auto).

### State C — Generating (skeleton + realtime progress)
- On tap: disable the button (`플랜을 짜고 있어요…` + inline spinner) to prevent double-spend (RESEARCH Pitfall 5).
- Subscribe to trip-scoped channel **`plan:{trip_id}`** (RESEARCH Pattern 1 — `planChannelName(tripId)`).
- Render the progress card mirroring `step-indicator.tsx` (white `rounded-3xl` card, `ActivityIndicator color="#2979FF"`, title `플랜을 짜고 있어요`). Steps reflect EF broadcast: **장소 불러오기 → 동선 짜기 → 이동시간 계산 → 완료** (loading/clustering/routing/done; pct 10/50/80/100). Active step = brand-500 dot+label; done = neutral.500; future = outline dot neutral.300.
- Optionally show skeleton day-section blocks (neutral.100/200 shimmer) beneath the card.

### State D — Draft plan rendered
- A **"초안" chip** sits next to the plan title (always visible while `status==='draft'`).
- **Day sections**: `Day 1 / Day 2 / …` headers (count from trip start/end date range, Phase 17). Each header = `text-sm font-semibold neutral-900` "Day N" + `text-xs neutral-500` date sublabel. Days are ordered top-to-bottom.
- Within a day: ordered **place-item rows** mirroring `place-list.tsx` cards (white `rounded-2xl`, ROW_SHADOW, vibe-tint leading icon, `text-sm/600` name, `text-xs neutral-500` category·address subtitle). Each row carries a **drag handle** glyph (Ionicons `reorder-three`, neutral-400, ≥44px hit area) and a **필수 star** + **제거(미배치로)** glyph.
- **Between adjacent rows in a day**: a centered **leg pill** — brand-50 stadium chip with the active-mode glyph + `{n}분`, or `이동시간 —` neutral when null (D-04/D-07). Only between consecutive same-day items (N-1 per day).
- **Travel-mode toggle** (전철/도보/차): a 3-segment control at plan scope (top of the plan, near the 초안 chip). Default 전철. Active segment = brand. Toggling recomputes all legs (D-08) — show a brief inline spinner on the affected leg pills.
- **미배치 풀 section** at the bottom, separated by an `xl`(32px) gap and a neutral-100 hairline. Title `미배치` + helper copy. Pool rows reuse the place-card idiom but with an **"일정에 추가"** affordance (brand-50 chip / `add-circle-outline` brand-500) instead of a drag handle. Coordinateless places show the `위치 정보가 없어…` helper variant (D-09).
- **Regenerate** action `플랜 다시 만들기` (brand ghost, secondary) accessible from the plan header area; opens the overwrite confirm modal.
- **Collaborative toggle** `친구와 같이 정하기` accessible (header/overflow); toggling sets `plans.collaborative` + reveals share (D-14). No vote UI.

### State E — Editing (drag + pool move + anchor)
- **Reorder within a day** = drag (long-press → lift → drop), `react-native-reanimated-dnd` sortable OR hand-rolled (RESEARCH Open Q1 — sketch first, both Reanimated-4-safe; do NOT use `react-native-draggable-flatlist`, Pitfall 3). On drop, PATCH `sort_order` + recompute only the ≤2 affected legs (D-07 / Open Q2).
- **Placed ↔ 미배치 move** = explicit affordance, NOT fragile cross-zone drag (D-13 / RESEARCH A4): a placed row's `제거` glyph → 미배치; a pool row's `일정에 추가` chip → appended to a day (default: last day, end). This is the recommended v1 shape; true cross-zone drag is deferred.
- **필수 장소** = tap the star on a row → marks anchor (filled brand star + `필수` tag). Marking does NOT immediately re-cluster; re-cluster happens on `플랜 다시 만들기` with anchors passed (D-10). (If a dedicated "필수로 다시 짜기" affordance is added, it routes through the same overwrite confirm.)

### State F — Error
- Generation failure → replace skeleton with the error copy + **다시 시도** retry button (mirrors TRUST-03). Channel `error` event or invoke rejection drives this.
