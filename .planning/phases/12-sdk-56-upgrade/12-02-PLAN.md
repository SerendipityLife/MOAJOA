# Plan 12-02: prebuild + deployment target 16.4 + expo run:ios 복귀 검증

**Phase:** 12 (SDK 55 → 56)
**Wave:** 2 (depends 12-01)
**autonomous:** 빌드/테스트 true · 시각 회귀는 스크린샷
**Requirements:** UPGRADE-04 (표준 경로 복귀)

## 목표

SDK 56 위에서 ios/ 재생성 + deployment target 16.4 + 빌드 + **표준 `expo run:ios`가 Xcode 26에서 동작**함을 검증(이 마일스톤 실질 목표).

## Tasks

- **T1** — `expo prebuild --clean -p ios` (SDK 56 네이티브 재생성). react-native-maps 플러그인 유지 확인.
- **T2** — iOS deployment target **16.4** 확인 (pbxproj IPHONEOS_DEPLOYMENT_TARGET + Podfile platform). 15.1로 남으면 app.config.ts에 `expo-build-properties` plugin(`ios.deploymentTarget: '16.4'`) 추가 후 재prebuild.
- **T3** — 네이티브 보존 grep: GMSApiKey / App Group / Hermes(jsEngine) / `react-native-maps/Google`.
- **T4** — **표준 경로 검증:** `pnpm --filter @moajoa/ios exec expo run:ios` 시도. Xcode 26 사이닝 에러 없이 빌드·설치·실행되면 UPGRADE-04 핵심 충족. (실패 시 에러 기록 + pnpm sim으로 빌드 자체는 확인)
- **T5** — 앱 부팅 후 웰컴/보드 화면 스크린샷(회귀 0, New Arch 등록 OK).
- **T6** — `tsc --noEmit` + `jest` 38/38.

## Success Criteria (verify)

1. SDK 56 prebuild + pod install + 빌드 green
2. deployment target 16.4
3. **`expo run:ios`가 Xcode 26 시뮬레이터에서 사이닝 에러 없이 동작** (또는 명확한 잔여 사유 기록)
4. 웰컴/보드 화면 렌더 회귀 0 · tsc + jest 통과
5. GMSApiKey/App Group/Hermes/maps subspec 보존

## 잠금 결정

- prebuild --clean (RN 메이저 점프)
- `expo run:ios` 성공 = pnpm sim 우회 제거(Phase 13)의 전제 — 이 plan에서 표준 경로 동작을 먼저 입증
- 실제 MapView 타일 렌더는 로그인 필요 → Phase 13 실기기 UAT
