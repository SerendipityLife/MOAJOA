# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1 (Build Unblock & Hygiene). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 scope is foundational brand assets + typography pipeline + web dev-tool gate.** No new product UI (boards/login/share flows are Phase 3+). Every value below is a *permanent* contract that downstream phases (4 OG, 5 Trust UI) will inherit.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual tokens via `@moajoa/ui-tokens`) |
| Preset | not applicable — shadcn intentionally skipped; project uses Tailwind v4 (web) + NativeWind 4.2 (iOS), shared via single token preset (`packages/ui-tokens/src/tailwind.ts`) |
| Component library | none in Phase 1 (Phase 3+ may introduce primitives) |
| Icon library | none in Phase 1 (no in-app icons yet beyond brand mark) |
| Font | **Pretendard** (4 weights: 400/500/600/700) primary · IBM Plex Sans KR · Noto Sans JP · system-ui (fallback chain locked in `ui-tokens` typography.fonts.sans) |

**Why no shadcn:** Phase 1 ships zero application UI — only static brand assets, fonts, and a redirect. Introducing shadcn before Phase 3 (iOS save flow) or Phase 4 (web public board) would lock component choices before the consumer surface exists. Re-evaluate at Phase 3/4 kickoff.

## Spacing Scale

Declared values (must be multiples of 4 — taken directly from existing `@moajoa/ui-tokens` `spacing`):

| Token | Value | Usage |
|-------|-------|-------|
| 1 | 4px | Inline icon gap |
| 2 | 8px | Compact element spacing |
| 3 | 12px | Tight stacks |
| 4 | 16px | Default element spacing |
| 5 | 20px | Form field gaps |
| 6 | 24px | Section padding |
| 8 | 32px | Layout gaps |
| 10 | 40px | iOS touch target (min) |
| 12 | 48px | Major section breaks |
| 16 | 64px | Page-level spacing |
| 20 | 80px | Splash wordmark margin from edges (vertical) |
| 24 | 96px | Reserved (large hero spacing in later phases) |

**Phase 1 specific spacing exceptions:**

- **iOS App Icon foreground safe-area:** content kept inside the **inner 80%** square (10% padding each side). Apple HIG recommendation; ensures no clipping when iOS applies its mask.
- **Splash wordmark vertical position:** wordmark vertically centered on screen, baseline aligned to **device vertical midpoint**. Single image, no flex layout (expo-splash-screen rasterizes).
- **Web favicon:** 32×32 / 16×16 — sizes set by spec, not the spacing scale.

## Typography

Locked from `@moajoa/ui-tokens` (no new sizes added in Phase 1). For Phase 1 only the **wordmark** (image, not text) and **OG-image text** (Phase 4 consumer, but assets bundled now) use these values.

| Role | Size | Weight | Line Height | Phase 1 use |
|------|------|--------|-------------|-------------|
| Body | 16px (1rem) | 400 (Regular) | 1.5 (normal) | Default for all future product copy. Korean readability tuned. |
| Label | 14px (0.875rem) | 500 (Medium) | 1.5 (normal) | Form labels, captions (Phase 3+) |
| Heading | 20px (1.25rem) | 600 (SemiBold) | 1.25 (tight) | Section headings (Phase 3+) |
| Display | 36px (2.25rem) | 700 (Bold) | 1.25 (tight) | OG image board title (Phase 4), splash text alternative |

**Phase 1 font pipeline contract:**

- **iOS:** `expo-font` loaded in `app/_layout.tsx`. Files bundled in `apps/ios/assets/fonts/` — `Pretendard-Regular.otf`, `Pretendard-Medium.otf`, `Pretendard-SemiBold.otf`, `Pretendard-Bold.otf`. All 4 loaded at startup (no lazy loading — total ~1.6MB acceptable for a Korean-first app).
- **Web:** `next/font/local` in `apps/web/app/layout.tsx`. Same 4 files in `apps/web/public/fonts/` (or `apps/web/assets/` if imported). Generates CSS variable consumed by `--font-sans` in `globals.css`.
- **OG image (Phase 4 consumer, file present in Phase 1):** `apps/web/assets/Pretendard-Bold.ttf` (Bold only — single weight needed for ImageResponse). `.ttf` format required by Satori (`.woff2` unsupported).
- **License:** SIL OFL 1.1 — `LICENSE-Pretendard.txt` checked into `apps/ios/assets/fonts/` and `apps/web/public/fonts/`.
- **Priority loading:** Regular + Bold are highest priority (cover body + display). Medium + SemiBold are also bundled but used later.

## Color

60/30/10 split. All values taken directly from existing `@moajoa/ui-tokens` `colors` — no new colors introduced in Phase 1.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (neutral.0) | App background, splash background, OG image background, web body |
| Secondary (30%) | `#F8FAFC` (neutral.50) + `#0F172A` (neutral.900) text | Surfaces (cards/sidebar later) and primary text on white |
| Accent (10%) | `#F97316` (brand.500 — "warm travel orange") | **Phase 1 reserved for:** (1) iOS app icon mark color, (2) wordmark "MOAJOA" letter color (or a single accent letter — see Wordmark spec below), (3) splash screen wordmark color if wordmark is rendered in brand color on white background. **Not used for any interactive element in Phase 1** (no interactive elements exist). |
| Destructive | `#DC2626` (semantic.danger) | Not used in Phase 1 — declared so contract is complete for Phase 3+ destructive confirmations |

**Accent reserved-for list (Phase 1):**

1. App icon mark fill (the single brand-color shape on the icon)
2. Wordmark letter color (full wordmark in `brand.500`, or "M" accent letter — see Wordmark Composition below)
3. Splash screen wordmark color (matches the wordmark spec)
4. Web favicon mark fill (derived from app icon)

**Explicitly NOT accented in Phase 1:** body text, links (none exist), backgrounds (must stay white), borders.

**Splash background color:** White (`#FFFFFF`). Brand color appears only on the wordmark/mark itself. Rationale: brand color background was considered (D-07 mentions "brand color splash") but a white background with brand-color wordmark reads cleaner against iOS's dark status bar transition and matches in-app surfaces (no jarring background flash on launch). **This is a deliberate refinement of D-07** — the "brand color" still anchors the splash visually via the wordmark; the surface stays neutral. If the planner/user prefers literal D-07 (brand color full-bleed background with white wordmark), flip both values in the splash spec below; everything else stays identical.

**Dark mode handling:** **None in Phase 1.** `userInterfaceStyle: 'light'` is locked in `app.config.ts` (already present). Splash, icon, and wordmark have a single light variant only. Dark mode is a v2 item per Out of Scope.

## Brand Assets (Phase 1 specific — not in default template)

### Wordmark Composition

| Property | Value |
|----------|-------|
| Source file | `packages/ui-tokens/src/brand/wordmark.svg` (single source of truth) |
| Text | `MOAJOA` (all caps, Latin) |
| Typeface | Pretendard Bold (700) — outlined to paths in SVG so font isn't required at render time |
| Letter spacing | -0.02em (tight, since all caps) |
| Color variant — color | All letters in `#F97316` (brand.500) |
| Color variant — mono dark | All letters in `#0F172A` (neutral.900) — for printing/dark-on-light contexts |
| Color variant — mono light | All letters in `#FFFFFF` — for future dark surfaces (not used in Phase 1, exported for completeness) |
| Aspect ratio | ~5:1 (width:height). Exact viewBox set in SVG; downstream code uses viewBox, no hard width. |
| Minimum render width | 96px (below this, switch to icon mark) |

### Icon Mark

| Property | Value |
|----------|-------|
| Source file | `packages/ui-tokens/src/brand/icon.svg` (single source of truth) |
| Shape | Single geometric mark — **a map pin formed from the letter "M"** (the mountain-like apex of M doubles as a pin head; bottom point indicates location). Simple, single-color, no gradient. Two-stroke construction so it renders cleanly at 16px (favicon). |
| Color (default) | `#F97316` (brand.500) on transparent/white background |
| Color (mono variants) | Same dark/light variants as wordmark for parity |
| Aspect ratio | 1:1 (square) |
| Foreground safe area | Content occupies inner **80%** of the 1024×1024 canvas (10% padding each side) per Apple HIG |
| Background | Solid white (`#FFFFFF`) for iOS app icon — Apple applies its own corner radius mask; we ship a square |
| Adaptive icon (Android future) | Foreground PNG (mark on transparent) + background PNG (solid white) exported now even though Android is v2 — costs nothing extra and locks the contract |

**Lockup rule:** Icon mark and wordmark are **separate marks**, not a combined lockup. The icon mark is used standalone for iOS app icon + favicon; the wordmark is used standalone for splash + web header (Phase 3+) + OG image (Phase 4). No "icon + wordmark together" composition in Phase 1.

### Splash Composition (iOS)

| Property | Value |
|----------|-------|
| Background color | `#FFFFFF` (neutral.0) |
| Content | Wordmark (color variant — `#F97316`) only — no icon mark, no tagline |
| Wordmark size | 60% of device width (e.g. ~234px wide on iPhone 15's 390pt width) |
| Wordmark position | Horizontally centered, vertically centered on screen |
| Generated file | `apps/ios/assets/splash.png` (@1x ≈ 1242×2688 for iPhone Pro Max, plus @2x/@3x variants per Expo's splash plugin) |
| Status bar style | Default (dark content on light background) — set via Expo `userInterfaceStyle: 'light'` |
| Animation | None (single static image). Lottie/animated splash deferred per Deferred Ideas. |

### iOS App Icon

| Property | Value |
|----------|-------|
| Canvas | 1024×1024 PNG (App Store size) |
| Background | Solid `#FFFFFF` |
| Foreground | Icon mark in `#F97316`, occupying inner 80% (10% padding each side) |
| Corner radius | **None applied by us** — iOS applies its own rounded-square mask. We ship a flat square. |
| File location | `apps/ios/assets/icon.png` (referenced from `app.config.ts` via `icon` field added in Phase 1) |
| Adaptive icon (future) | `apps/ios/assets/adaptive-icon.png` (foreground only, transparent bg) — generated now, wired up if/when Android arrives |

### Web Favicon + OG Default

| Property | Value |
|----------|-------|
| favicon.ico | 32×32 + 16×16 multi-resolution ICO, derived from icon mark (mark in `#F97316` on white) — `apps/web/app/favicon.ico` |
| apple-touch-icon | 180×180 PNG — `apps/web/public/apple-touch-icon.png` |
| OG default image | 1200×630 PNG, wordmark centered on white — `apps/web/public/og-default.png` (used for `/`, `/login`; per-board OG generated dynamically in Phase 4) |
| OG image (per-board, Phase 4) | Same 1200×630 dimensions, dynamically generated via `next/og`. **Template defined here so the Phase 4 executor inherits the contract:** wordmark top-left at 64px from top/left at ~120px width; board title (Pretendard Bold 72px, `#0F172A`) center-left; metadata line (Pretendard Regular 36px, `#475569` — neutral.600) below title; 80px page padding all sides. |

### Asset Export Matrix

Single source `packages/ui-tokens/src/brand/{wordmark,icon}.svg` → all derivatives via export script (`packages/ui-tokens/scripts/export-assets.{ts|mjs}`, sharp-based per D-06). Script outputs:

| Destination | Files | Source SVG |
|-------------|-------|------------|
| `apps/ios/assets/` | `icon.png` (1024²), `adaptive-icon.png` (1024²), `splash.png` (1242×2688) | icon.svg + wordmark.svg |
| `apps/web/app/` | `favicon.ico` (32+16 multi-res) | icon.svg |
| `apps/web/public/` | `apple-touch-icon.png` (180²), `og-default.png` (1200×630) | icon.svg + wordmark.svg |
| `apps/web/assets/fonts/` (Phase 4 consumer, present in Phase 1) | `Pretendard-Bold.ttf` (copied from `apps/ios/assets/fonts/`) | n/a (font file) |

Script is **idempotent** (re-run produces identical bytes) and **manually invoked** (not in CI) — runs when SVG changes. Outputs are committed (not in `.gitignore`) so CI/EAS builds don't need sharp.

## Web Dev-Tool Gate (Phase 1 interaction contract)

Per D-09/D-10/D-11/D-12. The only "interactive" UI contract Phase 1 introduces.

| Property | Value |
|----------|-------|
| Trigger condition | `NEXT_PUBLIC_ENABLE_DEV_TOOLS` env var is unset OR not exactly `"1"` |
| Affected route | `/boards` (and `/boards/[id]`) |
| Behavior | **Server-side redirect to `/login`** before any HTML is sent to the client — implemented in the page Server Component via `redirect('/login')` from `next/navigation`. No flash of dev-tool UI. |
| Component-level secondary gate | `<CreateBoardButton />` and `<AddLinkForm />` each check the env at render time and return `null` if gate is closed (defense-in-depth per D-09 "이중 게이트") |
| Loading state | None — redirect happens before render, so the user sees `/login` directly with no intermediate spinner |
| Error state | None — env check cannot throw; absence is treated as gate-closed (safe default) |
| User-visible copy | None added in Phase 1 — `/login` page already exists and is unchanged |

**No new visible UI** is added to the web in Phase 1. The "boards" forms remain in the codebase (per D-12 — path stays `/boards`, not moved to `/(dev)/boards`) but are invisible in any environment where the env var isn't set.
