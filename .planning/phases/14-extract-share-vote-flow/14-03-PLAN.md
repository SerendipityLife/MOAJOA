<objective>
장소 영상 미리보기(B안). `react-native-webview`를 추가하고, `VideoPreview` 컴포넌트가 `https://www.youtube.com/embed/{videoId}?start={startSec}&autoplay=1&playsinline=1`을 WebView로 로드(`allowsInlineMediaPlayback`, `mediaPlaybackRequiresUserAction={false}`). 기존 `PinBottomSheet`(마커/카드 탭 시 열리는 장소 상세 시트)에 이 플레이어 섹션을 추가한다. `videoId`는 `place.link_id`→link.url에서 `extractVideoId`(@moajoa/core 재사용)로 뽑는다. 타임스탬프 없거나 비유튜브면 플레이어 대신 출처 링크 버튼.

**autonomous:false** — 신규 네이티브 의존성(react-native-webview) 추가 → prebuild + pod install + 재빌드 + 시뮬레이터 재생 확인이 필요(현 iOS 빌드 절차, 14-CONTEXT 참고). 빌드/UAT는 사람 게이트.
</objective>

## Steps
1. `pnpm --filter @moajoa/ios add react-native-webview` (SDK 56 호환 라인). app.config plugins에 자동/수동 등록 확인.
2. `components/boards/video-preview.tsx` 신규: `VideoPreview({ videoId, startSec })` → WebView embed. 16:9 비율 박스. 로딩/에러 폴백.
3. `pin-sheet.tsx`: 장소가 유튜브 출처 + `source_timestamp_sec` 있으면 상단에 `<VideoPreview>`; 아니면 출처 링크 버튼(`buildYouTubeWatchUrl`/link.url). 14-02 `onPressSource`가 이 시트를 그 장소로 연다.
4. 빌드(게이트): `pnpm install` → `expo prebuild --clean -p ios` → `LANG/LC_ALL=en_US.UTF-8 pod install` → `pnpm sim "iPhone 16 Pro"`.
