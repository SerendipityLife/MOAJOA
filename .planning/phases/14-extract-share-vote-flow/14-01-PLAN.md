<objective>
백엔드 다리: 기존 `ensure_share_slug` 트리거는 **BEFORE INSERT 전용**이라 기존 보드의 visibility를 UPDATE해도 slug가 생성되지 않는다. 새 append-only 마이그레이션 `0014_share_board_rpc.sql`에 **SECURITY DEFINER** `share_board(p_board_id uuid) returns text` RPC를 추가해 (1) owner 확인, (2) `private`면 `shared`로 전환, (3) `share_slug` null이면 `ensure_share_slug`와 동일 레시피로 생성, (4) 최종 slug 반환을 원자적으로 처리한다. 그리고 `packages/api`에 `shareBoard()` 타입드 래퍼를 추가한다.

오프라인 검증 가능(grep + tsc). 실제 `db push` + `pnpm supabase:types`는 14-04 실행 게이트 또는 별도 BLOCKING 단계.
</objective>

## Steps

1. **마이그레이션 작성** — `supabase/migrations/0014_share_board_rpc.sql`
   - 패턴 참고: `0001_init.sql`의 `ensure_share_slug` 본문(slug 생성식), `0012`/`0009`의 SECURITY DEFINER RPC 스타일.
   - boards owner 컬럼명은 `0001_init.sql`에서 확인해 그대로 사용(직접 `auth.uid()` 비교만 안전 — CLAUDE.md §4.4).
   - 로직:
     ```
     declare v_slug text; declare v_owner uuid; declare v_vis text;
     select <owner_col>, visibility, share_slug into v_owner, v_vis, v_slug from boards where id = p_board_id;
     if not found then raise exception 'board not found'; end if;
     if v_owner <> auth.uid() then raise exception 'not board owner'; end if;
     if v_slug is null then v_slug := <ensure_share_slug 레시피 동일>; end if;
     update boards set visibility = case when visibility = 'private' then 'shared' else visibility end,
                       share_slug = v_slug
       where id = p_board_id;
     return v_slug;
     ```
   - `security definer` + `set search_path = public` + `grant execute ... to authenticated`.
2. **api 래퍼** — `packages/api/src/queries/boards.ts`에 `export async function shareBoard(client, boardId): Promise<string>` 추가 (`client.rpc('share_board', { p_board_id: boardId })`, error throw, data 반환). memberships.ts의 `joinSharedBoard` 형태 재사용.
