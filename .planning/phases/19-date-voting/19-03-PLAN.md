```typescript
export type PollEvent =
  | { kind: 'vote'; payload: unknown }
  | { kind: 'comment'; payload: unknown }
  | { kind: 'presence'; viewers: number };

export function subscribePollChannel(
  tripId: string,
  onEvent: (e: PollEvent) => void,
): RealtimeChannel {
  const channel = supabase
    .channel(pollChannelName(tripId))
    .on('broadcast', { event: 'vote' }, (msg) => onEvent({ kind: 'vote', payload: msg.payload }))
    .on('broadcast', { event: 'comment' }, (msg) => onEvent({ kind: 'comment', payload: msg.payload }))
    .on('presence', { event: 'sync' }, () => {
      const state = channel.presenceState();
      onEvent({ kind: 'presence', viewers: Object.keys(state).length });
    })
    .subscribe();
  return channel;
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host (authenticated) → confirm_poll_date | Host JWT crosses into the owner-guarded confirm RPC. The iOS client supplies pollId + chosen dates; the server enforces am_trip_owner. |
| host (authenticated) → date_polls.update (setPollMode) | Host JWT crosses into the mode-switch update; the existing `date_polls_write` host RLS (`can_edit_trip`, Plan 01) is the authorization boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04 | Elevation of Privilege | iOS confirm calling confirm_poll_date / setPollMode | mitigate | confirm_poll_date is SECURITY INVOKER + am_trip_owner guard (Plan 01). setPollMode is a `.update({mode})` under the existing `date_polls_write` host RLS (Plan 01) — a non-owner call is denied at the DB. The iOS card is only shown inside a trip the user owns; iOS adds no privileged path. |
| T-19-11 | Tampering | mid-poll mode flip stranding cast votes | mitigate | The mode toggle is gated client-side to the 0-vote / pre-share window (`canChangeMode = N === 0`) and rendered disabled once any vote exists, so a host can't flip range↔grid after votes are cast and orphan them. The DB write itself stays owner-guarded. |
| T-19-08 | Spoofing | invite URL exposing a forgeable code | accept | The /poll/{code} URL carries the server-minted poll_code (bearer capability by design, D-02). iOS only reads/shares the code; it cannot forge it. |
| T-19-02 | Tampering | host card trusting client tally | accept | The summary is advisory; the authoritative tally + confirm validation live server-side. contiguousBlock is advisory only (host picks — D-09). |
</threat_model>
