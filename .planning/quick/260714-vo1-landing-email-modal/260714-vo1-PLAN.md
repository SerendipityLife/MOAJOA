**F3 — 캐러셀 스크롤 락은 body 락으로 풀 문제가 아니다. 구조로 푼다.**
캐러셀이 스크롤하는 건 body 가 아니라 내부 트랙(`overflow-x-auto`)이고, 랜딩의 `<main>` 은 `h-[100svh] overflow-hidden` 이라 body 는 애초에 스크롤되지 않는다. body 락은 이 문제를 **전혀 못 막으면서** 호출부 0개인 공유 컴포넌트에 전역 부작용만 심는다 → 넣지 않는다.

실제 메커니즘 3겹:
1. **Dialog 를 트랙의 형제로 렌더** (트랙의 자손이 아니라). 백드롭이 트랙의 DOM 자손이면 백드롭 위의 wheel/touch 가 트랙으로 버블해 스크롤된다. 형제면 스크롤 체인이 트랙에 닿지 않는다. ← 이게 핵심
2. **포커스 트랩** → 포커스가 트랙 안으로 못 가므로 방향키 스크롤도 불가
3. **포커스 복귀 시 `focus({ preventScroll: true })`** → 복귀가 트리거를 화면에 넣으려고 트랙을 스크롤하는 것도 차단

## EmailAuthForm 인터페이스 (Task 1 에서 이걸 그대로 구현)

`/login` 과 모달이 **구조적으로 다른** 지점만 prop 으로 뺀다. `SocialAuthButtons` 가 `onProvider` 를 받는 것과 같은 논리.

```ts
export type EmailAuthSurface = 'blue' | 'light';

interface EmailAuthFormProps {
  surface: EmailAuthSurface;
  getCallbackUrl: () => string;
  onAuthenticated: () => void;
  socialSlot?: React.ReactNode;
}
```

| prop | 왜 caller 소유인가 |
|---|---|
| `surface` | `/login` 은 brand-700 파란 캔버스(흰/바나나 잉크), 모달은 흰 Dialog 패널(컴포넌트 기본값). 배경이 다르면 잉크가 반대다 — 진짜 구조적 차이 |
| `getCallbackUrl` | `/login` 은 `?next=` 를 라운드트립에 실어보내고, 랜딩은 **일부러 쿼리 없는** URL 을 만든다(open-redirect 표면 제거). 현 `landingCallbackUrl()` 의 docstring 이 이미 이 이유를 적어놨다. **함수로 받는다** — 렌더 시점 `window` 접근 회피, 제출 시점에만 호출 |
| `onAuthenticated` | `/login` 은 `postLoginDestination()`(`?next=` 존중), 랜딩은 항상 `/moa`. caller 가 `router.replace` 를 소유 → 폼은 `next/navigation` 무의존 |
| `socialSlot` | `/login` 은 CTA 와 회원가입 **사이**에 소셜을 끼운다. 모달은 안 넣는다(바로 뒤 슬라이드에 이미 있음). slot 이라 `/login` 의 기존 시각 순서가 그대로 보존된다 |

**폼이 소유하는 것 (= /login 에서 그대로 들어냄):** `mode`/`email`/`password`/`pending`/`error`/`magicSent` state, `signIn`/`signUp`/`sendMagicLink`, 그리고 3개 렌더 분기(magicSent / password / magic).

**/login 에 남는 것:** `postLoginDestination()`, `callbackUrl()`, `oauth()`, `/login?error=` 콜백 에러 effect, `useToast`, wordmark/chip/h1/`MapIllustration`.
