<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EAS Build 서버 → Apple Developer ad-hoc provisioning | EAS가 우리의 Apple Developer 계정 위임 받아 sign |
| EAS dashboard QR → 실기기 .ipa install | QR이 인증된 URL인지 확인 (eas-cli 출력 신뢰) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | EAS build cache가 path A의 부분 상태 사용 | mitigate | `eas build --clear-cache` 옵션 (필요 시). 첫 빌드는 cache 없이 깨끗하게 |
| T-04-02 | Information Disclosure | eas.json에 secret 포함 | accept | eas.json에 secrets 없음 — projectId만 (public OK). secrets는 EAS Secret API (이번 phase는 안 씀) |
| T-04-03 | Denial of Service | EAS 무료 tier 한도 초과 | accept | Expo free tier — 월 30 빌드 제한. dogfooding 단계에서 충분. 한도 초과 시 paid plan ($29/mo) — Phase 6 dogfooding 후 평가 |
| T-04-04 | Elevation of Privilege | ad-hoc provisioning이 의도치 않은 device에도 install 가능 | mitigate | `eas device:create`로 본인 iPhone UDID만 등록. 다른 device는 install 불가 |
| T-04-05 | Repudiation | Path B 전환 사유 미기록 | mitigate | D-14: SESSION-NOTES에 timeline + 사유 필수 기록. Task 1 Step 0에서 헤더 작성 강제 |
</threat_model>
