export async function getPollByTrip(
  client: MoajoaSupabaseClient,
  tripId: string,
): Promise<{ id: string; poll_code: string; mode: 'range' | 'grid'; status: 'open' | 'closed' } | null> {
  const { data, error } = await client
    .from('date_polls')

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client code → Supabase RPC | The api wrappers are the single call site that translates app intent to `rpc('...')`. They must never offer a direct table-write path for votes/comments (that would re-open T-19-01). |
| host client → date_polls.update (setPollMode) | The mode-switch wrapper writes `date_polls.mode` directly; the existing host RLS (`date_polls_write` / `can_edit_trip`) is the authorization boundary — a non-owner update is rejected by RLS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Elevation of Privilege | api offering a direct date_votes/date_comments insert | mitigate | date-polls.ts writes go through `client.rpc('cast_date_vote'|'post_poll_comment')` ONLY; test asserts 0 `from('date_votes').insert(` calls. Inherits the DB-level grant lockdown from Plan 01. |
| T-19-04 | Elevation of Privilege | setPollMode writing date_polls.mode | mitigate | `setPollMode` is a plain `.update({mode})` under the existing `date_polls_write` host RLS (`can_edit_trip(trip_id)`, Plan 01) — a non-owner update is denied at the DB. No new grant/surface; reuses the host policy that already guards poll create/edit. |
| T-19-05 | Input Validation | unvalidated payloads reaching the RPC | mitigate | @moajoa/core Zod (CastDateVoteRequestSchema/PostCommentRequestSchema) narrows availability/nickname/body before the call; server re-validates in the DEFINER RPC (Plan 01) — defense in depth. mode is narrowed to `'range'|'grid'` by the setPollMode signature + the date_polls CHECK. |
| T-19-07 | Information Disclosure | wrapper exposing raw rows | accept | getPollTally returns only the shaped jsonb from poll_vote_tally (Plan 01 guarantees no device_token). getPollByTrip returns only id/poll_code/mode/status (host-readable poll metadata, no voter PII). |
