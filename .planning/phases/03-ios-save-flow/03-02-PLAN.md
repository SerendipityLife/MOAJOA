<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iOS Share sheet → Share Extension process | Untrusted URL from any source app (Safari, KakaoTalk, Mail) enters the extension. |
| Share Extension → App Group SharedDefaults | Data written here is readable by main app — must be validated before use (Plan 03-04 Zod step). |
| EAS Build / Apple Developer Portal | App Group capability provisioning. Misconfiguration = silent SharedDefaults nil. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | URL received from share sheet (any source app) | mitigate | Main app validates URL via Zod LinkAddSchema before INSERT — handled in Plan 03-04 drainPendingLinks. Share Extension itself does not trust-decide; it just enqueues. |
| T-03-02-02 | Spoofing | App Group identifier collision with malicious app | mitigate | Apple sandbox: App Groups are scoped to the developer team. A different team's app cannot read/write our App Group (transferred to Apple's signing/entitlement enforcement). |
| T-03-02-03 | Information Disclosure | SharedDefaults read by another extension on the same device | accept | Same-team apps can read App Group data. MOAJOA has only one app target + one extension target both under our Team ID. No PII or auth tokens are written here (Plan 03-04 D-02 lock: only board_id + URL). |
| T-03-02-04 | Denial of Service | Adversary spams share to fill pending_links | mitigate | retry > 3 fallback (D-06) caps queue growth. Failed banner surfaces accumulation to user. |
</threat_model>
