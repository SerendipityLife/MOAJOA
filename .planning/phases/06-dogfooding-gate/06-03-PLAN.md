### 파일 1: `scripts/dogfooding/p90-duration.sql`

D-11 lock + Phase 3 D-11 reuse. extraction_costs.duration_ms 집계 (link_id 별 SUM → percentile_cont(0.9)).

핵심 쿼리:
```sql
-- p90 + avg + n for places-extraction calls only
-- Excludes resolve-place calls (link_id IS NULL after Phase 3 0005 migration)
SELECT
  percentile_cont(0.9) WITHIN GROUP (ORDER BY total_ms)::int AS p90_ms,
  AVG(total_ms)::int AS avg_ms,
  COUNT(*) AS n_links
FROM (
  SELECT link_id, SUM(duration_ms) AS total_ms
  FROM extraction_costs
  WHERE link_id IS NOT NULL
    AND created_at >= COALESCE(:'since', NOW() - INTERVAL '7 days')
  GROUP BY link_id
) t
WHERE total_ms IS NOT NULL;
```

`:since`은 optional `-v since="'2026-MM-DD'"` 인자. 미설정 시 7일.

SAVE-02 target은 < 30000 ms. 결과를 daily-log "End-of-Week SQL Snapshot" 섹션에 paste.

### 파일 2: `scripts/dogfooding/daily-aggregate.sql`

D-11 그대로. 한 보드의 일자별 links + places 카운트 + 핀 누적.

핵심 쿼리:
```sql
-- Daily activity for dogfooding board
-- Usage: psql ... -f daily-aggregate.sql -v board_id="'<uuid>'"
WITH daily_links AS (
  SELECT date_trunc('day', created_at)::date AS day, COUNT(*) AS links_added
  FROM links WHERE board_id = :board_id::uuid
  GROUP BY 1
),
daily_places AS (
  SELECT date_trunc('day', created_at)::date AS day, COUNT(*) AS places_added
  FROM places
  WHERE board_id = :board_id::uuid AND hidden_at IS NULL
  GROUP BY 1
),
all_days AS (
  SELECT day FROM daily_links UNION SELECT day FROM daily_places
)
SELECT
  d.day,
  COALESCE(l.links_added, 0) AS links_added,
  COALESCE(p.places_added, 0) AS places_added,
  SUM(COALESCE(p.places_added, 0)) OVER (ORDER BY d.day) AS places_cumulative
FROM all_days d
LEFT JOIN daily_links l USING (day)
LEFT JOIN daily_places p USING (day)
ORDER BY d.day;
```

D-13 enforce: `hidden_at IS NULL`. places_cumulative ≥ 10 by Day 7 → Pass.

### 파일 3: `scripts/dogfooding/measure-accuracy.sql`

Baseline 측정 시 actual extraction side 데이터 추출. 본 SQL은 places 테이블만 — ground truth 비교는 본인이 markdown 표에서 수동 (D-08).

12 sample 영상의 URL을 array로 받아서 각 영상의 추출 결과 dump:

```sql
-- Usage: psql ... -f measure-accuracy.sql -v urls="ARRAY['https://youtu.be/A','https://youtu.be/B',...]"
-- Output: per-sample row with extracted places (jsonb agg)
SELECT
  l.url,
  l.extraction_status,
  COUNT(p.id) FILTER (WHERE p.hidden_at IS NULL) AS extracted_count,
  jsonb_agg(
    jsonb_build_object(
      'id', p.id,
      'name_local', p.name_local,
      'name_ko', p.name_ko,
      'google_place_id', p.google_place_id,
      'inferred_city', p.inferred_city,
      'source_kind', p.source_kind,
      'confidence', p.confidence,
      'video_offset_sec', p.source_timestamp_sec,
      'quote', p.source_quote
    ) ORDER BY p.source_timestamp_sec NULLS LAST
  ) FILTER (WHERE p.id IS NOT NULL AND p.hidden_at IS NULL) AS extracted_places
FROM links l
LEFT JOIN places p ON p.link_id = l.id
WHERE l.url = ANY(:'urls'::text[])
GROUP BY l.id, l.url, l.extraction_status
ORDER BY l.url;
```

본인은 위 SQL 결과를 `.planning/dogfooding/extraction-baseline-YYYY-MM-DD.md`의 영상별 표 "extracted_count / extracted_places" 칸에 paste. 그 옆에 ground_truth 칸을 본인이 채워서 markdown 비교.
