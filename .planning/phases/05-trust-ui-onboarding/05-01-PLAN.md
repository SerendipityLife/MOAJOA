## Goal

`places.confidence` 컬럼 + `public_board_view` RPC 재정의(confidence·source_kind 포함) + `profiles_create_first_board` 트리거 + 기존 사용자 backfill을 단일 마이그레이션 `0006_trust_ui_onboarding.sql`에 lock한다. 동시에 `packages/core/src/constants.ts`에 Phase 5 신규 상수 3개(`EXTRACT_STEP_KO`, `LOW_CONFIDENCE_THRESHOLD`, `OnboardKeys`)를 export하고, Edge Function `extract-youtube/index.ts`의 places upsert에 `confidence` 1줄을 wire한다.

## Why

- TRUST-04는 `confidence < 0.7` 시각 분기 → 컬럼 + RPC 반환이 먼저 있어야 iOS·Web 양쪽이 데이터 받음 (CONTEXT D-01/D-03)
- TRUST-01 (Web parity)도 `source_kind`가 RPC jsonb에 누락되어 있어 추가 필요 — 현재 `public_board_view` SELECT list에 source_kind 없음(0001_init.sql:539-549 확인)
- ONBOARD-01은 Postgres 트리거 + backfill = 클라이언트 코드 0줄 (CONTEXT D-16/D-17/D-18)
- 상수가 코어에 먼저 lock되어야 02~06 plan이 import (Karpathy §3.1 — 가정 lock, 단일 출처)

## Verification Loop

```
1. Migration apply (local supabase) → verify
2. pnpm supabase:types → packages/api/src/types/database.ts에 confidence 컬럼 + source_kind 등장 확인
3. Edge Function 재배포 → 새 추출 1건 실행 → places.confidence not null
4. SQL: SELECT id, title FROM boards WHERE owner_id IN (SELECT id FROM profiles) — 모든 profile에 보드 ≥ 1
5. packages/core build → confirmAiPlace 등 후속 import 가능
```

### T1 — Migration 0006 작성

내용 (CONTEXT.md `<specifics>` skeleton 기반):

```sql
-- =============================================================================
-- Phase 5: Trust UI & Onboarding foundation
-- =============================================================================
-- Part 1: places.confidence numeric(3,2) — per-pin AI confidence (D-01)
-- Part 2: public_board_view RPC redefine — append confidence + source_kind (D-03)
-- Part 3: profiles_create_first_board trigger — auto "내 첫 여행" (D-16/D-17)
-- Part 4: backfill existing profiles → 1 board each (D-18)
-- =============================================================================

-- ---- Part 1: per-place confidence column -----------------------------------

alter table places
  add column if not exists confidence numeric(3,2)
    check (confidence is null or (confidence >= 0 and confidence <= 1));

-- ---- Part 2: public_board_view RPC redefinition ----------------------------
-- Append `confidence` + `source_kind` to places jsonb. Existing fields preserved.
-- (Re-issue full CREATE OR REPLACE so SELECT list is explicit — append-only intent.)

create or replace function public_board_view(p_slug text)
returns jsonb
language plpgsql
stable
security definer
set search_path = public
as $$
declare
  v_board boards%rowtype;
  v_owner profiles%rowtype;
  v_result jsonb;
begin
  select * into v_board from boards
  where share_slug = p_slug and visibility = 'public'
  limit 1;

  if not found then return null; end if;

  select * into v_owner from profiles where id = v_board.owner_id;

  v_result := jsonb_build_object(
    'board', jsonb_build_object(
      'id', v_board.id,
      'title', v_board.title,
      'description', v_board.description,
      'city_code', v_board.city_code,
      'cover_image_url', v_board.cover_image_url,
      'updated_at', v_board.updated_at
    ),
    'owner_display_name', coalesce(v_owner.display_name, 'MOAJOA user'),
    'links', coalesce(
      (select jsonb_agg(jsonb_build_object(
        'id', l.id,
        'source_kind', l.source_kind,
        'url', l.url,
        'title', l.title,
        'thumbnail_url', l.thumbnail_url,
        'author_name', l.author_name
      ) order by l.created_at desc)
      from links l where l.board_id = v_board.id and l.extraction_status = 'ready'),
      '[]'::jsonb
    ),
    'places', coalesce(
      (select jsonb_agg(jsonb_build_object(
        'id', p.id,
        'link_id', p.link_id,
        'name_local', p.name_local,
        'name_ko', p.name_ko,
        'name_en', p.name_en,
        'lat', p.lat,
        'lng', p.lng,
        'category', p.category,
        'source_timestamp_sec', p.source_timestamp_sec,
        'source_kind', p.source_kind,        -- NEW (Phase 5 TRUST-01 web parity)
        'confidence', p.confidence            -- NEW (Phase 5 TRUST-04)
      ) order by p.created_at)
      from places p where p.board_id = v_board.id and p.hidden_at is null),
      '[]'::jsonb
    )
  );

  return v_result;
end;
$$;

grant execute on function public_board_view(text) to authenticated, anon;

-- ---- Part 3: profiles_create_first_board trigger ---------------------------
-- Runs AFTER INSERT on profiles. profiles itself is created by
-- handle_new_auth_user (0001_init.sql:85) which fires AFTER INSERT on auth.users
-- — so this trigger runs in the same transaction, AFTER the profile row exists.
-- Idempotent guard: skip if owner already has any board.

create or replace function profiles_create_first_board()
returns trigger
language plpgsql
security definer
set search_path = public
as $$
begin
  if not exists (select 1 from boards where owner_id = NEW.id) then
    insert into boards (owner_id, title, visibility)
    values (NEW.id, '내 첫 여행', 'private');
  end if;
  return NEW;
end;
$$;

drop trigger if exists profiles_first_board_trigger on profiles;
create trigger profiles_first_board_trigger
  after insert on profiles
  for each row execute function profiles_create_first_board();

-- ---- Part 4: backfill existing profiles ------------------------------------
-- One-shot: existing dogfooders get the same "내 첫 여행" experience.
-- Idempotent (NOT EXISTS guard) — safe to re-run if migration is replayed.

insert into boards (owner_id, title, visibility)
select p.id, '내 첫 여행', 'private'
from profiles p
where not exists (select 1 from boards where owner_id = p.id);
```

```ts
/**
 * Phase 5 TRUST-04 — D-04/D-14.
 * Confirms a low-confidence AI pin: flips source_kind to 'manual' and clears
 * confidence (so it's no longer rendered with the "low conf" treatment).
 * Reusing source_kind avoids a separate `confirmed_at` column (D-04 schema lock).
 */
export async function confirmAiPlace(
  client: MoajoaSupabaseClient,
  id: string,
): Promise<Place> {
  const { data, error } = await client
    .from('places')
    .update({ source_kind: 'manual', confidence: null })
    .eq('id', id)
    .select('*')
    .single();
  if (error) throw error;
  return data as Place;
}

/**
 * Phase 5 TRUST-04 — D-14 [잘못됨] action.
 * Soft-delete via hidden_at — same semantics as hidePlace/deletePlace
 * (places_board_idx WHERE hidden_at IS NULL already excludes hidden rows).
 * Exported with intent-aligned name for the UI.
 */
export const rejectAiPlace = hidePlace;
```

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (authenticated user) → places UPDATE | 보드 멤버가 confirm/reject 액션으로 places UPDATE — RLS gate |
| Edge Function (service role) → places INSERT | extract-youtube가 admin client로 INSERT, confidence 값 신뢰 |
| auth.users → profiles → boards (트리거 chain) | 신규 가입 트랜잭션. SECURITY DEFINER 함수가 RLS 우회로 boards INSERT |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | client UPDATE places.confidence (악의적 999) | mitigate | DB CHECK constraint `confidence is null or (confidence >= 0 and confidence <= 1)` (Part 1) |
| T-05-02 | Elevation of Privilege | 비-멤버가 confirm/reject helper로 다른 보드 places UPDATE | mitigate | 기존 RLS policy `places: update if can_edit_board` (0001) — helper는 단순 wrapper, 별도 검증 불필요 |
