5. **행 구조 — 이 task의 핵심.** 지금은 행 전체가 `DropdownMenuItem` 하나다. Radix `Item`의 `onSelect`는 Item 안 아무 데나 클릭해도 발화하고 메뉴를 닫는다. 아이콘 버튼을 Item **안에** 넣으면 (a) 모아 라우팅이 같이 터지고 (b) 메뉴가 닫혀 인라인 편집이 불가능해진다. 따라서 행을 `flex` `<div>`로 감싸고, `DropdownMenuItem`(이름 영역, `className="min-w-0 flex-1"`)과 아이콘 버튼들을 **형제**로 둔다. 버튼이 Item의 자손이 아니면 Radix가 selection 이벤트를 만들 수 없다 — 이게 "연필 눌러도 메뉴가 안 닫힌다"의 구조적 보장이다.
   - `DropdownMenuItem`은 기존 `onSelect={() => onPick(m.id)}` + `aria-current` 스프레드를 그대로 유지한다(기존 테스트 2·3·4가 이 계약에 의존).
   - 아이콘 버튼: `type="button"`, `aria-label="이름 수정"` / `aria-label="삭제"`, 안에 `<Pencil className="size-4" aria-hidden />` / `<X className="size-4" aria-hidden />`. 클래스는 주변 톤에 맞춰 `shrink-0 rounded-md p-1.5 text-neutral-400` 정도로 최소한만.
   - `const isOwner = m.owner_id === currentUserId;` 일 때만 두 버튼을 렌더한다. 비소유 모아에서 `trips` UPDATE/DELETE RLS(0016 `trips: owner full access`)는 조용히 0행을 반환하므로, 실패할 버튼은 아예 보여주지 않는다.

6. **편집 행**: `editingId === m.id`이면 이름 영역 대신 `<input>`을 렌더한다 (`autoFocus`, `aria-label="모아 이름"`, `value={draft}`, `onChange`로 draft 갱신). 이 행에는 아이콘 버튼을 렌더하지 않는다.

7. **입력창 keyDown** — 세 가지를 한 핸들러에서 처리한다:
   - 맨 먼저 `e.stopPropagation()`. Radix Menu Content는 printable key를 typeahead로 가로채 포커스를 항목으로 튕겨낸다. 입력창에서 발생한 키가 Content의 React onKeyDown까지 올라가지 않게 막아야 한글 타이핑이 정상 동작한다.
   - 그 다음 `if (e.nativeEvent.isComposing || e.keyCode === 229) return;` — 한글 IME 조합 확정용 Enter를 저장으로 오인하지 않는다.
   - `Enter` → `preventDefault()` + 저장 호출. `Escape` → `preventDefault()` + `setEditingId(null)`.
   - 참고: Radix의 Escape 처리는 document capture 리스너라 stopPropagation으로 막히지 않아 메뉴가 함께 닫힐 수 있다. Esc = 취소이므로 허용한다. 다만 **메뉴가 닫힐 때 편집 상태가 남지 않도록** `<DropdownMenu onOpenChange={(open) => { if (!open) setEditingId(null); }}>`을 추가한다 (다음 오픈 때 입력창이 유령처럼 남는 것 방지).

8. **저장(commitRename)**: `draft.trim()`이 빈 문자열이거나 기존 제목과 같으면 저장 없이 편집만 종료. 아니면 이전 `rows`를 보관 → 낙관적으로 `rows`의 해당 title 갱신 + `setEditingId(null)` → try 안에서 `TripUpdateSchema.parse({ title: next })`로 검증(외부 입력, CLAUDE.md §4.5 — 길이 초과 시 throw)하고 `updateTrip(getSupabaseBrowser(), id, patch)` → `router.refresh()`. catch: `console.error(err)` + `setRows(보관본)` 롤백 + `toast('이름을 바꾸지 못했어요', { variant: 'error' })`.

9. **삭제(onDelete)**: `window.confirm`을 account-sheet 선례 그대로 쓴다. 문구에 **담긴 장소도 함께 사라진다는 사실을 포함**한다 (예: `` `'${m.title}' 모아를 삭제할까요? 담긴 장소도 함께 사라져요.` ``). 취소면 즉시 return. 승인이면 이전 `rows` 보관 → 낙관적으로 목록에서 제거 → `deleteTrip(getSupabaseBrowser(), m.id)`. 실패 시 `console.error` + 롤백 + `toast('삭제하지 못했어요', { variant: 'error' })` 후 return. 성공 시: 삭제 대상이 현재 모아면 `router.push('/moa')` — `/moa`는 이미 리다이렉트 허브라 남은 최근 모아(없으면 `/onboarding`)로 알아서 보낸다. **별도 분기 로직을 만들지 말 것.** 현재 모아가 아니면 `router.refresh()`.
