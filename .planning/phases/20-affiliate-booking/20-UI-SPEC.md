## Spacing Scale

Declared values (Tailwind 4px base; tokens map `spacing.N` = N×4px) — inherited unchanged from the approved 19-UI-SPEC:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inline padding, status-glyph gaps |
| sm | 8px | Compact spacing — compare-frame row gaps, checklist row `mb-2.5` (10px is the existing plan-item-row exception, see below) |
| md | 16px | Default card inner padding (`px-4 py-4` big cards), field padding |
| lg | 24px | Screen gutter (`px-6` — matches plan tab ScrollView `paddingHorizontal: 24`), card-to-section breaks |
| xl | 32px | Gap between checklist list and 항목 추가 button (`mt-8` — mirrors 친구와 같이 정하기 placement) |
| 2xl | 48px | ScrollView `paddingBottom: 48` (plan tab idiom) |
| 3xl | 64px | not used this phase |

Exceptions (all pre-existing idioms, carried over):
- **44px minimum touch targets** — checklist status control, [보기] buttons, delete affordances: visual size may be smaller but hit area ≥44px via `w-11 h-11` containers or `hitSlop` (plan-item-row idiom).
- **`mb-2.5` (10px)** row rhythm — checklist rows mirror plan-item-row's existing 10px card gap for visual continuity inside the same app.
- **ROW_SHADOW** (`shadowOpacity 0.05, radius 4, elevation 1`) on list-row cards; `cardShadow` on standalone cards — both existing constants, reuse verbatim.
- Bottom-sheet top radius `radii.3xl` (28px) per token.

## Typography

Weights restricted to **regular (400)** + **semibold (600)** (19-UI-SPEC discipline). Four sizes:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 (`normal`) |
| Label | 16px (`text-base`) | 600 semibold | 1.5 (`normal`) |
| Heading | 18px (`text-lg`) | 600 semibold | 1.25 (`tight`) |
| Display | 20px (`text-xl`) | 600 semibold | 1.25 (`tight`) — empty-state headings only (current book.tsx stub already uses `text-xl font-semibold`) |

Caption / metadata convention (inherited from 19-UI-SPEC, **not a 5th tier**): 12px (`text-xs`), regular 400 only, `neutral-400/500` — used for compare-frame provider labels, status hints, progress counts, date·city prefill captions, '플랜에 없음' badges. Card titles inside list rows use Body size at semibold (`text-sm font-semibold`) per plan-item-row idiom.

## Color

60/30/10 split from `packages/ui-tokens`. **Booking surfaces must NOT read as an ad billboard (D-02)** — cards are white/neutral; brand appears only on the small set below.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `neutral-50 #F9FAFB` | Screen background (`bg-white`), card surfaces, sheet surfaces |
| Secondary (30%) | `neutral-100 #F3F4F6` / `neutral-200 #E5E7EB` | Icon chips on booking cards (`bg-neutral-100` + `neutral-600` glyph — deliberately NOT brand-50), borders (`border-neutral-200`), compact-strip fills (`bg-neutral-50`), disabled/empty surfaces |
| Accent (10%) | MOAJOA Blue — `brand-500 #2979FF`, `brand-600 #2563EB`, `brand-50 #F5F7FF` | See reserved list below |
| Success (semantic) | `semantic.success #10B981` | '완료' status glyph ONLY (`checkmark-circle`) — semantic completion, not accent |
| Destructive | `semantic.danger #EF4444` | Checklist 항목 삭제 action + confirm button only |

Accent reserved for (explicit — NEVER "all interactive elements"):
- **[보기] buttons** — every provider open button: `bg-brand-50` stadium/rounded fill + `text-brand-600 text-xs font-semibold` (LegPill idiom scaled to a button). This is THE booking CTA of the phase.
- **'확인함' status** — clicked-state glyph `ellipse-outline` in `brand-500` + the `확인함` badge (`bg-brand-50` chip, `text-brand-600 text-[10px]` — mirrors the 필수 tag idiom).
- **항목 추가 glyph** — `add-circle-outline` in `brand-500` (mirrors UnplacedPool's 일정에 추가).
- **플랜 탭으로 가기 text link** — `text-brand-600` in the no-plan empty state.
- Neutral-only (NOT brand): booking card icon chips, provider names, static comparison labels, 미완료 status circles, section labels, hint text "예약했으면 체크해주세요", '플랜에 없음' badge, expand chevrons.

'완료' is green (semantic), NOT brand — the three checklist states are visually: neutral → brand → green (D-11 progression reads at a glance).

All copy Korean, warm/casual — matches existing tone ("곧 제공돼요", "이동시간 —", "친구와 같이 정하기").

| Element | Copy |
|---------|------|

| Stay card title | **숙소 예약** · caption `{도시} · {MM.DD–MM.DD}` (prefill context, `text-xs neutral-400`) |
| Stay compare rows (D-05 비제휴, provider-swap ready) | **Agoda ─ 아시아 숙소 강세 [보기]** / **Booking.com ─ 전 세계 숙소 폭넓게 [보기]** |
| eSIM row | **여행 유심** · **Airalo ─ eSIM 즉시 발급 [보기]** |
| Transport row | **{패스명 labelKo, e.g. JR 패스}** · **{provider} [보기]** (from `BOOKING_REGION_MAP`; row hidden when mapping is null — D-09 억지 추천 금지) |
| Activity compare labels (D-06, RESEARCH-locked) | **Klook ─ 즉시확정·전세계 상품 [보기]** / **KKday ─ 한국어 가이드 상품 강세 [보기]** |
| Activity compact strip (plan tab) | **예약 비교** + `Klook [보기] · KKday [보기]` |
| Book tab title | **예약 체크리스트** · progress caption **{done}/{total} 완료** |
| Status badge (clicked) | **확인함** |
| Return hint (D-15, clicked rows only) | **예약했으면 체크해주세요** (`text-xs text-neutral-500` — quiet, no popup) |
| Plan-desync badge (D-13) | **플랜에 없음** (`bg-neutral-100 text-neutral-500 text-[10px]` chip) |
| Manual add CTA | **항목 추가** → sheet title **예약 항목 추가**, input placeholder **예: 항공권, 레스토랑 예약**, submit **추가하기** |
| Empty state (dateless trip, D-04) | heading **일정이 정해지면 예약을 시작할 수 있어요** / body **날짜 투표가 끝나면 예약 체크리스트가 열려요.** |
| Empty state (dated, no plan draft, D-04) | heading **먼저 플랜을 만들어주세요** / body **플랜 탭에서 플랜을 만들면 예약할 항목이 자동으로 정리돼요.** + text link **플랜 탭으로 가기** |
| Error state (checklist load fail) | **체크리스트를 불러오지 못했어요. 잠시 후 다시 시도해주세요.** + **다시 시도** button (mirrors plan tab State F) |
| Error state (booking click log fail) | **NONE — silent.** fire-and-forget (D-14); the browser already opened, no toast, no blocking |
| Destructive confirmation (항목 삭제) | **이 항목을 삭제할까요?** / 버튼: **삭제** (`danger #EF4444`) · **취소** (`Alert.alert` — existing idiom) |
| 완료 check | no confirmation — direct toggle (D-11: 완료의 원천은 사용자) |
| Affiliate footer slot (flag-gated, DEFAULT OFF — D-16 + RESEARCH Open Q2) | **예약 시 수수료를 받을 수 있어요** (`text-[10px] text-neutral-400`, card bottom) |
| Settings disclosure (D-16 home) | section **제휴 안내** — **MOAJOA는 일부 예약 링크에서 제휴 수수료를 받을 수 있어요. 결제 금액은 달라지지 않아요.** |

## Screen Contracts

Five iOS surfaces + one shared component. Icon refs: Ionicons.

### Component 0 — CompareFrameCard (shared, D-06/D-07)
**File:** new `apps/ios/components/booking/compare-frame-card.tsx` (RESEARCH structure)
**Goal:** One card that reads as a COMPARISON, not two thrown buttons. Reused for stay (Agoda/Booking) and activity (Klook/KKday).

**Anatomy (full variant):**
- Container: `bg-white rounded-2xl px-4 py-4` + `cardShadow` (standalone) or borderless when embedded inside an expanded checklist row.
- Header row: icon chip `w-10 h-10 rounded-xl bg-neutral-100` + kind glyph (`neutral-600`, size 18) + title (`text-sm font-semibold text-neutral-900`) + optional right-aligned caption (`text-xs text-neutral-400` — e.g. 도시·날짜 prefill context).
- Provider rows (2, `mt-3`, `space-y-2` rhythm): each row = provider name (`text-sm font-semibold text-neutral-900`) + `─` + static label (`text-xs text-neutral-500`) + **flex-1 spacer (THE PRICE SLOT — D-06: 가격이 후일 이 자리에 끼어도 레이아웃 무변경. Leave as `<View className="flex-1" />`)** + [보기] button.
- [보기] button: `bg-brand-50 rounded-full px-3 py-1.5`, text `text-xs font-semibold text-brand-600`, hit area ≥44px (`hitSlop`), `accessibilityRole="button"`, label "{provider}에서 보기". Pressed: `active:opacity-70`. **No loading state** — tap fires Safari immediately (D-14).
- Optional footer slot: flag-gated `text-[10px] text-neutral-400 mt-2` disclosure line (default off — D-16).

**Compact variant** (plan-tab activity strip, D-02 "작은 버튼/컴팩트"): single row `bg-neutral-50 border border-neutral-100 rounded-xl px-3 py-2` — `ticket-outline` glyph (14, `neutral-400`) + **예약 비교** (`text-xs text-neutral-500`) + spacer + `Klook [보기] · KKday [보기]` mini buttons (same brand-50 style, `py-1`). Static labels omitted in compact; the full labeled frame lives in the book tab.

### Screen 4 — Click Moment + Return (D-14/D-15, ATTR-02)
**Applies to every [보기] button (both tabs).**

- Tap → `Linking.openURL(trackingUrl)` **immediately** — no confirmation sheet, no interstitial, no spinner, logging never awaited before open (RESEARCH Pattern 1). Visual feedback = pressed opacity only.
- Log failure = silent (no toast; the '확인함' transition just doesn't happen — acceptable per RESEARCH).
- App return (foreground focus): checklist refetches quietly; affected row shifts 미완료→확인함 (glyph turns brand, 확인함 badge appears, hint line appears). **No popup, no banner, no animation callout** (D-15).

## Cross-Screen Interaction Notes

- **Accent discipline:** brand appears ONLY on [보기] buttons, 확인함 status, the add-circle glyph, and the one empty-state link. Booking cards themselves are white/neutral — the schedule must never read as an ad billboard (D-02). 완료 is green (semantic), deliberately distinct from brand.
- **One dataset, two surfaces (D-03):** plan-tab cards and book-tab rows point at the same checklist items; a click from either surface transitions the SAME item to 확인함.
- **Gate mutual exclusivity:** dateless trips show the 19-03 날짜투표 management card (plan tab) / dateless empty (book tab); dated+plan trips show booking surfaces. No state renders both.
- **Coverage honesty (D-09):** uncovered city ⇒ the esim/transport row simply doesn't exist — never a disabled or "준비중" booking row.
- **Touch targets:** every tappable (status control, [보기], chevron, delete) ≥44px hit area, matching plan-item-row's `w-11 h-11` pattern.
- **Copy source of truth:** static comparison labels and 패스명 live in `packages/core` (`booking-map.ts` per RESEARCH) — UI renders constants, never hardcodes provider copy in components.
- **UAT note (RESEARCH Pitfall 8):** Klook/KKday search-landing correctness is device-UAT-only (bot-blocked) — the [보기] contract here is visual/interaction; landing verification is a plan checkpoint.
