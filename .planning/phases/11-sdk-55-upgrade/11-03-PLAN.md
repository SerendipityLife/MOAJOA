# Plan 11-03: prebuild + 네이티브 회귀 검증 (SDK 55)

**Phase:** 11 (SDK 54 → 55)
**Wave:** 3 (depends 11-02)
**autonomous:** 빌드/테스트 true · 시각 회귀는 시뮬레이터 스크린샷 + 런타임 로그인 육안 게이트
**Requirements:** UPGRADE-01(부분), UPGRADE-03

## 목표

SDK 55 버전셋 위에서 ios/ 네이티브를 재생성하고, 네이티브 설정이 보존됐는지 + 핵심 기능(구글맵·share extension·bottom-sheet·gesture·로그인)이 회귀 없이 동작하는지 검증한다.

## Tasks

- **T1** — `pnpm --filter @moajoa/ios exec expo prebuild --clean -p ios` (SDK 55 네이티브 재생성).
- **T2** — grep으로 네이티브 설정 보존 확인:
  - `GMSApiKey` (Info.plist) · App Group entitlement `group.com.serendipitylife.moajoa` · ShareExtension target(`com.serendipitylife.moajoa.ShareExtension`) · jsEngine=Hermes(JSC 잔재 0).
- **T3** — `pnpm --filter @moajoa/ios sim` 빌드 통과 (Hermes + SDK 55 + share-intent 6).
- **T4** — 시뮬레이터 스크린샷으로 회귀 확인: `moajoa://boards` 진입 → 지도/핀 렌더, bottom-sheet 열림, gesture 동작. (ios-sim-ui-automation 메모리의 idb 절차 재사용)
- **T5** — `pnpm --filter @moajoa/ios test` (jest 전체) + `tsc --noEmit` 통과.

## Success Criteria (verify)

1. SDK 55 prebuild 후 pnpm sim 빌드 green
2. GMSApiKey / App Group / ShareExtension target 네이티브 보존 (grep 증거) · JSC 잔재 0
3. 지도·핀·bottom-sheet·gesture 시각 회귀 0 (스크린샷)
4. jest 전체 + tsc 통과
5. (육안 게이트) supabase 로그인 런타임 동작 — autonomous:false, 미확정 시 Phase 12/13 EAS 빌드에서 재확인

## 잠금 결정

- prebuild --clean(증분 아님) — RN 메이저 점프라 stale Pod/빌드 산물 제거가 안전
- 네이티브 설정은 app.config.ts 단일 소스에서 재생성 — ios/ 수동 편집 0 목표(grep으로 검증)
- Phase 11 종료 = 이 plan verify 통과 (release/EAS 빌드의 Hermes 최종 확정은 Phase 13)
