(a) `[id].tsx`의 `<Marker>` 렌더에 `source_kind`·`confidence` 분기를 wire — AI high(brand-500/1.0), AI low(brand-500/0.5 + `?` 배지), manual(neutral-900/1.0).

## Why

- TRUST-01: AI vs 수동을 한눈에 — confident-wrong (Pitfall 1) 차단의 1번 시각 layer
- TRUST-04: confidence < 0.7 핀에 사용자가 명시적으로 confirm/reject — AI 신뢰 회복 surface
- D-04 schema 단순성 lock: confirm = source_kind 'ai'→'manual' + confidence null. 별도 컬럼 X.

## Verification Loop

```

2. iOS 보드 상세 진입 — 3 marker 시각 차이 확인 (색 + opacity + ? 배지)
```

```tsx
{places.map((p) => {
  const isAi = p.source_kind === 'ai';
  const isLowConf = isAi && p.confidence !== null && p.confidence < LOW_CONFIDENCE_THRESHOLD;
  const pinColor = isAi ? '#F97316' : '#0F172A';
  const opacity = isLowConf ? 0.5 : 1.0;

      coordinate={{ latitude: p.lat, longitude: p.lng }}

      opacity={opacity}
      onPress={() => setSelectedPlace(p)}
    >

        <View className="w-7 h-9 items-center justify-center">
          <View className="w-6 h-6 rounded-full items-center justify-center" style={{ backgroundColor: 'rgba(249,115,22,0.5)' }}>

  );
})}
```

## Risk / Pitfalls

- **Apple Maps `opacity` prop:** custom children marker가 default pin을 대체. children 활성 시 pinColor는 무시될 가능성 — **시각 의도가 같은 색 + 옅음**이므로 children 안에서 색·opacity 모두 직접 표현하는 게 더 견고. 위 코드는 default pin + children을 함께 두는 형태인데, react-native-maps 동작상 children present 시 default pin이 사라짐 (의도된 결과). 첫 plan 시각 캡처로 확정.
- **`?` 배지 위치:** Marker children은 marker anchor에 정렬. 별도 absolute positioning 없으면 marker 정중앙 — 위 코드의 rounded-full View가 marker가 됨 (default pin 대체). 핀 형태(드롭) 손실 — **trade-off lock:** "low_conf = 의도적으로 약한 시각" 의도에 맞음.
- **onChanged 호출 후 sheet close 순서:** 위 코드는 onChanged() 후 onClose() — sheet가 unmount되는 동안 부모가 reload. UI flicker 가능. v1 허용.
- **`?` 배지가 high conf와 헷갈리지 않게:** opacity 0.5 + 배지 모두 적용 — 시각 cue 중복으로 안전.
