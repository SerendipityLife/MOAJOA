## Goal

Phase 3의 단순 `ActivityIndicator + "분석 중..." text` overlay (apps/ios/app/boards/[id].tsx:202-220)를 broadcast 5단계 step indicator로 augment한다. broadcast `metadata/transcript/llm/places` 수신 시 현재 step을 시각화. `done`/`error`는 Phase 3 동작 그대로 (overlay dismiss + toast).

UI-SPEC §"Component States" 1번 + CONTEXT D-07/D-08/D-09.

## Why

- TRUST-02: 사용자가 추출 진행 중에 "지금 뭐 하는지" falsifiable하게 알아야 함
- Phase 3 spinner는 progress-blind — 5~15초 transcript fetch 동안 사용자 신뢰 ↓
- Karpathy §3.4: "current step === 'llm'을 사용자가 보고 있다"로 검증 가능

### T2 — `_step-indicator.tsx` 신규 컴포넌트 (저-범위 단일 파일)

**File:** `apps/ios/app/boards/_step-indicator.tsx` (NEW — 기존 `_pin-sheet.tsx`/`_pin-add-modal.tsx`와 동일 prefix 패턴)

```tsx
import { EXTRACT_STEP_KO } from '@moajoa/core';
import { ActivityIndicator, Text, View } from 'react-native';

type Step = 'metadata' | 'transcript' | 'llm' | 'places';

const ORDER: Step[] = ['metadata', 'transcript', 'llm', 'places'];

interface Props {
  current: Step | null;
}

/**
 * Phase 5 TRUST-02 step indicator overlay (D-07/D-08).
 *
 * Replaces Phase 3 single-spinner overlay content. Layout:
 *   - ActivityIndicator (always spinning — idle/transition feedback)
 *   - 4 step rows. Future steps: text-xs / font-medium / neutral-300 (D-22).
 *
 * Renders nothing when `current` is null (parent shows the bare overlay container).
 */
export function StepIndicator({ current }: Props) {
  const currentIdx = current ? ORDER.indexOf(current) : -1;

  return (
    <View className="items-center" style={{ gap: 12 }}>
      <ActivityIndicator size="large" color="#F97316" />
      <View style={{ gap: 8 }} className="items-start">
        {ORDER.map((step, idx) => {
          const isDone = idx < currentIdx;
          const isCurrent = idx === currentIdx;
          // Future = neither done nor current.
          const labelClass = isCurrent
            ? 'text-base font-semibold text-brand-500'
            : isDone
              ? 'text-sm text-neutral-500'
              : 'text-xs font-medium text-neutral-300';
          const dotChar = isDone || isCurrent ? '●' : '○';
          const dotColor = isCurrent
            ? 'text-brand-500'
            : isDone
              ? 'text-neutral-500'
              : 'text-neutral-300';
          return (
            <View key={step} className="flex-row items-center" style={{ gap: 8 }}>
              <Text className={`text-base ${dotColor}`}>{dotChar}</Text>
              <Text className={labelClass}>{EXTRACT_STEP_KO[step]}</Text>
            </View>
          );
        })}
      </View>
    </View>
  );
}
```

**Verify:** TypeScript compiles. EXTRACT_STEP_KO type narrows to 4 keys (no `done`/`error`).

## Risk / Pitfalls

- **`pointerEvents="auto"` 유지:** Phase 3 overlay가 사용자 인터랙션 차단 중. step indicator content 교체로 그 동작 영향 X (CONTEXT code_context 명시).
- **`done`/`error` 이후 currentStep 클리어:** 다음 추출이 metadata 받기 전 stale step 표시 방지 위해 cleanup에서 `setCurrentStep(null)`.
- **broadcast 순서 보장 X:** out-of-order metadata가 들어와도 `setCurrentStep`은 단순 상태 — UI가 잠깐 backward 점프 가능. v1 허용 (broadcast는 push-once-per-step, Phase 2 D-02 lock).
