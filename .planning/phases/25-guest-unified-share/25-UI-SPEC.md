# Phase 25 — UI Design Contract

> Visual and interaction contract for 게스트 통합 공유화면(`/t/[slug]`). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **핵심 원칙:** 이 phase는 새 시각 언어를 만들지 않는다. `packages/ui-tokens` 토큰 + 호스트 `moa/[id]` 컴포넌트 + 기존 poll 닉네임 게이트를 **재사용**한다(D-07/D-08/D-09). 아래 계약은 대부분 "무엇을 재사용하는가"와 "게스트 맥락에서 무엇이 달라지는가"를 고정한다.

## Spacing Scale

토큰 `spacing`은 4px 베이스(rem). 이 phase가 쓰는 값(전부 4의 배수):

| Token | Value | Usage |
|-------|-------|-------|
| 1 | 4px | 아이콘-라벨 간격, presence 점 gap |
| 2 | 8px | 버튼 그룹·칩 간격, 리스트 항목 간격(`space-y-2`) |
| 3 | 12px | 카드 내부 패딩(`p-3`), 아바타-텍스트 gap |
| 4 | 16px | 기본 컨테이너 패딩(`px-4`), 섹션 간격, FAB↔시트 오프셋 |
| 5 | 20px | SSR 헤더 세로 패딩(`py-5`) |
| 6 | 24px | 섹션 브레이크(`mt-6`), 집계 블록 상단 여백 |
| 8 | 32px | 출처 섹션·닫힌 화면 대여백 |
| 12 | 48px | 푸터 상단 여백 |

Exceptions:
- **터치 타깃 최소 40px** — 투표 버튼·그리드 셀·닉네임 CTA는 `py-2.5`(10px)+텍스트로 ≥40px 높이 확보(`min-h-10` 그리드 셀 선례). 아이콘-온리 버튼(뒤로·FAB·삭제)은 44px 이상.
- **FAB 56px** — `moa-island` FAB 규격 그대로(brand-600, `shadow-fab`).
- **바텀시트 상단 라운드 28px** (`rounded-t-3xl` = radii `3xl`) — BottomSheet/PlaceSheet 확립값.

## Typography

이 phase의 계약 폰트 세트 (토큰 `typography.sizes`/`weights`/`lineHeights` 참조):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption / meta | 12px (`text-xs`) | 400 regular | 1.25 (tight) |
| Body | 14px (`text-sm`) | 400 regular | 1.5 (normal) |
| Label / emphasis | 15–16px (`text-[15px]`·`text-base`) | 600 semibold | 1.375 (snug) |
| Heading (모아 제목) | 20px (`text-xl`) | 600 semibold | 1.25 (tight) |

**계약 weight = 2개: regular(400) + semibold(600).**

Exceptions (확립된 자산 — 이 phase에서 변경 금지):
- 기존 `/t` SSR `<h1>` 모아 제목은 `font-bold`(700). 게스트 통합 화면에서 SSR 헤더를 유지·재배치하되 **weight는 semibold(600)로 정렬 권장**(호스트 `/moa` 제목과 일치). 700 유지 시 pre-existing 예외로 기록.
- `moa-tab-bar` 탭 라벨은 `font-medium`(500) — 확립된 하단탭 시각언어, 무변경.
- 한글 가독성상 heading은 English 기본보다 tight(1.25) — 토큰 `lineHeights.tight` 사용.

## Color

60/30/10 분배 (토큰 참조):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` white / `#F9FAFB` neutral-50 | 페이지 배경, 카드·시트·말풍선 표면, 지도 컨테이너 밖 여백 |
| Secondary (30%) | `#F3F4F6` neutral-100 / `#F5F7FF` brand-50 | 초대 카드·집계 행·게스트 말풍선(neutral-100)·내 말풍선(brand-50)·선택 안된 버튼 border(neutral-200) |
| Accent (10%) | `#2979FF` brand-500 / `#2563EB` brand-600 | 아래 reserved 리스트 한정 |
| Destructive | `#EF4444` semantic.danger | 게스트 자기 장소/찜 삭제 확인·추출 실패 행 아이콘·에러 토스트 |

**Accent(brand) reserved for (이 목록 외 사용 금지):**
- Primary CTA 배경 (닉네임 게이트 `시작하기`, 닫힌 투표 `이 여행에 함께하기`)
- 활성 탭(`모으기`/`채팅`) 아이콘·라벨 (brand-500)
- 선택된 투표 버튼(`가능`)·선택된 그리드 셀 fill (brand-500)
- 찜(하트) 활성 상태 & 호스트 지도 핀 (brand-500 = loved/owner)
- presence 점, 집계 최다 배지(brand-100 bg / brand-700 text), 진행바 최다(brand-600)
- FAB(brand-600), MOAJOA 브랜드 워드마크/로고 배지

**기능적 팔레트 (accent 아님 — 데이터 인코딩):**
- `colors.member` 6색 `[#FF7043 #AB47BC #26A69A #FFB300 #EC407A #7CB342]` — 게스트 join순 색 배정(D-07). 순번 배지 원·지도 핀·"닉네임님이 담음" 라벨 점에만 사용. `apps/web/lib/member-color.ts` `memberColor()` 소비. **사용자 문자열을 색으로 보간 금지** — 토큰값만.
- 호스트 = brand-500 고정(팔레트 밖).

## Copywriting Contract

기존 verbatim 카피와 정합(Phase 27 NAME-01 카피 스윕 전까지 "찜"/"모아" 표기 유지):

| Element | Copy |
|---------|------|
| Primary CTA (닉네임 게이트 확정) | `시작하기` |
| 닉네임 게이트 heading | `닉네임을 정해주세요` |
| 닉네임 게이트 body | `이 이름으로 담고 투표한 게 표시돼요.` (D-04 첫 참여 액션 공통 — poll의 "누가 가능한지…"는 투표 전용이라 액션-중립으로 조정) |
| 닉네임 입력 placeholder | `닉네임` (maxLength 20) |
| 닉네임 미입력 에러 | `닉네임을 정해야 참여할 수 있어요.` |
| Empty state — 장소 없음 (SSR) | heading `아직 분석 중이에요` / body `잠시 후 다시 열어주세요` (기존 `/t` verbatim) |
| Empty state — 채팅 | `아직 메시지가 없어요` (moa-chat 확립) |
| Empty state — 집계 없음 (dates) | `아직 아무도 투표하지 않았어요` (poll verbatim) |
| Error — 추가 실패 | `추가하지 못했어요. 다시 시도해 주세요` (add-sheet verbatim) |
| Error — 투표 저장 실패 | `투표를 저장하지 못했어요. 잠시 후 다시 시도해주세요.` (poll verbatim) |
| 게스트 기여 라벨 | `{닉네임}님이 담음` (place-list MOA-06 verbatim) |
| 추출 진행 행 (D-11) | `분석 중…` (neutral-400 스피너, place-list 확립) |
| 링크 실패 행 (D-15) | `지원하지 않는 링크예요` (재시도 없음, 24 후속 픽스 verbatim) |
| 계정 승격 진입점 (D-03) | `로그인하고 내 여행에 담기` (기존 `/login` 재사용 — 별도 병합 화면 없음) |

**Destructive actions (D-12 — 게스트는 `added_by = auth.uid`인 자기 장소/찜만):**
- 자기 추가 장소 삭제 — 확장된 아코디언 행 내 삭제 아이콘(danger, 44px 타깃) → **인라인 확인**(아이콘 탭 → 같은 자리에 `삭제할까요?` + `삭제`(danger) / `취소` 노출; 별도 모달 없음, 최소 심). 성공 시 realtime reconcile로 소멸. 실패 토스트 `삭제하지 못했어요. 다시 시도해 주세요`.
- 자기 찜 취소 — 하트 토글(파괴적 확인 불필요, 낙관적+rollback). 남의 기여 하트/삭제는 애초에 렌더 안 함(권한 UI, D-12).

## Phase-Specific Interaction Contracts

> 아래는 `<key_ui_surfaces_to_contract>` 6개 표면의 시각·인터랙션 계약. planner/executor의 구현 진실원본.

### C1. 닉네임 게이트 바텀시트 (D-04/D-05/D-06)
- **표현:** `@/components` BottomSheet (title `닉네임을 정해주세요`, white bg, `rounded-t-3xl`, 핸들 `h-1.5 w-10`). 내부는 poll 닉네임 게이트 입력+CTA 패턴 이식 — `input`(neutral-200 border, `rounded-lg`, `focus:border-brand-300`, maxLength 20) + `시작하기` 버튼(brand-600, `rounded-lg`, semibold, ≥40px).
  - *Assumption:* CONTEXT/key_surfaces가 "바텀시트"로 명시 → poll의 inline-card 대신 BottomSheet로 승격(첫-액션 인터럽트 모달 성격 + 호스트 add/share 시트와 시각 일관). 기록: 오픈 질문 OQ-1.
- **트리거:** 첫 참여 액션(찜/장소·링크 추가/날짜투표) 핸들러 진입부에서만(D-04). 단순 열람은 게이트 없음(D-02 lazy). Enter 키 = 확정.
- **흐름:** 확정 → `signInAnonymously({options:{data:{name}}})` → `join_moa(slug)` → 원래 눌렀던 액션 이어서 실행(끊김 없이). 시트는 성공 시 닫힘.
- **재접속(D-05):** 저장 닉네임 + 익명 세션 지속 시 게이트 생략, 곧장 액션 실행. 닉네임 수정 UI 없음(deferred).
- **중복 허용(D-06):** 닉네임 중복 검증 없음. 신원 구분은 색(member-color)이 담당.

### C2. share_mode별 화면 구성 (D-09)
게스트 통합 `/t/[slug]`는 SSR 브랜드 한 줄 + 초대 카드(기존 A안, `bg-brand-50` 카드) 위에 아래 모드별 인터랙티브 영역을 하이드레이션:

| share_mode | 레이아웃 |
|-----------|---------|
| `places` | 호스트 `moa-island` 재사용 — 지도 + PlaceSheet(place-list) + `[모으기][채팅]` 탭 + FAB. 날짜투표 없음 |
| `dates` | 날짜투표 전면 — poll-vote-island 임베드(range/grid). 상단 초대 카드 + 하단 채팅. 지도/장소리스트 없음 |
| `both` | `[모으기]` 탭 상단에 **날짜투표 섹션**(poll-vote-island 임베드) + 그 아래 장소 리스트. `[채팅]` 탭 공존 |

- 날짜투표는 신규 제작 금지 — 기존 anon poll RPC(`castDateVote`)/컴포넌트 임베드. `device_token` 인자 자리에 익명 `auth.uid` 주입(D-01). 시각(가능/불가 버튼·집계 진행바·최다 배지)은 poll-vote-island verbatim.
- `both` 모드에서 날짜투표 섹션과 장소 리스트 사이 구분: 섹션 헤딩(`text-sm font-semibold text-neutral-700`) + `mt-6` 브레이크. 새 divider 스타일 도입 금지.

### C3. 게스트 통합 화면 = 호스트 컴포넌트 재사용 (D-08)
- `/t/[slug]/page.tsx`의 `VoteIsland`를 **게스트용 moa-island(또는 확장)**로 교체. 레이아웃 셸 그대로: `fixed inset-0`, `max-w-lg` 중앙, 지도 풀채움, PlaceSheet collapsed(~30vh)/expanded(85vh) 드래그, 하단 `MoaTabBar`.
- 게스트도 `[모으기][채팅]` 탭 포함(D-08) — 탭은 라우팅 아닌 클라이언트 상태(`activeTab`), 단일 `moa:{tripId}` 채널 보존(hidden 토글, 언마운트 안 함).
- SSR 헤더(브랜드 한 줄·초대 카드)와 island의 관계: **초대 카드는 게스트 화면 최상단 컨텍스트로 유지**(누가·무엇을 3초 안에), island는 그 아래 인터랙티브 영역. 호스트 `/moa/[id]`엔 없는 게스트 전용 요소.
  - *Assumption:* 초대 카드를 fixed-island 위에 어떻게 배치할지(스크롤 상단 vs 시트 내부)는 planner 재량 — 시각 계약은 "초대 카드 스타일·카피 유지". 기록: OQ-2.

### C4. 게스트 기여 시각 표시 (D-07)
- **순번 배지:** 색 원(`size-6~7`, `backgroundColor = memberColor(added_by, ...)`) + 흰 `seq_no`(place.seq_no 불변, MOA-02/Pitfall 9). 게스트 추가 장소는 서버 채번으로 이어지는 #N+1(SC4).
- **기여 라벨:** `{닉네임}님이 담음`(place-list verbatim), 색 점 동반.
- **색 배정:** 호스트=brand-500 고정, 게스트=join(created_at asc)순 member 6색 순환. `listTripMembers` + `memberColor()` 소비. 신규 색 도입 0.
- **실시간 반영(SC4):** 게스트 쓰기가 호스트와 같은 `moa:{tripId}` 채널·테이블 → 호스트 화면 자동 반영. 추가 UI 없음(채널 재사용).

### C5. 게스트 권한 UI (D-11/D-12)
- **링크 추가 행(D-11):** 게스트 링크 추가 → `join_moa`로 멤버 → 자동 추출 fire-and-forget. 진행 표시 = `분석 중…` neutral-400 스피너 행(place-list D-13 패턴 재사용). 실패 = AlertCircle danger + `지원하지 않는 링크예요`.
- **삭제 권한(D-12):** 행의 삭제 아이콘은 `added_by === 내 auth.uid`일 때만 렌더. 찜 하트는 항상 렌더(내 찜만 토글, 남의 찜 카운트는 열람). 남의 기여엔 파괴적 어포던스 0 — 권한을 UI 부재로 표현.

### C6. 계정 승격 진입점 (D-03 — 최소 심)
- 별도 병합/승격 화면 **없음**. 게스트 화면 어딘가(초대 카드 하단 or 헤더)에 기존 `/login` 재사용 진입점 1개: `로그인하고 내 여행에 담기`(secondary 스타일 — brand text/outline, primary CTA와 경합 방지).
- 클릭 → 기존 카카오 로그인 → `linkIdentity`로 익명 uid 정식 전환(백엔드). UI는 진입점만, 충돌 처리·병합 뷰는 deferred.
  - *Assumption:* 진입점 위치/문구는 최소. planner가 초대 카드 vs 상단바 중 택1. 기록: OQ-3.

## Open Questions / Assumptions

오케스트레이션 논블로킹 규칙에 따라 아래는 기존 디자인 시스템 근거로 합리적 가정을 세우고 진행. planner가 확정 가능:

- **OQ-1 (닉네임 게이트 형태):** BottomSheet로 가정(CONTEXT "바텀시트" 문구 + 호스트 시트 일관성). poll의 inline-card 유지도 가능 — 시각 토큰은 동일.
- **OQ-2 (초대 카드 배치):** fixed island 화면에서 SSR 초대 카드를 상단 스크롤 영역 vs 시트 헤더 내부 중 어디 둘지. 스타일(`bg-brand-50` 카드)·카피는 계약 고정, 배치는 planner 재량.
- **OQ-3 (승격 진입점 위치):** 초대 카드 하단 vs 상단바. 최소 심이라 어느 쪽이든 1개 진입점.
