## Typography

기존 `ui-tokens/src/index.ts` `typography` 토큰을 underlying source로 사용. 토큰 raw scale은 5 sizes / 4 weights를 유지하지만, **Phase 3 active 사용은 4 sizes / 2 weights로 좁힌다** (checker rule 준수). NativeWind Tailwind class로 접근.

### Active set (Phase 3)

| Role | Class | px equivalent | Weight | Phase 3 Usage |
|------|-------|---------------|--------|---------------|
| Display (보드 제목 큰 헤더) | `text-2xl` | 24px | `font-semibold` (600) | boards.tsx의 "내 보드" (기존 유지) |
| Heading (보드 상세 헤더, modal 타이틀, bottom sheet 장소명) | `text-lg` | 18px | `font-semibold` (600) | [id].tsx의 board.title (기존 유지), "핀 추가" modal 헤더, bottom sheet `place.name_local` |
| Body (기본 본문, 토스트, 버튼 라벨, 검색 결과 표시명, 액션 버튼) | `text-base` | 16px | `font-regular` (400) — 기본; `font-semibold` (600) — primary CTA · 화면 헤더 · list item 강조 타이틀 | toast 메시지, modal 결과 행 메인, bottom sheet 액션 라벨, primary CTA 본문 |
| Label / Caption (메타 정보, helper, source_kind badge, 보조 설명) | `text-sm` | 14px | `font-regular` (400) — 기본; `font-semibold` (600) — list item title (보드 이름) | URL helper text, list item 주소·메타, source_kind badge "AI" / "수동", "검색 중...", banner 텍스트, chevron `›` 표시 |

**Active total: 4 sizes (24/18/16/14), 2 weights (400 regular + 600 semibold).**

### Tokens available but NOT used in Phase 3

| Token | Reason held out | Future use |
|-------|-----------------|------------|
| `text-xs` (12px) | Phase 3 active scale에서 제외 — 모든 12px 후보 use site를 14px로 reassign | **Phase 5 Trust UI**에서 low_confidence 라벨·미세 메타 위계에 revisit 가능 |
| `font-medium` (500) | Phase 3 active scale에서 제외 — list item title은 600으로 promote, 본문/메타/토스트는 400으로 fold | **Phase 5 Trust UI**에서 low_confidence vs confirmed 텍스트 위계 필요 시 도입 |
| `font-bold` (700) | Phase 3 사용 안 함 (Phase 1 D-08 — OG 이미지 한정) | Phase 4 (web OG 이미지 렌더) |

### Weight assignment policy

- **400 regular** (기본): 본문 텍스트, helper/caption, placeholder, toast 메시지, 검색 결과 주소, source_kind badge 텍스트, list item 메타 (예: "링크 N개 · 장소 M개"), banner 본문, modal 검색 input.
- **600 semibold** (강조): primary CTA 버튼 라벨, 화면 헤더 ("내 보드", board.title, modal "핀 추가"), bottom sheet 장소명, **list item title (보드 이름, 핀 장소명)** — 이전 500 medium 사용 site에서 promote.

### Line height policy

- 본문 (`text-base`, `text-sm`) → `leading-normal` (1.5) — Korean 가독성
- 헤더 (`text-2xl`, `text-lg`) → `leading-tight` (1.25)
- Toast 한 줄이므로 lineHeight 명시 불필요

### Reassignment audit (from 12px → 14px and 500 → 400/600)

이전 spec에서 12px 또는 500 medium을 가정했던 모든 use site를 아래로 매핑.

| Use site | Before | After |
|----------|--------|-------|
| source_kind badge "AI" / "수동" | `text-xs font-medium` (12 / 500) | `text-sm font-regular` (14 / 400). **Badge 시각적 비대화 방지를 위해 padding을 `px-1.5 py-0.5`로 축소**, rounded-md 유지 |
| URL helper "유튜브 — 자동 분석 시작" | `text-xs text-neutral-500` (12 / 400) | `text-sm text-neutral-500` (14 / 400) |
| Banner "저장 실패 N개" chevron `›` | `text-neutral-400 text-sm` (이미 14) | (변동 없음) |
| Banner 본문 텍스트 | `text-sm` (이미 14) | (변동 없음) |
| "검색 중..." status | `text-xs text-neutral-500` (12 / 400) | `text-sm text-neutral-500` (14 / 400) |
| 검색 결과 행 보조 (formattedAddress) | `text-xs text-neutral-500 mt-0.5` (12 / 400) | `text-sm text-neutral-500 mt-0.5` (14 / 400) |
| 검색 결과 행 메인 (displayName) | `text-base font-medium` (16 / 500) | `text-base font-regular` (16 / 400) — 위계는 16 vs 14 + 색 대비로 유지 |
| List item title (보드 이름 / 핀 장소명) | `font-medium` (500) | `font-semibold` (600) — 강조 유지 |
| List counter "링크 N개 · 장소 M개" | `font-medium` (500) | `font-regular` (400) — 메타이므로 강조 X |
| Primary 버튼 라벨 ("추가", "로그인", "핀 추가" modal CTA) | `text-base font-medium` (16 / 500) | `text-base font-semibold` (16 / 600) — CTA 강조 |
| Bottom sheet 액션 버튼 라벨 ("이름 수정" / "영상에서 위치 보기" / "삭제") | `text-base font-medium` (16 / 500) | `text-base font-regular` (16 / 400) — secondary 액션은 400. destructive "삭제"는 `text-danger`로 색 대비. |
| Toast 텍스트 | `text-sm font-medium` (14 / 500) | `text-sm font-regular` (14 / 400) — `text-white` + danger 배경으로 대비 확보 |

### Hierarchy verification (Dimension 1 — copy)

- **source_kind badge "AI" / "수동" at 14 / 400 with `bg-neutral-100 text-neutral-700`:** 14px는 본문과 동일 크기지만 `px-1.5 py-0.5` + rounded + 옅은 회색 배경으로 chip 형태가 유지됨. 시각적 무게는 색·배경으로 분리.
- **Bottom sheet handle ("───") + 장소명 (18 / 600) + 주소 (14 / 400) + badge (14 / 400):** 18 → 14 size step (Δ4px)과 600 → 400 weight step으로 위계가 명확. 주소와 badge는 색(neutral-600 vs neutral-700) + 위치(stack 순서)로 분리.
- **List item title (16 / 600) vs item meta (14 / 400):** size step 2px + weight step 200 — 양쪽 모두로 분리되어 강한 위계.
- **Banner "저장 실패 2개 — 탭하여 확인" (14 / 400) + chevron "›" (14 / 400, `text-neutral-400`):** chevron은 색으로만 약화. 두 요소 모두 14 / 400이지만 의미 단위가 분리되어 충돌 없음.

→ checker Dimension 1 (copy clarity / 위계) 재검 통과 예상.

## Color

기존 `ui-tokens` 토큰 그대로. iOS는 라이트 모드만 (다크 모드는 v2 — REQUIREMENTS.md THEME-01).

|------|-------|------------------|-----|---------------|
| Dominant (60%) | `neutral.0` | `bg-white` | #FFFFFF | 모든 화면 배경 (`SafeAreaView className="flex-1 bg-white"` 기존 유지) |
| Secondary (30%) | `neutral.50` ~ `neutral.300` | `bg-neutral-50` / `border-neutral-200` / `text-neutral-500/600/700` | #F8FAFC / #E2E8F0 / #64748B | 카드 border, list item, helper text, bottom sheet 배경 (`bg-neutral-50`), 입력 border |
| Accent (10%) | `brand.500` | `bg-brand-500` / `text-brand-500` | #F97316 | **다음 4가지에만 사용** (아래 reserved list) |
| Destructive | `semantic.danger` | `bg-danger` / `text-danger` / `border-danger` | #DC2626 | 핀 삭제 confirm 버튼, "저장 실패" 시스템 보드 dot, error toast 배경 |
| Success (feedback only) | `semantic.success` | `text-success` | #16A34A | "X개 핀 추가됨" done toast 좌측 아이콘 (옵션) |

**Accent (`brand-500`) reserved for:**
1. **Primary CTA** — "추가" 버튼 (URL add), "새 보드" 버튼, "+ 핀 추가" FAB, modal "추가" 버튼, "MOAJOA 앱 열기" deeplink 버튼 (Share Extension 안내)
2. **Active selection feedback** — list item `active:bg-brand-50` (이미 [id].tsx와 boards.tsx에서 사용 중)
3. **Back arrow / navigation hint** — 보드 상세 `text-brand-500` "← 뒤로" (기존 유지)
4. **Spinner color** — 추출 진행 spinner (`ActivityIndicator color={brand-500}`)

> Bottom sheet의 [이름 수정], [영상에서 위치] 버튼은 **`brand-500` 사용 X** — `bg-neutral-100` + `text-neutral-800` 사용 (secondary 액션). `[삭제]`만 destructive 색.
> Manual 핀 vs AI 핀의 시각 차별화 색(점선·옅은 색)은 **Phase 5 Trust UI에서 도입** (Phase 3 scope lock).

**Source kind badge (bottom sheet 안 D-09):**
- `source_kind='ai'` → `bg-neutral-100 text-neutral-700` + label "AI" (text-sm / 400, `px-1.5 py-0.5 rounded-md`)
- `source_kind='manual'` → `bg-neutral-100 text-neutral-700` + label "수동" (동일)
- 색 차별화 X (Phase 5에서 시각 분리). Phase 3는 텍스트 label로만 구분.

| Result item | `flex-row items-start px-6 py-3 border-b border-neutral-100 active:bg-neutral-50` | 우측 텍스트 stack: `text-base text-neutral-900` (16 / 400, `displayName`) + `text-sm text-neutral-500 mt-0.5` (14 / 400, `formattedAddress`). 좌측 핀 아이콘 (Map pin lucide / emoji 📍) `mr-3 mt-1`. **위계는 size step (16 vs 14) + color step (neutral-900 vs neutral-500)로 확보** |

| Tap action | `resolve-place` 응답을 `addManualPlace` helper에 넘김 → 성공 시 modal dismiss → boards/[id] reload → "수동 핀 추가됨" toast 1초 노출 | toast: `"핀 추가됨"` |

### 5. Boards List (D-06 — 시스템 보드 표시)

기존 `boards.tsx`에 "저장 실패" 시스템 보드 영역 추가.

| Element | Class | Behavior |
|---------|-------|----------|
| 시스템 보드 banner (저장 실패 N개 있을 때만) | `mx-6 mb-4 bg-danger/5 border border-danger/20 rounded-lg px-4 py-3 flex-row items-center` | 좌측 빨간 dot `w-2 h-2 rounded-full bg-danger mr-3` + 텍스트 `text-sm text-neutral-800 flex-1` (14 / 400) `"저장 실패 {N}개 — 탭하여 확인"` + 우측 chevron `text-neutral-400 text-sm` (14 / 400) `"›"` |
| 시스템 보드 진입 | 탭 시 router.push('/boards/_failed') 또는 modal | 실패 큐(`pending_links_failed`) 리스트 + 각 항목에 `[재시도]` `[삭제]` 버튼 |

**List item title 통일:** 보드 카드의 보드 이름은 `text-base font-semibold text-neutral-900` (16 / 600). 보드 메타("링크 N개 · 장소 M개")는 `text-sm text-neutral-500` (14 / 400).
