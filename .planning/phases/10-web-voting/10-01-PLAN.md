```sql
create or replace function public_board_view(p_slug text)
returns jsonb
language plpgsql
stable
security definer
set search_path = public
as $$
declare
  v_board boards%rowtype;
  v_owner profiles%rowtype;
  v_result jsonb;
begin
  select * into v_board from boards
  where share_slug = p_slug and visibility = 'public'      -- CHANGE (ii): broaden to in ('public','shared')
  limit 1;
  if not found then return null; end if;
  select * into v_owner from profiles where id = v_board.owner_id;
  v_result := jsonb_build_object(
    'board', jsonb_build_object(
      'id', v_board.id,                                      -- CHANGE (i): already present in 0008 — keep it
      'title', v_board.title,
      'description', v_board.description,
      'city_code', v_board.city_code,
      'cover_image_url', v_board.cover_image_url,
      'updated_at', v_board.updated_at
    ),
    'owner_display_name', coalesce(v_owner.display_name, 'MOAJOA user'),
    'links', coalesce( (select jsonb_agg(jsonb_build_object(
        'id', l.id, 'source_kind', l.source_kind, 'url', l.url, 'title', l.title,
        'thumbnail_url', l.thumbnail_url, 'author_name', l.author_name, 'summary_ko', l.summary_ko
      ) order by l.created_at desc)
      from links l where l.board_id = v_board.id and l.extraction_status = 'ready'), '[]'::jsonb ),
    'places', coalesce( (select jsonb_agg(jsonb_build_object(
        'id', p.id, 'link_id', p.link_id, 'name_local', p.name_local, 'name_ko', p.name_ko,
        'name_en', p.name_en, 'lat', p.lat, 'lng', p.lng, 'category', p.category,
        'source_timestamp_sec', p.source_timestamp_sec, 'source_kind', p.source_kind,
        'confidence', p.confidence, 'summary_ko', p.summary_ko
      ) order by p.created_at)
      from places p where p.board_id = v_board.id and p.hidden_at is null), '[]'::jsonb )
  );
  return v_result;
end;
$$;
grant execute on function public_board_view(text) to authenticated, anon;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon/auth client → join_shared_board RPC | A logged-in user submits a slug to self-add to a board they don't own |
| anon client → public_board_view RPC | Any unauthenticated viewer resolves a slug to board title/pins |
| anon/auth client → accepted_member_count RPC | Any client reads the member count for any board id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Elevation of Privilege | join_shared_board (self-add to board) | mitigate | Only `visibility in ('shared','public')` boards joinable; role hard-coded `'voter'` (never owner/editor); private boards never resolve. This IS the intended bearer-invite model — slug possession = invite. |
| T-10-02 | Elevation of Privilege | join_shared_board role escalation | mitigate | `role='voter'` is a literal in the INSERT; no caller-supplied role param exists. Cannot escalate to editor/owner. |
| T-10-03 | Tampering | duplicate / spam memberships | mitigate | `on conflict (board_id, user_id) do nothing` — idempotent; the unique(board_id,user_id) constraint caps one membership per user per board. |
| T-10-04 | Spoofing | injection of another user's membership | mitigate | `user_id = auth.uid()` (not a parameter); a caller can only join AS themselves. |
| T-10-05 | Elevation of Privilege | search_path injection on DEFINER funcs | mitigate | All three functions `set search_path = public` — no schema shadowing of memberships/boards. |
| T-10-06 | Information Disclosure | shared-board title/pins exposed to slug-holders | accept | `public_board_view` broadened to `'shared'` deliberately — bearer-invite model (CONTEXT B). Slug entropy ~60 bits (ensure_share_slug, 0001 lines 117-133). Human sanity-check gated in 10-03. No PII beyond owner display name (already public). |
| T-10-07 | Denial of Service | brute-force slug guessing to join boards | accept | 8-32 char unique slug, ~60-bit entropy; guessing is infeasible. No rate-limit added (out of scope; Supabase platform throttles). |
| T-10-08 | Spoofing / bypass | island casting votes without membership | mitigate | Voting RLS (`votes: insert if can vote in board` → `can_vote_board`, 0001 lines 451-460) already enforces membership server-side; join_shared_board is the only path to membership. The island cannot bypass — castVote hits RLS. |
</threat_model>
