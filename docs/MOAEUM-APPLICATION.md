## Q1. 아이디어를 떠올린 배경 이야기를 들려주세요.

**[3박 4일을 준비하는데 앱을 열 개 열었습니다]**
도쿄 여행 하나를 준비하며 깨달았습니다. 유튜브에서 맛집을 보고 캡처하고, 지도에 일일이 옮겨 적고, 친구와 카톡으로 링크를 주고받으며 어디 갈지 정하고, 숙소는 비교 사이트 세 곳을, 입장권과 유심은 또 다른 앱을 돌았습니다. 예약이 끝나면 항공·호텔·티켓 확정 메일과 영수증이 메일함에 뿔뿔이 흩어졌고, 어느 카드로 얼마를 냈는지, 달러 결제는 환율이 얼마나 바뀔지, 잔금은 언제 빠지는지 아무도 정리해 주지 않았습니다.

**[발견은 폭발했는데, '실행'은 10년 전 그대로입니다]**
콘텐츠 덕분에 "여기 가고 싶다"를 발견하는 일은 너무 쉬워졌습니다. 그러나 그 발견을 실제 '내 여행'으로 바꾸는 과정 — 모으고, 정하고, 짜고, 예약하고, 정산하는 일 — 은 여전히 수십 개의 앱과 수작업으로 쪼개져 있습니다. 발견과 실행 사이의 이 거대한 간극이, 설레야 할 여행 준비를 노동으로 만듭니다.

**[이제는 AI로 이 전 과정을 자동화할 수 있습니다]**
최근의 AI는 영상·블로그 같은 비정형 콘텐츠에서, 또 제각각인 예약 메일에서 '장소·일정·금액' 같은 구조화된 정보를 안정적으로 뽑아냅니다. 사람이 직접 옮겨 나르던 일을 기계가 대신할 수 있게 된 것입니다. MOAJOA는 이 기술을 무기로, 여행의 시작(발견)부터 끝(정산)까지를 자동으로 잇고, 그렇게 쌓인 데이터로 다음 여행을 더 똑똑하게 만드는 서비스입니다.

## Q2. 아이디어는 누구의 어떤 문제를 해결해주나요?

MOAJOA는 콘텐츠를 보며 여행을 직접 설계하는 2030 자유여행객, 특히 한국인 해외 자유여행객을 시작 고객으로 삼아 여행의 전 주기에 걸친 네 가지 문제를 해결합니다.

**첫째, 흩어진 발견을 한곳에 모읍니다.** 유튜브·인스타·블로그 링크를 던지면, 영상과 글 속 장소가 좌표가 찍힌 지도 보드로 자동 정리됩니다. 더 이상 캡처하고 손으로 옮겨 적을 필요가 없습니다.

**둘째, 함께 정하는 과정을 단순하게 만듭니다.** 보드를 친구와 공유해 실시간으로 투표하고, 가고 싶은 곳을 함께 '확정'합니다. 카톡에 링크를 흩뿌리며 "여기 어때?"를 반복하던 결정의 피로를 없앱니다.

**셋째, 결정에서 멈추지 않고 예약까지 잇습니다.** 확정한 장소로 동선과 일정을 자동 설계하고, 여러 예약 플랫폼의 가격을 비교해 한 화면에서 숙소·액티비티·교통·유심을 예약합니다.

**넷째, 가장 답답했던 '정산'을 자동화합니다.** 예약 확정 메일을 전달하기만 하면, 어느 카드·어느 플랫폼·원화/달러·환율 차이·결제 시점까지 정리된 '여행 가계부'가 완성됩니다. 우리 앱을 거치지 않은 예약(직접 예약한 항공권 등)까지 메일만 오면 함께 정리되어, 비로소 여행의 모든 지출이 한곳에 모입니다.

이렇게 쌓인 실제 예약·사용 데이터는 다음 여행자의 추천을 더 정확하게 만듭니다. 한국인 여행자의 데이터를 시작으로 일본 현지인 및 타 국가로 확장해, 로컬 경험을 잇는 글로벌 여행 생태계를 지향합니다.

## Q3. 아이디어를 어떻게 실현하고 싶으신가요?

**[제품 — 발견부터 정산까지의 자동화 레이어]**
MOAJOA는 분절된 여행 과정을 하나의 흐름으로 잇습니다.
콘텐츠 링크 → ① AI 장소 자동 추출 → ② 친구와 투표·결정 → ③ 동선·일정 자동 설계 → ④ 멀티플랫폼 가격비교 → ⑤ 제휴 예약·결제 → ⑥ 예약 메일 전달 시 AI가 파싱해 여행 가계부로 자동 정리. 여행 시작 시점에는 "일정이 정해졌으면 바로 입력, 미정이면 일행과 날짜 투표"로 일행과 일정부터 맞춥니다.

**[핵심 — 데이터가 복리로 쌓이는 선순환]**
이 흐름이 한 바퀴 돌 때마다 실제 예약·사용 데이터가 쌓이고, 이 데이터가 다음 여행자의 발견과 추천을 더 정확하게 만듭니다. 사용자가 늘면 데이터가 늘고, 데이터가 늘면 추천이 좋아져 다시 사용자가 느는 자기강화 구조입니다. 특히 여행을 마친 뒤의 '가계부(기록)'가 다음 여행의 '발견'을 끌어내, 여행이라는 저빈도 서비스의 최대 약점인 재방문을 구조적으로 해결합니다.

**[수익 모델 — 수익화는 미래가 아니라 MVP의 기본 동작입니다]**
MOAJOA의 강점은 '여행에서 실제로 돈이 쓰이는 순간(예약·결제)'을 제품의 한가운데 두었다는 점입니다.
- **거래 수수료 (핵심 · MVP 첫날부터):** 자동 설계된 일정에 맞춰 숙소·액티비티·교통·유심을 연결합니다. Travelpayouts·Stay22 같은 글로벌 제휴 네트워크의 딥링크를 활용하면 별도 인프라나 선투자 없이 출시 즉시 수수료가 발생합니다. (숙소 3~5%, 액티비티·티켓 5~8%, 유심은 그 이상) — 여행 정리 앱 TripIt, 경비 앱 Expensify가 이미 검증한 모델입니다.
- **프리미엄 구독 (상시):** 항공 게이트·지연, 공항 주차 혼잡도 같은 여행 당일 실시간 알림, 고급 일정 최적화, 광고 제거.
- **로컬 상점 광고 (데이터 밀도 확보 후):** 실제 예약·결제 데이터를 기반으로 한 맥락 광고.
- **여행 행동 데이터 (장기):** 고순도 인텐트 데이터를 관광청·지자체·여행사 대상 B2B로.

**[실행 로드맵]**
- **1단계(MVP · 도쿄):** AI 자동 추출로 인기 일본여행 콘텐츠를 사전 수집해 '첫날부터 쓸 만한 지도'를 채웁니다(운영 인력 없이 데이터 밀도와 확장성을 동시에 확보). 동시에 자동 플랜 + 딥링크 예약으로 첫 매출을, 예약 메일 전달 기반 가계부로 재방문 고리를 검증합니다.
- **2단계:** 항공·주차 실시간 레이어(프리미엄), 가격비교 API 고도화, 로컬 상점 광고 파일럿.
- **3단계:** 도시·결제경로 확장, 데이터 라이선싱, 일본 성공 모델의 동남아·유럽 이식.

**[실현 가능성과 차별점]**
장소 추출에 쓰는 AI를 예약 메일 파싱에도 그대로 재활용해 추가 개발 부담이 적고, 공항 주차 혼잡도 등은 무료 공공데이터를 활용해 운영 비용이 낮습니다. 결제를 하는 대표가 어떤 기기를 쓰든 막히지 않도록 iOS·Android·Web을 함께 지원합니다. 무엇보다 '발견 → 결정 → 예약 → 정산 → 데이터 축적'의 전 과정을 닫는 선순환은, 한두 단계에만 머무는 경쟁 서비스가 쉽게 모방할 수 없는 해자가 됩니다.

모두의 창업을 통해 초기 사용자 확보(GTM) 전략을 정교화하고, 검증된 수익 모델을 더 빠르게 실행하기 위한 전문가 멘토링을 희망합니다.
