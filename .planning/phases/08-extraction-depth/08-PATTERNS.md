## CRITICAL DATA-FLOW FINDING (read first)

`summary_ko` reaching the web is a **4-link chain that must ALL change in lockstep**.
Miss any one and the field silently never reaches the browser:

1. **DB column** — `0008` migration adds `places.summary_ko` / `links.summary_ko`.
2. **RPC `public_board_view`** — `supabase/migrations/0006_trust_ui_onboarding.sql:45-84`
   builds the response with an **EXPLICIT `jsonb_build_object` field list**. It does
   NOT `select *`. A column not named there is invisible to web. → `0008` MUST
   `create or replace function public_board_view` re-issuing the full body with
   `'summary_ko', p.summary_ko` added to the places object and `'summary_ko', l.summary_ko`
   to the links object. (This is exactly the append pattern 0006 used for `source_kind`/`confidence`.)
3. **`PublicBoardView` TS type** — `packages/core/src/types/index.ts:25-43` shapes the
   view as `Pick<Place, ...>` / `Pick<Link, ...>`. `summary_ko` must be added to BOTH
   Pick unions or the web sees `unknown`/compile error.
4. **Generated `database.ts`** — user runs `pnpm supabase:types` (CLAUDE.md §4.3). Not hand-edited.

The query layer `getPublicBoardBySlug` (`packages/api/src/queries/boards.ts:71-78`) does
`return (data as PublicBoardView | null)` — a raw cast, **zero field shaping**. So whatever
the RPC jsonb emits flows straight to `view.places[i].summary_ko`. No change needed there
beyond the type it casts to.

iOS exposure is automatic per CONTEXT (server-side); this phase's UI work = web only (VIEW-08).

## Shared Patterns

### Optional-vs-nullable idiom (the one rule to get right everywhere)
- **LLM-side / candidate schemas** (`claude.ts:11-23`, `place.ts:81-90`): `.optional()` → `undefined`.
- **Persisted / row schemas** (`place.ts:13-67`, `link.ts:4-24`): `.max(N).nullable()` → DB null.
- **Bridge at the write site** (`index.ts:207-214`): `r.cand.field ?? null`.
Apply this trichotomy to `summary_ko` exactly as the codebase already does for `source_quote`/`name_ko`.

### Error-handling contract (CONTEXT-LOCKED: summary never fails extraction)
**Source:** `index.ts` — summary is derived from the already-parsed `candidates`/`rows`; it
introduces NO new try/catch and NO new failure branch. Missing summary degrades to `null` via
`?? null`, never throws. **Apply to:** claude.ts, index.ts. Do not add defensive code (Karpathy §3.2).

### RPC explicit-field contract (the silent-failure trap)
**Source:** `0006_trust_ui_onboarding.sql:45-84` — `public_board_view` lists every web-visible
field by hand in `jsonb_build_object`. **Apply to:** 0008 migration MUST re-issue it with
`summary_ko` added, or the column never reaches web regardless of #6.

### Conditional render (CONTEXT-LOCKED: legacy rows must not break layout)
**Source:** `page.tsx:124-126` (`{link.author_name && (...)}`), `page.tsx:80-82`
(`{view.board.description && (...)}`). **Apply to:** every new summary `<p>`.
