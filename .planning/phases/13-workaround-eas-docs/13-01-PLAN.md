## 사용자 EAS 실기기 share-sheet UAT 체크리스트

> 목적: (1) **release/Hermes 빌드에서 supabase OTEL 회귀 0 최종 확정**(로컬 Debug는 Metro+Hermes로 검증됨, hermesc 바이트코드 빌드는 EAS가 처음) + (2) v1.1 잔여 실기기 share-sheet 추출 트리거.

1. `npm i -g eas-cli` (또는 `npx eas-cli@latest`) → `eas login` (Expo 계정).
2. `cd apps/ios && eas build -p ios --profile development` (클라우드 빌드 ~15분; 실폰 등록 안 됐으면 `eas device:create`로 UDID 등록 선행).
   - **여기서 hermesc가 supabase-js OTEL로 깨지면** → app.config.ts 주석대로 babel transform fallback 적용 후 재빌드(Phase 11 11-01 노트).
3. 빌드 산출물(.ipa 또는 QR)로 실폰 설치 → 앱 실행, 로그인까지 정상 동작 확인(release Hermes 런타임).
4. 카톡/사파리/유튜브 앱에서 **공유 시트 → "MOAJOA 저장"** 선택 → youtube/blog/insta 링크 1건씩.
5. 메인 앱 진입 시 추출이 발화(보드에 핀/진행상태)하는지 확인 → **통과 시 UPGRADE-05 + v1.1 잔여 종료.**
6. 결과를 `.planning/STATE.md`(또는 docs/SESSION-NOTES)에 PASS/FAIL 기록.
