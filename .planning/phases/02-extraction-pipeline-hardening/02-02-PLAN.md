## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output -> application | Claude may produce hallucinated places without transcript evidence |
| Edge Function -> Supabase Realtime | Broadcast payload visible to channel subscribers |
| Edge Function -> extraction_costs | Cost data written via service role |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | LLM output (claude.ts) | mitigate | source_quote required in Zod schema + post-parse filter in index.ts discards entries without transcript evidence (D-04) |
| T-02-05 | Information Disclosure | Realtime broadcast payload | accept | Payload contains only step name, progress percentage, and places_extracted count. No PII or secrets. Channel scoped to link_id which requires auth to know. |
| T-02-06 | Denial of Service | Broadcast/cost logging failure cascading to extraction failure | mitigate | Both broadcastStep and logCost wrapped in try/catch -- failures logged but never block extraction pipeline |
| T-02-07 | Spoofing | Unauthenticated broadcast subscription | accept | Realtime channels are subscribable by anyone who knows the link_id. However, broadcast only sends progress status (no sensitive data). Auth is enforced at extraction trigger level (JWT required in index.ts line 56-59). |
</threat_model>
