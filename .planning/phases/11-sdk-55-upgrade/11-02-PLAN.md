# Plan 11-02: SDK 55 lockstep 버전 상향

**Phase:** 11 (SDK 54 → 55)
**Wave:** 2 (depends 11-01 — Hermes 검증 후)
**autonomous:** true
**Requirements:** UPGRADE-01(부분)

## 목표

`apps/ios`의 expo + react-native + react + 전체 expo-* / RN 생태계 의존성을 **SDK 55 호환 버전으로 lockstep 상향**한다. 버전 핀은 하드코딩하지 않고 **`expo install` 리졸버**에 위임(SDK별 정확 버전은 expo가 결정).

## Tasks

- **T1** — `apps/ios/package.json`의 `expo`를 `^55`(또는 latest 55 라인)로 올리고 `pnpm --filter @moajoa/ios exec expo install expo@^55` 실행.
- **T2** — `pnpm --filter @moajoa/ios exec expo install --fix` 로 react-native(0.83)/react(19.2)/reanimated/worklets/gesture-handler/screens/safe-area-context/maps/async-storage/expo-* 모듈 일괄 정렬.
- **T3** — `expo-share-intent` `^5.1.1` → `^6.x`(SDK 55 호환 라인). app.config.ts plugin 옵션 shape(iosAppGroupIdentifier/iosShareExtensionName/iosActivationRules) 변경 여부 확인 — 변경 시 surgical 패치.
- **T4** — devDeps 상향: `jest-expo ~55`, `eslint-config-expo`(SDK 55 라인), `@types/react ~19.2`. `react-native-css-interop`/`nativewind`는 expo install --check 결과 따라 정렬.
- **T5** — `pnpm install` 후 `npx expo install --check`(@moajoa/ios) 가 **잔여 불일치 0** 보고하는지 확인. 남으면 수동 핀.
- **T6** — `tsc --noEmit` 통과 (타입 회귀 확인 — RN 0.83/React 19.2 타입 변화 흡수).

## Success Criteria (verify)

1. package.json이 expo ~55 / react-native 0.83 / react 19.2로 상향되고 expo-share-intent 6.x
2. `expo install --check` 잔여 불일치 0
3. `tsc --noEmit` exit 0

## ⚠️ 실행 노트 (2026-06-12 1차 시도 학습 — 다음 시도 시 반영)

1차 시도에서 두 가지 함정 발견 → 롤백 후 11-01 클린 상태로 복원. 다음 시도는 아래대로:

- **`expo install expo@^55`는 메이저를 안 넘는다.** SDK 54 CLI의 `expo install`은 현재 설치된 expo의 bundledNativeModules 기준으로 해석해 `~54.0.35`(54 최신 패치)로만 올림. **올바른 bump:** `pnpm --filter @moajoa/ios add expo@~55.0.26`(실제 55 버전 직접 핀) **후** `expo install --fix`로 나머지 정렬. T1을 이 명령으로 교체할 것.
- **`expo install --fix`를 백그라운드로 반복 실행 금지.** 좀비 프로세스가 누적되어 node_modules/lockfile 동시 경합 → 상태 오염. **포그라운드 1회**만, 끝까지 대기.
- **롤백 절차(검증됨):** `git checkout HEAD -- apps/ios/package.json pnpm-lock.yaml` → `pnpm install --frozen-lockfile` → expo 54.0.34 + jest 38/38 복원 확인.
- `pnpm install`(plain)은 범위 내 최신 패치로 lockfile을 드리프트시킴 → 정확 복원엔 항상 `--frozen-lockfile`.
